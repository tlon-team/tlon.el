This manual describes the features and customization options for the Emacs Lisp file =tlon-local.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

The =tlon-local= module manages local development environments for the Uqbar website. It handles the full lifecycle of a local stack: ensuring Docker Desktop is running, starting the web server (Traefik) and the Uqbar application for a chosen language, and opening the local site in your browser when the stack is ready.

The module also provides a log viewer that queries Grafana's Loki proxy for the latest content build, aggregating errors, warnings, informational messages, and frontend (NextJS) errors into a single navigable buffer. Log entries display expanded file paths that you can visit directly, and a follow mode keeps the corresponding source file visible as you move through the log.

All operations are accessible through a transient menu ([[#h:transient-menu][transient menu]]).

* User options
:PROPERTIES:
:CUSTOM_ID: h:user-options
:END:

** Log query parameters
:PROPERTIES:
:CUSTOM_ID: h:log-query-parameters
:END:

#+vindex: tlon-local-logs-minutes
The user option ~tlon-local-logs-minutes~ controls the lookback window (in minutes) used when querying logs from Loki. Increase this value if you need to inspect logs from builds that ran further in the past. The default is =43200= (30 days).

#+vindex: tlon-local-logs-limit
The user option ~tlon-local-logs-limit~ sets the maximum number of log entries retrieved per query. If you see truncated results, increase this value. The default is =1000=.

** Log display
:PROPERTIES:
:CUSTOM_ID: h:log-display
:END:

#+vindex: tlon-local-logs-show-filename
The user option ~tlon-local-logs-show-filename~ controls whether the source filename is appended to each log line in the logs buffer. When non-nil (the default), each line includes a clickable path suffix of the form =PATH: position 1= that works with ~tlon-visit-file-at-point~.

#+vindex: tlon-local-logs-follow-mode
The user option ~tlon-local-logs-follow-mode~ controls whether navigating lines in a logs buffer automatically visits the file at point in the other window. When non-nil (the default), moving up or down in the log keeps the corresponding source file visible and scrolled in sync. You can toggle this interactively with ~tlon-local-logs-toggle-follow-mode~ ([[#h:logs-toggle-follow][toggling follow mode]]).

** Log filters
:PROPERTIES:
:CUSTOM_ID: h:log-filters
:END:

The following options define the LogQL pipeline fragments appended to the Loki selector when querying different log categories. You can customize these to match your Loki label and field conventions.

#+vindex: tlon-local-logs-errors-filter
The user option ~tlon-local-logs-errors-filter~ selects error and critical log entries. The default is =| json | level =~ "(ERROR|CRITICAL)"=.

#+vindex: tlon-local-logs-warnings-filter
The user option ~tlon-local-logs-warnings-filter~ selects warning log entries. The default is =| json | level = "WARNING"=.

#+vindex: tlon-local-logs-informational-filter
The user option ~tlon-local-logs-informational-filter~ selects informational log entries. The default is =| json | level = "INFO"=.

#+vindex: tlon-local-logs-frontend-errors-filter
The user option ~tlon-local-logs-frontend-errors-filter~ selects frontend (NextJS) error entries. Some frontend containers use lowercase level values, so the default is =| json | level = "error"=.

** Environment management
:PROPERTIES:
:CUSTOM_ID: h:environment-management-options
:END:

#+vindex: tlon-local-enforce-single-env
The user option ~tlon-local-enforce-single-env~ controls whether starting a local environment automatically stops all other running language environments first. When non-nil (the default), only one Uqbar environment runs at a time, which conserves resources. Set to =nil= if you need multiple environments running simultaneously.

* Commands
:PROPERTIES:
:CUSTOM_ID: h:commands
:END:

** Running environments
:PROPERTIES:
:CUSTOM_ID: h:running-environments
:END:

*** Starting the Uqbar environment
:PROPERTIES:
:CUSTOM_ID: h:run-uqbar
:END:

#+findex: tlon-local-run-uqbar
The command ~tlon-local-run-uqbar~ is the primary entry point for launching a local Uqbar environment. When invoked interactively, it prompts you to select a language (ISO 639-1 code). It then performs the following steps:

1. Ensures Docker Desktop is running. On macOS, it starts Docker Desktop automatically and waits up to 90 seconds for the daemon to respond.
2. Starts the web server (=./up-dev.sh= in the =web-server= repository) if it is not already running.
3. If ~tlon-local-enforce-single-env~ is non-nil, stops any other running Uqbar environments.
4. Launches the Uqbar application for the chosen language (=./launch.py start LANG= in the =uqbar= repository). If the environment is already running, you are asked whether to relaunch it.
5. After the process finishes, opens the local site in your browser (with a one-minute delay to allow the stack to initialize).

The launch command respects three toggle constants that append flags to =launch.py start=: ~tlon-local-rebuild-content-database~ (=--rebuild-content-database=), ~tlon-local-content-branch-production~ (=--content-branch=production=), and ~tlon-local-no-include-testing~ (=--no-include-testing=). These can be toggled from the transient menu ([[#h:transient-menu][transient menu]]).

** Viewing logs
:PROPERTIES:
:CUSTOM_ID: h:viewing-logs
:END:

*** Showing all logs
:PROPERTIES:
:CUSTOM_ID: h:logs-all
:END:

#+findex: tlon-local-logs
The command ~tlon-local-logs~ is the primary entry point for inspecting logs. It prompts for a language, then queries Loki for the latest content build label and retrieves four categories of log entries (errors, warnings, informational, and frontend errors) from that build. The results are displayed in a =*tlon: logs LANG*= buffer using ~tlon-local-logs-mode~, organized under clearly labeled headings.

If the =check-pages= container is still running for the selected language, the command waits (polling every 2 seconds, up to a 10-minute timeout) until it finishes before querying logs.

*** Refreshing logs
:PROPERTIES:
:CUSTOM_ID: h:logs-refresh
:END:

#+findex: tlon-local-logs-refresh
The command ~tlon-local-logs-refresh~ re-fetches logs for the language and kind shown in the current logs buffer. Press =g= in the logs buffer to invoke it.

*** Opening logs in Grafana
:PROPERTIES:
:CUSTOM_ID: h:logs-open-grafana
:END:

#+findex: tlon-local-logs-open-in-grafana
The command ~tlon-local-logs-open-in-grafana~ opens the Grafana dashboard for the content build label shown in the current logs buffer. Press =o= in the logs buffer to invoke it.

*** Toggling follow mode
:PROPERTIES:
:CUSTOM_ID: h:logs-toggle-follow
:END:

#+findex: tlon-local-logs-toggle-follow-mode
The command ~tlon-local-logs-toggle-follow-mode~ toggles whether navigating lines in a logs buffer automatically visits the file at point in the other window. Press =f= in the logs buffer to toggle it.

*** Navigating log lines
:PROPERTIES:
:CUSTOM_ID: h:logs-navigation
:END:

#+findex: tlon-local-next-line
#+findex: tlon-local-previous-line
The commands ~tlon-local-next-line~ and ~tlon-local-previous-line~ move point forward or backward by one line in a logs buffer. When follow mode is enabled ([[#h:logs-toggle-follow][toggling follow mode]]), they also visit the file referenced on the current line in the other window. These commands are bound to the standard up/down navigation keys (as well as =k= and =l=) in ~tlon-local-logs-mode~.

** Logs major mode
:PROPERTIES:
:CUSTOM_ID: h:logs-mode
:END:

#+findex: tlon-local-logs-mode
The major mode ~tlon-local-logs-mode~ (derived from ~special-mode~) provides a read-only, line-oriented interface for viewing Uqbar logs. It highlights the current line with ~hl-line-mode~ and defines the following key bindings:

| Key     | Command                                  |
|---------+------------------------------------------|
| =g=     | ~tlon-local-logs-refresh~                |
| =o=     | ~tlon-local-logs-open-in-grafana~        |
| =f=     | ~tlon-local-logs-toggle-follow-mode~     |
| =i=     | ~simple-extras-visual-line-mode-enhanced~ |
| =k=     | ~tlon-local-previous-line~               |
| =l=     | ~tlon-local-next-line~                   |
| =RET=   | ~tlon-visit-file-at-point~               |
| up/down | ~tlon-local-previous-line~ / ~tlon-local-next-line~ |

** Transient menu
:PROPERTIES:
:CUSTOM_ID: h:transient-menu
:END:

#+findex: tlon-local-menu
The command ~tlon-local-menu~ opens a transient menu organized into two columns:

- *Run environment*: language-specific launchers (=q a= through =q u=) and toggle options for =--rebuild-content-database=, =--content-branch=production=, =--no-include-testing=, and single-environment enforcement.
- *Show logs*: the ~tlon-local-logs~ command, plus toggles for follow mode and the "open in other window" option used by ~tlon-visit-file-at-point~.

The menu also defines transient infix toggles for the following constants and variables:

#+findex: tlon-local-infix-rebuild-content-database
- ~tlon-local-infix-rebuild-content-database~ toggles ~tlon-local-rebuild-content-database~.

#+findex: tlon-local-infix-content-branch-production
- ~tlon-local-infix-content-branch-production~ toggles ~tlon-local-content-branch-production~.

#+findex: tlon-local-infix-no-include-testing
- ~tlon-local-infix-no-include-testing~ toggles ~tlon-local-no-include-testing~.

#+findex: tlon-local-infix-enforce-single-env
- ~tlon-local-infix-enforce-single-env~ toggles ~tlon-local-enforce-single-env~.

#+findex: tlon-local-infix-logs-follow-mode
- ~tlon-local-infix-logs-follow-mode~ toggles ~tlon-local-logs-follow-mode~.
