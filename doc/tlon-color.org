This manual describes the features and customization options for the Emacs Lisp file =tlon-color.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

The =tlon-color= module manages color palettes for the Tlon frontend projects (specifically =uqbar-front=).  It lets you load predefined palettes into CSS (=globals.css=) and JavaScript (=theme-colors.js=) files, store palettes parsed from the current CSS state, delete stored palettes, and interactively adjust individual color values -- hue, saturation, lightness -- directly within the CSS file.  Changes are automatically propagated to the JavaScript file.

The module depends on =tlon-core=, =transient=, and the external =color-extras= package for color conversion and manipulation.

* User options
:PROPERTIES:
:CUSTOM_ID: h:user-options
:END:

** ~tlon-color-format-in-frontend-files~
:PROPERTIES:
:CUSTOM_ID: h:tlon-color-format-in-frontend-files
:END:
#+vindex: tlon-color-format-in-frontend-files

Color format to use when writing color values to the frontend CSS file.  Allowed values are ="hex"= and ="hsl"=.  The default is ="hex"=.  Note: the JavaScript file (=theme-colors.js=) currently always uses ="hsl"= regardless of this setting.

** ~tlon-colors-change-step~
:PROPERTIES:
:CUSTOM_ID: h:tlon-colors-change-step
:END:
#+vindex: tlon-colors-change-step

The step value (integer) used when increasing or decreasing hue, saturation, or lightness via the interactive manipulation commands ([[#h:color-manipulation-commands][color manipulation commands]]).  The default is =1=.

*** ~tlon-color-open-globals-file~

:CUSTOM_ID: h:tlon-color-open-globals-file

Open the =globals.css= file where frontend color palettes are defined.

*** ~tlon-color-load-palette~
:PROPERTIES:
:CUSTOM_ID: h:tlon-color-load-palette
:END:
#+findex: tlon-color-load-palette

Load a previously stored color PALETTE into the frontend files.  Prompts you to select a palette name from ~tlon-color-palettes~.  An optional LANGUAGE argument (=css= or =js=) restricts the update to one file; when =nil= (the default), both files are updated.  For each target file the command first removes the existing color sections and then inserts the selected palette.

*** ~tlon-color-save-frontend-files~
:PROPERTIES:
:CUSTOM_ID: h:tlon-color-save-frontend-files
:END:
#+findex: tlon-color-save-frontend-files

Save the current =globals.css= buffer and propagate its color state to =theme-colors.js=.  Internally the command stores the parsed CSS colors as a temporary palette (="temp-palette"=) and then loads that palette into the JavaScript file.  Use this after making manual edits to the CSS file.

*** ~tlon-color-store-palette~
:PROPERTIES:
:CUSTOM_ID: h:tlon-color-store-palette
:END:
#+findex: tlon-color-store-palette

Parse the current color definitions from =globals.css= and store them as a named palette.  Prompts you for PALETTE-NAME.  If a palette with that name already exists, asks for confirmation before overwriting (unless OVERWRITE is non-nil).  The updated palette list is persisted to ~tlon-color-palette-file~.

*** ~tlon-color-delete-palette~
:PROPERTIES:
:CUSTOM_ID: h:tlon-color-delete-palette
:END:
#+findex: tlon-color-delete-palette

Delete a stored color PALETTE.  Prompts you to select a palette name from the existing palettes in ~tlon-color-palettes~, removes it, and persists the change.

** Color manipulation commands
:PROPERTIES:
:CUSTOM_ID: h:color-manipulation-commands
:END:

These commands modify color values (HSLuv) at point in the =globals.css= file.  They replace the color string at point with the adjusted value and automatically call ~tlon-color-save-frontend-files~ to synchronize the JavaScript file.  They rely on =ct-edit-hsluv-*= functions from the =color-extras= package.

** Transient menu
:PROPERTIES:
:CUSTOM_ID: h:tlon-color-menu
:END:
#+findex: tlon-color-menu

The command ~tlon-color-menu~ activates a =transient= menu interface for color management.

The menu includes the following groups:

- *Change colors*: access to hue, saturation, and lightness increase/decrease/change commands, plus an option to set ~tlon-colors-change-step~.
- *Palette*: load, store, and delete palettes; open =globals.css=; save frontend files; convert colors and calculate contrast (via =color-extras=); set ~tlon-color-format-in-frontend-files~.

** Encoding and parsing
:PROPERTIES:
:CUSTOM_ID: h:encoding-and-parsing
:END:

#+findex: tlon-color-insert-colors-section
- ~tlon-color-insert-colors-section~ :: Insert encoded PALETTE into the file for LANGUAGE (=css= or =js=).  After insertion, converts colors to the appropriate format (respecting ~tlon-color-format-in-frontend-files~ for CSS, forcing HSL for JS) and saves the buffer.

#+findex: tlon-color-encode
- ~tlon-color-encode~ :: Encode PALETTE for LANGUAGE by calling the appropriate CSS or JS encoding function for both light and dark themes and concatenating the results.

#+findex: tlon-color-encode-globals-css
- ~tlon-color-encode-globals-css~ :: Generate the =globals.css= section string for a given THEME of PALETTE.

#+findex: tlon-color-encode-theme-colors-js
- ~tlon-color-encode-theme-colors-js~ :: Generate the =theme-colors.js= section string for a given THEME of PALETTE.

#+findex: tlon-color-encode-frontend-variables
- ~tlon-color-encode-frontend-variables~ :: Encode the frontend variables for PALETTE of THEME in the specified LANGUAGE (=css= or =js=).

#+findex: tlon-color-remove-colors-section
- ~tlon-color-remove-colors-section~ :: Remove the color definition sections from the LANGUAGE file.  For CSS, uses regex replacement; for JS, erases the entire buffer.

#+findex: tlon-color-parse-css
- ~tlon-color-parse-css~ :: Parse =globals.css= to extract color variable definitions for both light and dark themes.  Returns the result in the standard palette alist format.

#+findex: tlon-color-parse-css-theme
- ~tlon-color-parse-css-theme~ :: Parse a specific THEME section from CSS-BUFFER, converting all colors to hex for consistent storage.  Returns an alist of =(CSS-VARIABLE . HEX-VALUE)= pairs.

** Color manipulation
:PROPERTIES:
:CUSTOM_ID: h:color-manipulation-functions
:END:

#+findex: tlon-color-change-value-at-point
- ~tlon-color-change-value-at-point~ :: Core function for interactive color manipulation.  Takes COMPONENT (="h"=, ="s"=, or ="l"=), DIRECTION (="+"= or ="-"=), and optional STEP.  Finds the color at point, applies the appropriate =ct-edit-hsluv-*= function, replaces the match, and calls ~tlon-color-save-frontend-files~.
