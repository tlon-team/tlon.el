This manual describes the features and customization options for the Emacs Lisp file =tlon-import.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

The =tlon-import= module provides functions for importing documents from
various sources -- web URLs, local PDFs, and the Effective Altruism Forum
(EAF) API -- into the Tlon content structure.  It handles fetching or
reading source content, converting it to Markdown via Pandoc or
=pdftotext=, running cleanup and autofix passes, setting up YAML
metadata, and downloading referenced images.  The primary entry point is
~tlon-import-document~, which dispatches to the appropriate import
pipeline based on whether the source is a URL or a PDF file path.

* User options
:PROPERTIES:
:CUSTOM_ID: h:user-options
:END:

This module does not define =defcustom= user options.  The following
variables can be customized by setting them before loading or via
=setq=:

* Commands
:PROPERTIES:
:CUSTOM_ID: h:commands
:END:

** ~tlon-import-document~
:PROPERTIES:
:CUSTOM_ID: h:tlon-import-document
:END:
#+findex: tlon-import-document

Import a new document from a URL or a PDF file.  This is the primary
interactive entry point for the import workflow.

When called from an =ebib= entry (either =ebib-entry-mode= or
=ebib-index-mode=), the command reads the identifier (URL or file path),
title, and BibTeX key from the current entry.  If the entry is a
translation entry, the command navigates to the original entry and asks
for confirmation before proceeding.

When called outside of =ebib=, you are prompted for each detail manually.

Depending on whether the identifier is a URL or a file path:

- URLs are dispatched to ~tlon-import-html~, which checks for EAF
  content first and falls back to generic HTML-to-Markdown conversion.
- File paths are dispatched to ~tlon-import-pdf~.

After import, the command inserts the BibTeX =key= into the YAML metadata,
suggests tags via ~tlon-yaml-suggest-tags~, and downloads images from the
Markdown via ~tlon-images-download-from-markdown~.

** HTML import

:CUSTOM_ID: h:html-import

*** ~tlon-import-html~
:PROPERTIES:
:CUSTOM_ID: h:tlon-import-html
:END:
#+findex: tlon-import-html

Import the HTML in URL and convert it to Markdown.  TITLE optionally
specifies the title of the file to be imported.  If the URL corresponds
to an EAF entity (article or tag), the EAF API pipeline is used instead
of generic conversion.

*** ~tlon-import-convert-html-to-markdown~
:PROPERTIES:
:CUSTOM_ID: h:tlon-import-convert-html-to-markdown
:END:
#+findex: tlon-import-convert-html-to-markdown

Convert HTML text in SOURCE to Markdown.  SOURCE can be a URL or a file
path.  If TITLE is not provided, prompt for one.  The function determines
the bare directory (e.g., "articles") for the English originals repo, sets
the target path, and runs the appropriate Pandoc command.  After
conversion, it applies common cleanup (~tlon-cleanup-common~), autofixes
(~tlon-autofix-all~), and when the source is a URL under
=https://80000hours.org/=, it also applies ~tlon-cleanup-80k~ for
site-specific footnote formatting.

*** ~tlon-import-save-html-to-file~
:PROPERTIES:
:CUSTOM_ID: h:tlon-import-save-html-to-file
:END:
#+findex: tlon-import-save-html-to-file

Save the HTML string HTML to a temporary file and return the file path.

** PDF import
:PROPERTIES:
:CUSTOM_ID: h:pdf-import
:END:

*** ~tlon-import-pdf~
:PROPERTIES:
:CUSTOM_ID: h:tlon-import-pdf
:END:
#+findex: tlon-import-pdf

Import the PDF in PATH and convert it to Markdown.  This command requires
you to supply values for the header and footer elements to be excluded
from the conversion, which are different for each PDF.  To determine
these values, measure the distance between the top/bottom of the PDF
(which will open in the other window) and note the number of pixels until
the end of the header/footer.  Then enter these values when prompted.

If TITLE is nil, prompt for one.

Requires the =pdftotext= executable (install with =brew install poppler=).

*** ~tlon-convert-pdf~
:PROPERTIES:
:CUSTOM_ID: h:tlon-convert-pdf
:END:
#+findex: tlon-convert-pdf

Convert PDF in SOURCE to Markdown in DESTINATION.  If DESTINATION is nil,
return the Markdown string.

** EAF API integration
:PROPERTIES:
:CUSTOM_ID: h:eaf-api-integration
:END:

*** ~tlon-import-eaf-html~
:PROPERTIES:
:CUSTOM_ID: h:tlon-import-eaf-html
:END:
#+findex: tlon-import-eaf-html

Import the HTML of an EAF entity with ID-OR-SLUG and convert it to
Markdown.  TITLE optionally specifies the title.  Delegates to specific
handlers for articles and tags.

*** ~tlon-import-eaf-request~
:PROPERTIES:
:CUSTOM_ID: h:tlon-import-eaf-request
:END:
#+findex: tlon-import-eaf-request

Run an EAF request for ID-OR-SLUG.  If ASYNC is t, run the request
asynchronously.  Sends a GraphQL query to the EAF API endpoint and
returns the parsed response data.

*** ~tlon-import-eaf-article-query~
:PROPERTIES:
:CUSTOM_ID: h:tlon-import-eaf-article-query
:END:
#+findex: tlon-import-eaf-article-query

Return an EA Forum GraphQL query for a post whose ID is ID.  The query
fetches the post's title, HTML body, URL, author information, coauthors,
slug, and other metadata.

*** ~tlon-import-eaf-tag-query~
:PROPERTIES:
:CUSTOM_ID: h:tlon-import-eaf-tag-query
:END:
#+findex: tlon-import-eaf-tag-query

Return an EA Forum GraphQL query for a tag whose slug is SLUG.  The query
fetches the tag's name, slug, HTML description, and parent tag.

*** ~tlon-get-import-details-from-ebib~
:PROPERTIES:
:CUSTOM_ID: h:tlon-get-import-details-from-ebib
:END:
#+findex: tlon-get-import-details-from-ebib

Get the relevant details for importing a document from the current BibTeX
entry.  Returns a list of (identifier title key) when called from
=ebib-entry-mode= or =ebib-index-mode=.  Signal an error if the entry is
missing one of =url= or =file=, =title=, or =key=.

*** ~tlon-import-set-target~
:PROPERTIES:
:CUSTOM_ID: h:tlon-import-set-target
:END:
#+findex: tlon-import-set-target

Set the target file path for the imported document.  If TITLE is nil,
prompt for one.  BARE-DIR specifies the bare directory of the entity
being imported (e.g., "articles" or "tags").  You are prompted to
confirm or edit the proposed save path.
