This manual describes the features and customization options for the Emacs Lisp file =tlon-ai.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

The =tlon-ai= module integrates AI capabilities into the Tlon editorial workflow. It leverages the =gptel= package to communicate with various large language model backends (OpenAI, Gemini, and others) for tasks that span the full content lifecycle: summarization and abstract generation, translation, reference article creation and proofreading, Markdown formatting restoration, phonetic transcription, mathematical expression handling, meta description and title generation, language detection, commit diff summarization, and cross-repository change propagation.

A key design principle is that each task can use a dedicated AI model, letting you pair computationally demanding tasks (such as book-length summarization) with large-context models and more routine tasks with faster, cheaper alternatives. The module also supports batch processing of BibTeX entries and offers a comprehensive transient menu ([[#h:transient-menu][transient menu]]) for quick access to all operations.

*** Batch processing function
:PROPERTIES:
:CUSTOM_ID: h:batch-fun
:END:

#+vindex: tlon-ai-batch-fun
The user option ~tlon-ai-batch-fun~ specifies which function to call when processing BibTeX entries in batch mode. When non-nil, after each AI request completes the module automatically advances to the next entry (in =bibtex-mode= or =ebib-entry-mode=) and invokes this function again. Set it to =nil= to disable batch processing. You can also set it interactively from the transient menu ([[#h:transient-menu][transient menu]]).

Typical values include ~tlon-get-abstract-with-or-without-ai~, ~tlon-get-abstract-with-ai~, ~tlon-fetch-and-set-abstract~, and ~tlon-ai-set-language-bibtex~. The default value is =nil=.

*** Editing prompts before sending
:PROPERTIES:
:CUSTOM_ID: h:edit-prompt
:END:

#+vindex: tlon-ai-edit-prompt
When the user option ~tlon-ai-edit-prompt~ is non-nil, every AI prompt is presented in the minibuffer for editing before it is sent to the model. This is useful for one-off adjustments---for example, adding clarifying context or constraining the output format---without modifying the underlying prompt constants. The default value is =nil=.

*** Automatic proofreading of reference articles
:PROPERTIES:
:CUSTOM_ID: h:auto-proofread
:END:

#+vindex: tlon-ai-auto-proofread
When the user option ~tlon-ai-auto-proofread~ is non-nil, ~tlon-ai-create-reference-article~ automatically triggers ~tlon-ai-proofread-reference-article~ after the article has been generated. If =nil= (the default), you are prompted interactively whether to proofread.

** Task-specific models
:PROPERTIES:
:CUSTOM_ID: h:task-specific-models
:END:

Each of the following user options overrides the default =gptel= model for a specific task. Every option takes a cons cell of the form =(BACKEND . MODEL)=, where =BACKEND= is a string naming a registered =gptel= backend (e.g. ="Gemini"=, ="ChatGPT"=) and =MODEL= is a symbol identifying the model (e.g. =gemini-flash-latest=, =gpt-4.5-preview=). Consult ~gptel-extras-ai-models~ for the full list of available backends and models. When an option is =nil=, the buffer-local or global ~gptel-model~ is used instead.

#+vindex: tlon-ai-summarization-model
The user option ~tlon-ai-summarization-model~ controls which model generates abstracts, synopses, and meta descriptions. Because some works exceed 100,000 tokens, you should choose a model with a context window of at least one million tokens. The default is =("Gemini" . gemini-flash-latest)=.

#+vindex: tlon-ai-markdown-fix-model
The user option ~tlon-ai-markdown-fix-model~ controls which model restores Markdown formatting lost during translation ([[#h:fix-markdown-formatting][fixing Markdown formatting]]). The default is =("Gemini" . gemini-flash-latest)=.

#+vindex: tlon-ai-create-reference-article-model
The user option ~tlon-ai-create-reference-article-model~ controls which model drafts reference articles ([[#h:create-reference-article][creating reference articles]]). The default is =nil= (use the global model).

#+vindex: tlon-ai-proofread-reference-article-model
The user option ~tlon-ai-proofread-reference-article-model~ controls which model proofreads reference articles ([[#h:proofread-reference-article][proofreading reference articles]]). Proofreading benefits from a model with strong factual reasoning. The default is =("ChatGPT" . gpt-4.5-preview)=.

#+vindex: tlon-ai-summarize-commit-diffs-model
The user option ~tlon-ai-summarize-commit-diffs-model~ controls which model summarizes commit diffs ([[#h:summarize-commit-diffs][summarizing commit diffs]]). The default is =("Gemini" . gemini-pro-latest)=.

#+vindex: tlon-ai-help-model
The user option ~tlon-ai-help-model~ controls which model answers documentation-based help queries ([[#h:ask-for-help][asking for help]]). The default is =("Gemini" . gemini-flash-latest)=.

#+vindex: tlon-ai-translation-model
The user option ~tlon-ai-translation-model~ controls which model performs AI-powered translation when the translate engine is set to =ai=. The default is =nil= (use the global model).

*** Generating translation variants
:PROPERTIES:
:CUSTOM_ID: h:translate
:END:

#+findex: tlon-ai-translate
When you need to choose the best phrasing for a translated passage, the command ~tlon-ai-translate~ generates ten alternative Spanish translations of the text you provide. It prompts for the source text interactively, sends it to the AI along with the prompt in ~tlon-ai-translate-variants-prompt~, and presents the alternatives in the minibuffer for selection. The chosen translation is copied to the kill ring.

** Rewriting
:PROPERTIES:
:CUSTOM_ID: h:rewriting
:END:

*** Generating rewrite variants
:PROPERTIES:
:CUSTOM_ID: h:rewrite
:END:

#+findex: tlon-ai-rewrite
The command ~tlon-ai-rewrite~ works similarly to ~tlon-ai-translate~ but operates on existing Spanish text. If a region is active, the region text is used; otherwise you are prompted for input. The AI produces ten variant rewrites, and the one you select is copied to the kill ring.

** Reference articles
:PROPERTIES:
:CUSTOM_ID: h:reference-articles
:END:

*** Creating a reference article
:PROPERTIES:
:CUSTOM_ID: h:create-reference-article
:END:

#+findex: tlon-ai-create-reference-article
The command ~tlon-ai-create-reference-article~ generates an encyclopedia-style article for the Tlon effective altruism encyclopedia. It reads the =title= from the YAML front matter of the current buffer, determines the article language, and constructs a prompt from ~tlon-ai-write-reference-article-prompt~.

Before sending the request, the command collects source material. It scans the "Further reading" section for =<Cite>= tags. For each citation, if the tag contains a =locator= attribute (e.g. a chapter number), you are prompted to select the PDF that covers that specific section; otherwise, the full PDF linked in the BibTeX entry is used. All PDFs are converted to text and added to the =gptel= context. A language-specific glossary is also attached when translating from English.

The generated article is placed in a new Markdown buffer. You are then offered the option to proofread it immediately ([[#h:proofread-reference-article][proofreading]]). The model can be customized via ~tlon-ai-create-reference-article-model~ ([[#h:task-specific-models][task-specific models]]).

*** Proofreading a reference article
:PROPERTIES:
:CUSTOM_ID: h:proofread-reference-article
:END:

#+findex: tlon-ai-proofread-reference-article
The command ~tlon-ai-proofread-reference-article~ sends the current buffer's content to the AI for expert proofreading, focusing on factual errors, calculation mistakes, and other substantive issues. The language-appropriate prompt is drawn from ~tlon-ai-proofread-reference-article-prompt~. The proofread version or commentary appears in a new buffer. The model can be customized via ~tlon-ai-proofread-reference-article-model~ ([[#h:task-specific-models][task-specific models]]).

** Summarization
:PROPERTIES:
:CUSTOM_ID: h:summarization
:END:

*** Generating abstracts
:PROPERTIES:
:CUSTOM_ID: h:get-abstract
:END:

#+findex: tlon-get-abstract-with-or-without-ai
The command ~tlon-get-abstract-with-or-without-ai~ is the primary entry point for obtaining an abstract. It first attempts non-AI extraction via ~tlon-fetch-and-set-abstract~ (e.g. scraping publisher metadata). If that fails, it falls back to AI generation via ~tlon-get-abstract-with-ai~.

#+findex: tlon-get-abstract-with-ai
The command ~tlon-get-abstract-with-ai~ generates an abstract (100--250 words) using the AI. It determines the source text from context: the region, the current buffer, or the HTML/PDF file linked in the BibTeX entry at point. The AI first checks whether the text already contains an abstract; if so, it extracts and returns it. Otherwise it creates one from scratch, ending with the phrase "AI-generated abstract." The result is set as the =abstract= field of the BibTeX entry, or copied to the kill ring if no BibTeX context is available. The model can be customized via ~tlon-ai-summarization-model~ ([[#h:task-specific-models][task-specific models]]).

#+findex: tlon-shorten-abstract-with-ai
The command ~tlon-shorten-abstract-with-ai~ takes the existing =abstract= field of the BibTeX entry at point and asks the AI to shorten it to the maximum length specified in ~tlon-max-abstract-length~. The shortened abstract replaces the existing one.

#+findex: tlon-get-synopsis-with-ai
The command ~tlon-get-synopsis-with-ai~ generates a detailed synopsis (1000--2000 words) of the same source material that ~tlon-get-abstract-with-ai~ would use. The result is copied to the kill ring rather than inserted into a BibTeX field.

#+findex: tlon-get-abstract-with-ai-from-pdf
#+findex: tlon-get-abstract-with-ai-from-html
The commands ~tlon-get-abstract-with-ai-from-pdf~ and ~tlon-get-abstract-with-ai-from-html~ are convenience wrappers that call ~tlon-get-abstract-with-ai~ on the PDF or HTML file, respectively, associated with the BibTeX entry at point.

*** Generating meta descriptions
:PROPERTIES:
:CUSTOM_ID: h:create-meta-description
:END:

#+findex: tlon-ai-create-meta-description
The command ~tlon-ai-create-meta-description~ generates an SEO-optimized meta description (up to 160 characters) for the Markdown article in the current buffer and inserts it as the =meta= field in the YAML front matter. The language is determined from the file or by prompting you. The model can be customized via ~tlon-ai-summarization-model~.

#+findex: tlon-ai-create-meta-descriptions-in-directory
The command ~tlon-ai-create-meta-descriptions-in-directory~ iterates over all Markdown files in a directory, generating meta descriptions for those that do not already have a =meta= field. Files whose language cannot be determined are skipped.

*** Generating titles
:PROPERTIES:
:CUSTOM_ID: h:create-title
:END:

#+findex: tlon-ai-create-title
The command ~tlon-ai-create-title~ generates an SEO-optimized title (50--60 characters) for the article in the current buffer and inserts it as the =title= field in the YAML front matter.

** Formatting and repair
:PROPERTIES:
:CUSTOM_ID: h:formatting-repair
:END:

*** Fixing Markdown formatting
:PROPERTIES:
:CUSTOM_ID: h:fix-markdown-formatting
:END:

#+findex: tlon-ai-fix-markdown-formatting
When Markdown formatting (links, emphasis, HTML elements, SSML tags) is lost or altered during translation, the command ~tlon-ai-fix-markdown-formatting~ restores it by comparing the translated file paragraph by paragraph with its original counterpart. The counterpart is identified via ~tlon-get-counterpart~.

Paragraphs are processed concurrently (up to three at a time) with automatic retry and exponential backoff. If any paragraph permanently fails after three attempts, the entire process aborts with diagnostic information. On success, the reconstructed file is saved as =[filename]--fixed.md= and you are offered an =ediff= session to review the changes. The model can be customized via ~tlon-ai-markdown-fix-model~ ([[#h:task-specific-models][task-specific models]]).

** Language detection
:PROPERTIES:
:CUSTOM_ID: h:language-detection
:END:

*** Setting BibTeX language
:PROPERTIES:
:CUSTOM_ID: h:set-language-bibtex
:END:

#+findex: tlon-ai-set-language-bibtex
The command ~tlon-ai-set-language-bibtex~ detects the language of the BibTeX entry at point using AI and sets the =langid= field accordingly. If a =langid= field already exists and the detected language matches, no change is made. If they conflict, you are prompted to choose. If no =langid= field exists, the detected value is added directly. The detected language is validated and standardized before being written.

*** Setting file language for spell-checking
:PROPERTIES:
:CUSTOM_ID: h:set-language-in-file
:END:

#+findex: tlon-ai-set-language-in-file
The command ~tlon-ai-set-language-in-file~ detects the language of the current buffer or file using AI and configures the Jinx spell-checker to use that language.

*** Transcribing text phonetically
:PROPERTIES:
:CUSTOM_ID: h:phonetically-transcribe
:END:

#+findex: tlon-ai-phonetically-transcribe
The command ~tlon-ai-phonetically-transcribe~ generates an International Phonetic Alphabet (IPA) transcription of the text you provide. It defaults to the active region or word at point and determines the language from the file or by prompting you. The transcription is copied to the kill ring.

#+findex: tlon-phonetically-transcribe-in-buffer
The command ~tlon-phonetically-transcribe-in-buffer~ processes every line in the current buffer, appending a comma-separated IPA transcription to each line.

*** Transcribing audio files
:PROPERTIES:
:CUSTOM_ID: h:transcribe-audio
:END:

#+findex: tlon-transcribe-audio
The command ~tlon-transcribe-audio~ transcribes an audio file using OpenAI's Whisper API. ** Mathematical expressions
:PROPERTIES:
:CUSTOM_ID: h:math
:END:

*** Converting and translating math
:PROPERTIES:
:CUSTOM_ID: h:math-commands
:END:

#+findex: tlon-ai-convert-math
The command ~tlon-ai-convert-math~ converts a natural-language or plain-text mathematical expression into LaTeX. If point is on a =Math= tag, the LaTeX result is inserted as the second value of the =alt= attribute. Otherwise, the result is copied to the kill ring.

#+findex: tlon-ai-translate-math
The command ~tlon-ai-translate-math~ converts a LaTeX expression into a natural-language description suitable for alt text (e.g. "one half times two to the fifth power equals sixteen"). If point is on a =Math= tag, the natural-language result is inserted as the first value of the =alt= attribute. Otherwise, the result is copied to the kill ring.

** Commit analysis
:PROPERTIES:
:CUSTOM_ID: h:commit-analysis
:END:

*** Summarizing commit diffs
:PROPERTIES:
:CUSTOM_ID: h:summarize-commit-diffs
:END:

#+findex: tlon-ai-summarize-commit-diffs
The command ~tlon-ai-summarize-commit-diffs~ summarizes the diffs of commits selected in a =magit-log-mode= buffer. Select a region covering the commits of interest, then invoke the command. The AI produces an Org-formatted summary organized by package or feature, designed for developers who need to quickly assess breaking changes and new functionality. The summary appears in a =*Commit Summary*= buffer. With a prefix argument, diffstats are included in the prompt. The model can be customized via ~tlon-ai-summarize-commit-diffs-model~ ([[#h:task-specific-models][task-specific models]]).

** Cross-repository propagation
:PROPERTIES:
:CUSTOM_ID: h:propagation
:END:

*** Propagating changes across repositories
:PROPERTIES:
:CUSTOM_ID: h:propagate-changes
:END:

#+findex: tlon-ai-propagate-changes
The command ~tlon-ai-propagate-changes~ automatically applies changes from a commit in the current repository to corresponding files in all other Uqbar content repositories (both originals and translations). If invoked from =magit-status-mode= with point on a commit, that commit is used; otherwise, the latest commit is used.

For each modified file in the source commit, the command retrieves the diff and locates the corresponding file in each target repository via metadata lookups (=original_path=). It then asks the AI to apply the semantic equivalent of the changes to the target file's content. The AI response overwrites the target file and is automatically committed with a message referencing the source commit.

The command requires a clean working tree (no uncommitted changes) and relies on consistent metadata across repositories.

** Documentation-based help
:PROPERTIES:
:CUSTOM_ID: h:help
:END:

*** Asking for help
:PROPERTIES:
:CUSTOM_ID: h:ask-for-help
:END:

#+findex: tlon-ai-ask-for-help
The command ~tlon-ai-ask-for-help~ lets you ask a free-form question about the Tlon ecosystem. It collects documentation files---=.org= files from the =doc/= directories of the =tlon= and =extras= repositories, readme files from Tlon-related repos, and the user's =config.org=---and adds them to the =gptel= context. Your question, together with this context, is sent to the model specified by ~tlon-ai-help-model~. The answer appears in a new =gptel-mode= buffer. You are then prompted to clear the context; if you have follow-up questions, you can decline and clear manually later with ~gptel-context-remove-all~.

** Transient menu
:PROPERTIES:
:CUSTOM_ID: h:transient-menu
:END:

#+findex: tlon-ai-menu
The command ~tlon-ai-menu~ opens a comprehensive transient menu that exposes all AI operations and their associated options. The menu is organized into three columns:

- *Summarize*: abstract generation (with and without AI), synopsis, meta description, and batch/overwrite options.
- *Math / Reference articles / Propagation*: math conversion and translation, reference article creation and proofreading, commit diff summarization, and change propagation.
- *Misc / Options / Models*: BibTeX language detection, Markdown fix, phonetic transcription, rewriting, translation, title generation, the edit-prompt toggle, debug toggle, and per-task model selection.

The "Models" section allows you to interactively select models for each task from all registered =gptel= backends.

* Functions
:PROPERTIES:
:CUSTOM_ID: h:functions
:END:

** Sending requests to the AI
:PROPERTIES:
:CUSTOM_ID: h:make-gptel-request
:END:

#+findex: tlon-make-gptel-request
The function ~tlon-make-gptel-request~ is the central dispatch point for all AI requests in the module. It accepts a prompt string, optional printf arguments, a callback, a full model specification (as a =(BACKEND . MODEL)= cons cell), and optional parameters for tools, MCP servers, and request buffers. It pins the selected backend and model buffer-locally so that follow-up tool calls reuse the same provider. All other commands in this module delegate to this function.

** Callback utilities
:PROPERTIES:
:CUSTOM_ID: h:callbacks
:END:

#+findex: tlon-ai-callback-return
#+findex: tlon-ai-callback-copy
#+findex: tlon-ai-callback-save
#+findex: tlon-ai-callback-insert
#+findex: tlon-ai-callback-fail
The module provides a family of generic callback functions for common response-handling patterns:

- ~tlon-ai-callback-return~ returns the response string.
- ~tlon-ai-callback-copy~ copies the response to the kill ring and messages it.
- ~tlon-ai-callback-save~ saves the response to a file.
- ~tlon-ai-callback-insert~ inserts the response at the position where the request was sent.
- ~tlon-ai-callback-fail~ displays an error message extracted from the response info.

** Retrieving text content
:PROPERTIES:
:CUSTOM_ID: h:get-string-dwim
:END:

#+findex: tlon-get-string-dwim
The function ~tlon-get-string-dwim~ ("Do What I Mean") returns text content from the most appropriate source. If a region is active, it returns the region text. In =bibtex-mode= or =ebib-entry-mode=, it returns the contents of the HTML or PDF file linked in the BibTeX entry. In =pdf-view-mode= or =eww-mode=, it returns the current document's contents. In =markdown-mode=, it returns the substantive content (excluding metadata). Otherwise, it returns the full buffer contents. Many AI commands use this function to determine what to send to the model.

#+findex: tlon-get-file-as-string
The function ~tlon-get-file-as-string~ reads a file into a string, handling PDF-to-Markdown conversion and HTML-to-text rendering via =eww= transparently.

** Translation helpers
:PROPERTIES:
:CUSTOM_ID: h:translation-helpers
:END:

#+findex: tlon-ai-translate-text
The function ~tlon-ai-translate-text~ translates a text string from one language to another, attaching a glossary when translating from English. It manages its own temporary request buffer and cleans up glossary files after the callback completes.

#+findex: tlon-ai-request-wrapper
The function ~tlon-ai-request-wrapper~ provides the integration point for the =tlon-translate= machinery, dispatching AI-backed translation requests with automatic glossary handling and buffer management.
