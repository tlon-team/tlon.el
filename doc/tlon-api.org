This manual describes the features and customization options for the Emacs Lisp file =tlon-api.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

The =tlon-api= module provides an Emacs interface to the Tlon Babel APIs, specifically the Uqbar API used for managing content updates, rebuilds, and related operations. It handles authentication (token retrieval via =auth-source=), making HTTP requests to API endpoints, processing and displaying JSON responses (with clickable file paths), retrieving formatted citations, and uploading files to a remote server via =scp=.

All interactive operations are accessible through a transient menu ([[#h:transient-menu][transient menu]]).

* User options
:PROPERTIES:
:CUSTOM_ID: h:user-options
:END:

The =tlon-api= module does not define any =defcustom= user options. Its behavior is controlled by constants and a single variable described below.

** API routes
:PROPERTIES:
:CUSTOM_ID: h:api-routes
:END:

#+vindex: tlon-uqbar-api-routes
The constant ~tlon-uqbar-api-routes~ defines the available endpoints for the Uqbar API as a list of plists. Each entry specifies a =:route= (URL path, which may contain =%s= as a placeholder for the current translation language), a =:type= (HTTP method such as ="POST"= or ="GET"=), and a =:docstring= explaining the endpoint's purpose. The =%s= placeholder is replaced dynamically by ~tlon-api-get-routes~ using the value of ~tlon-translation-language~.

The defined routes cover content updates (=update/babel-refs=, =update/uqbar/%s=), frontend rebuilds (=update/rebuild-frontend=), and the corresponding log endpoints for each operation.

** Local API URL
:PROPERTIES:
:CUSTOM_ID: h:local-url
:END:

#+vindex: tlon-api-local-url
The constant ~tlon-api-local-url~ specifies the base URL for the local development instance of the Uqbar API. All API requests are directed to this URL. Its default value is =https://local-dev.altruismoeficaz.net/=.

** Most recent log buffer
:PROPERTIES:
:CUSTOM_ID: h:most-recent-log-buffer
:END:

#+vindex: tlon-api-most-recent-log-buffer
The variable ~tlon-api-most-recent-log-buffer~ stores the name of the buffer containing the most recently received API log response. This is typically set to the route string of the last request. It is used by ~tlon-api-open-most-recent-log~ ([[#h:open-most-recent-log][opening the most recent log]]) to re-display or re-fetch the log.

* Commands
:PROPERTIES:
:CUSTOM_ID: h:commands
:END:

** Making API requests
:PROPERTIES:
:CUSTOM_ID: h:api-request
:END:

#+findex: tlon-api-request
The command ~tlon-api-request~ is the primary way to interact with the Uqbar API. It prompts you to select an API route from the available endpoints (displayed with their docstrings), authenticates with the API by retrieving a bearer token, and sends the HTTP request. The JSON response is pretty-printed in a dedicated buffer, with file paths made clickable and =source_filename= values expanded to absolute paths.

With a prefix argument (~C-u~), the request includes =?force=true= in the URL, forcing the update even if no changes are detected.

#+findex: tlon-api-request-force
The command ~tlon-api-request-force~ is a convenience wrapper that calls ~tlon-api-request~ with the force flag always enabled, skipping the need for a prefix argument.

** Authentication
:PROPERTIES:
:CUSTOM_ID: h:authentication
:END:

*** Copying the access token
:PROPERTIES:
:CUSTOM_ID: h:copy-token
:END:

#+findex: tlon-api-copy-token
The command ~tlon-api-copy-token~ retrieves a fresh authentication token from the local API and copies it to the kill ring. This is useful when you need to make manual API requests (e.g. via =curl= or a browser extension) and need a valid bearer token.

** Log handling
:PROPERTIES:
:CUSTOM_ID: h:log-handling
:END:

*** Opening the most recent log
:PROPERTIES:
:CUSTOM_ID: h:open-most-recent-log
:END:

#+findex: tlon-api-open-most-recent-log
The command ~tlon-api-open-most-recent-log~ displays the buffer associated with the most recently received API log response. If the buffer has been killed since the last request, the command re-fetches the log from the API and displays it. If no request has been made yet in the current session, an error is signaled.

*** Opening the local log file
:PROPERTIES:
:CUSTOM_ID: h:open-local-log
:END:

#+findex: tlon-api-open-local-log
The command ~tlon-api-open-local-log~ opens the local log file maintained by the =uqbar-api= backend (typically =logs/uqbar-api.log= within the =uqbar-api= repository). This provides access to the server-side logs without going through the API.

** File uploading
:PROPERTIES:
:CUSTOM_ID: h:file-upload
:END:

*** Uploading files to the server
:PROPERTIES:
:CUSTOM_ID: h:upload-file
:END:

#+findex: tlon-upload-file-to-server
The command ~tlon-upload-file-to-server~ uploads a local file to a remote destination using =scp=. It prompts for the file and destination directory, then starts the transfer asynchronously. A process sentinel monitors the upload: on success, it optionally deletes the local file (when =DELETE-AFTER-UPLOAD= is non-nil) and reverts the Dired buffer if applicable; on failure, it displays the =*scp-upload*= buffer with error details.

** Transient menu
:PROPERTIES:
:CUSTOM_ID: h:transient-menu
:END:

#+findex: tlon-api-menu
The command ~tlon-api-menu~ opens a transient menu that provides quick access to all API operations:

- *Requests*: =q= invokes ~tlon-api-request~; =Q= invokes ~tlon-api-request-force~.
- *Logs*: =l= opens the most recent log; =L= opens the local log file.
- *Misc*: =t= copies the access token to the kill ring.

* Functions
:PROPERTIES:
:CUSTOM_ID: h:functions
:END:

** Authentication and credentials
:PROPERTIES:
:CUSTOM_ID: h:auth-functions
:END:

#+findex: tlon-api-get-token
The function ~tlon-api-get-token~ retrieves an authentication token for a given site URL. It posts credentials to the =/api/auth/login= endpoint and passes the resulting =access_token= to the provided callback.

#+findex: tlon-api-get-credentials
The function ~tlon-api-get-credentials~ constructs the URL-encoded credential string for API authentication. The username is derived from ~user-full-name~ via ~tlon-user-lookup~, and the password is fetched from =auth-source-pass= using a key of the form =tlon/babel/altruismoeficaz.net/USERNAME=.

** Route handling
:PROPERTIES:
:CUSTOM_ID: h:route-functions
:END:

#+findex: tlon-api-get-routes
The function ~tlon-api-get-routes~ returns a processed copy of ~tlon-uqbar-api-routes~ with =%s= placeholders replaced by the current ~tlon-translation-language~.

#+findex: tlon-select-api-route
The function ~tlon-select-api-route~ prompts you to select an API route using ~completing-read~. Each choice displays both the route path and its docstring.

** Response processing
:PROPERTIES:
:CUSTOM_ID: h:response-functions
:END:

#+findex: tlon-api-print-response
The function ~tlon-api-print-response~ renders the parsed JSON response from an API request into a buffer. It pretty-prints the JSON, expands relative =source_filename= paths to absolute paths under ~paths-dir-tlon-repos~, makes file paths clickable, and optionally displays the buffer.

#+findex: tlon-fix-source-filename-paths
The function ~tlon-fix-source-filename-paths~ parses the JSON content of a buffer, prepends the Tlon repository base path to all =source_filename= values, and re-inserts the modified JSON.

#+findex: tlon-make-paths-clickable
The function ~tlon-make-paths-clickable~ scans a buffer for quoted strings that represent existing file paths and converts them into clickable buttons. It also enables ~goto-address-mode~ and binds =RET= to ~ffap~.

** Citation retrieval
:PROPERTIES:
:CUSTOM_ID: h:citation-functions
:END:

#+findex: tlon-api-get-citation
The function ~tlon-api-get-citation~ retrieves a formatted citation for a BibTeX key from the Babel API. The optional =CSL= argument selects the citation style: =long= (default), =short=, =long-audio=, or =short-audio=. Returns the citation string, or =nil= if not found.

#+findex: tlon-api-get-citation-url
The function ~tlon-api-get-citation-url~ constructs the API URL for a citation request based on the BibTeX key and the CSL style (which determines the =text= or =audio= endpoint).

#+findex: tlon-api-get-citation-json
The function ~tlon-api-get-citation-json~ fetches and parses the JSON response from a citation URL using =curl=. It includes error handling for host resolution failures and malformed responses.
