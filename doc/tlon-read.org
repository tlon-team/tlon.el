This manual describes the features and customization options for the Emacs Lisp file =tlon-read.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

The =tlon-read= module provides integration with the =read-aloud= package to control real-time text-to-speech playback, specifically targeting a designated output buffer (typically an EWW buffer displaying a Markdown preview whose name matches =**markdown-output* # eww*=).  It defines a minor mode, ~tlon-read-mode~, with keybindings for starting and stopping playback and for navigating forward or backward by "chunks" (sentence or paragraph boundaries) in the target buffer, all without leaving your source buffer.

This module is distinct from =tlon-tts.el=, which generates audio files via cloud TTS engines.  =tlon-read.el= controls live, local TTS playback through =read-aloud=.

* Commands
:PROPERTIES:
:CUSTOM_ID: h:commands
:END:

** Minor mode
:PROPERTIES:
:CUSTOM_ID: h:minor-mode
:END:
#+findex: tlon-read-mode

~tlon-read-mode~ is a buffer-local minor mode that activates key bindings for controlling TTS playback.  You enable it in any buffer from which you want to control playback in the target buffer (identified by ~tlon-get-target-buffer~).  The keymap provides the following bindings:

| Key   | Command                              | Description                       |
|-------+--------------------------------------+-----------------------------------|
| =H-;= | ~tlon-read-target-start-or-stop~    | Start or stop playback            |
| =H-,= | ~tlon-read-backward~                | Move playback position backward   |
| =H-.= | ~tlon-read-forward~                 | Move playback position forward    |
| =H-k= | =read-aloud-extras-increase-rate=   | Increase the speech rate          |
| =H-l= | =read-aloud-extras-decrease-rate=   | Decrease the speech rate          |

The last two bindings require the =read-aloud-extras= package.

** Start or stop playback
:PROPERTIES:
:CUSTOM_ID: h:start-or-stop
:END:
#+findex: tlon-read-target-start-or-stop

You call ~tlon-read-target-start-or-stop~ to toggle TTS playback in the target buffer.  The command switches focus to the target buffer, moves the cursor to the last known playback position (if any), toggles playback with =read-aloud-buf=, repositions the cursor one chunk boundary backward (so that restarting picks up from a natural point), and then switches focus back to your original buffer.  A "chunk boundary" is defined by common punctuation marks (=,=, =.=, =:=, =!=, =;=) or two or more consecutive hyphens or newlines.

** Move backward
:PROPERTIES:
:CUSTOM_ID: h:move-backward
:END:
#+findex: tlon-read-backward

You call ~tlon-read-backward~ to move the playback position backward by one chunk in the target buffer.  If playback is active, it is stopped first.  The command searches backward for the nearest chunk boundary, then returns focus to your original buffer.

** Move forward
:PROPERTIES:
:CUSTOM_ID: h:move-forward
:END:
#+findex: tlon-read-forward

You call ~tlon-read-forward~ to move the playback position forward by one chunk in the target buffer.  If playback is active, it is stopped first.  The command searches forward for the nearest chunk boundary, then returns focus to your original buffer.

* Functions
:PROPERTIES:
:CUSTOM_ID: h:functions
:END:

** Target buffer identification
:PROPERTIES:
:CUSTOM_ID: h:target-buffer
:END:
#+findex: tlon-get-target-buffer

~tlon-get-target-buffer~ identifies the buffer that =read-aloud= should read.  It searches the buffer list for buffers whose names match the pattern =**markdown-output* # eww*= (an EWW buffer rendering a Markdown preview).  It signals a user error if zero or more than one matching buffer is found, ensuring an unambiguous target.

** Directional navigation
:PROPERTIES:
:CUSTOM_ID: h:directional-navigation
:END:
#+findex: tlon-read-backward-or-forward

~tlon-read-backward-or-forward~ is the shared implementation for ~tlon-read-backward~ and ~tlon-read-forward~.  It takes a direction argument (='backward= or ='forward=), switches to the target buffer, stops playback if active, uses =re-search-backward= or =re-search-forward= to find the next chunk boundary, and returns to the original buffer.
