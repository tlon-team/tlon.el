This manual describes the features and customization options for the Emacs Lisp file =tlon-dispatch.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

The file =tlon-dispatch.el= defines the transient menus that serve as the primary user interface for the ~tlon~ package. Rather than requiring users to remember dozens of command names, this module organizes all functionality into a hierarchical menu system built on the ~transient~ package (the same framework used by Magit).

The central entry point is ~tlon-dispatch~, which presents a top-level menu with access to over 25 specialized submenus covering areas such as AI, bibliography management, translation, search, jobs, and repository browsing. In addition to the main dispatch, this module provides dedicated menus for browsing repositories in Magit or Dired, a files menu for common file operations, and dynamically generated commands for each configured repository.

This module requires =tlon.el=, =tlon-contacts.el=, and the ~transient~ package.

** Main menu
:PROPERTIES:
:CUSTOM_ID: h:main-menu

#+findex: tlon-dispatch

The third column provides *repository browsing* shortcuts and *GitHub* integration:

- =d= opens the Dired repository browser ([[#h:dired-repo-menu][Dired repo menu]]).
- =m= opens the Magit repository browser ([[#h:magit-repo-menu][Magit repo menu]]).
- =,= opens the Tlon GitHub dashboard in an external browser (~tlon-browse-dashboard~).
- =.= lists Forge notifications (~forge-list-notifications~).

The fourth column provides *meta* operations:

- =?= invokes ~tlon-ai-ask-for-help~ to ask the AI assistant for help.
- =!= invokes ~tlon-report-bug~ to file a bug report.
- =-D= toggles the ~tlon-debug~ variable via ~tlon-menu-infix-toggle-debug~.
- =H-u= updates and reloads the ~tlon~ package (~tlon-update-package-and-reload~).
- =H-o= opens a ~tlon~ package source file (~tlon-open-package-files~).
- =H-h= copies package version information (~tlon-copy-package-info~).

The bottom of the menu displays the current package version and latest commit hash as informational lines.

** Magit repo menu
:PROPERTIES:
:CUSTOM_ID: h:magit-repo-menu
:END:

#+findex: tlon-magit-repo-menu
The command ~tlon-magit-repo-menu~ displays a transient menu listing all configured Tlon repositories, grouped into categories: Languages, General, Uqbar (with per-language content repos), Projects, Meetings, Clock, Emacs, Docs, Web, and Sandbox. Selecting an entry opens the corresponding repository in a Magit status buffer.

The individual browse commands (e.g., ~tlon-magit-browse-uqbar-es~, ~tlon-magit-browse-babel-core~) are generated dynamically at load time by the ~tlon-generate-magit-browse-commands~ macro, which iterates over ~tlon-repos~ and creates one interactive command per repository that has an ~:abbrev~ and ~:dir~ property.

An additional entry, =Q= under Uqbar, invokes ~tlon-uqbar-pull-all~ to pull all Uqbar content repositories at once ([[info:tlon-core#h:uqbar-pull-all][pulling all Uqbar repos]]).

** Dired repo menu
:PROPERTIES:
:CUSTOM_ID: h:dired-repo-menu
:END:

#+findex: tlon-dired-repo-menu
The command ~tlon-dired-repo-menu~ is the Dired counterpart of ~tlon-magit-repo-menu~. It displays the same categorized list of repositories, but selecting an entry opens the repository directory in Dired instead of Magit. The individual browse commands (e.g., ~tlon-dired-browse-uqbar-es~) are generated by the ~tlon-generate-dired-browse-commands~ macro.

** Files menu
:PROPERTIES:
:CUSTOM_ID: h:files-menu
:END:

#+findex: tlon-files-menu
The command ~tlon-files-menu~ provides a transient menu for file-related operations, organized into five groups:

- *Find*: =c= opens a file in the current repository (~tlon-find-file-in-repo~); =a= opens a file across all repositories (~tlon-open-file-across-repos~).
- *Open counterpart*: =f= opens the counterpart file in the current window (~tlon-open-counterpart-dwim~); =H-f= opens it in another window (~tlon-open-counterpart-in-other-window-dwim~). The "counterpart" is typically the original or translation of the current file.
- *URL to file*: =u= opens the web URL of the current file in a browser (~tlon-browse-file~); =H-u= opens the local file corresponding to a URL (~tlon-find-file-of-url~).
- *Issue*: =i= opens the Forge issue file for the current context (~tlon-open-forge-file~); =H-i= opens the counterpart of that issue file (~tlon-open-forge-counterpart~).
- *Version control*: =l= shows the Magit log for the current buffer file (~magit-log-buffer-file~); =d= shows diffs since the last commit by the current user (~tlon-log-buffer-latest-user-commit~); =e= starts an Ediff session against the last commit by the current user (~tlon-log-buffer-latest-user-commit-ediff~).

** Debug toggle infix
:PROPERTIES:
:CUSTOM_ID: h:debug-toggle
:END:

#+findex: tlon-menu-infix-toggle-debug
The transient infix ~tlon-menu-infix-toggle-debug~ toggles the value of the ~tlon-debug~ user option ([[info:tlon-core#h:tlon-debug][tlon-debug]]). It appears in the main dispatch menu under the =-D= key. When enabled, the ~tlon~ package emits additional diagnostic messages throughout its modules.

** Command generation macros
:PROPERTIES:
:CUSTOM_ID: h:command-generation
:END:

The following macros are evaluated at load time to dynamically create interactive commands for every repository in ~tlon-repos~. You do not call these macros directly; they run automatically when =tlon-dispatch.el= is loaded.

#+findex: tlon-generate-magit-browse-commands
The macro ~tlon-generate-magit-browse-commands~ takes a repository NAME and DIR and defines an interactive command named ~tlon-magit-browse-NAME~ that calls ~magit-status~ on DIR. At load time, this macro is evaluated for every repository in ~tlon-repos~ that has both an ~:abbrev~ and a ~:dir~ property.

#+findex: tlon-generate-dired-browse-commands
The macro ~tlon-generate-dired-browse-commands~ works identically to ~tlon-generate-magit-browse-commands~ but creates commands that call ~dired~ on DIR instead of ~magit-status~.

#+findex: tlon-generate-open-file-in-repo-commands
The macro ~tlon-generate-open-file-in-repo-commands~ takes a repository path REPO and defines an interactive command named ~tlon-open-file-in-ABBREV~ (where ABBREV is the repository's ~:abbrev~ property) that calls ~tlon-find-file-in-repo~ for that repository. This allows you to jump to any file in a specific repository with a single command.
