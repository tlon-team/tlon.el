This manual describes the features and customization options for the Emacs Lisp file =tlon-url.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=tlon-url= provides tools for detecting and fixing dead URLs within project files. It integrates with the external =lychee= link checker to scan files and repositories for broken links, the Internet Archive's Wayback Machine to find archived versions of dead URLs, and a whitelist mechanism to skip known-good URLs. The module supports both Markdown and BibTeX file formats and can operate on individual files or entire repositories. It also provides commands for propagating URL changes across all Tlon content repositories.

* Commands
:PROPERTIES:
:CUSTOM_ID: h:commands
:END:

** Listing dead URLs
:PROPERTIES:
:CUSTOM_ID: h:listing-dead-urls
:END:

*** ~tlon-list-dead-urls-in-file~
:PROPERTIES:
:CUSTOM_ID: h:tlon-list-dead-urls-in-file
:END:
#+findex: tlon-list-dead-urls-in-file

Check all URLs in FILE for dead links using the external =lychee= tool.

When called interactively, you are prompted for the file (defaulting to the current buffer's file). If FILE is =nil=, it uses the current buffer's file. For =.bib= files, the command first extracts all URLs into a temporary file to work around lychee limitations with large BibTeX files, then runs lychee on that temporary file. For all other files, lychee is run directly. Results are displayed in an =eshell= buffer.

*** ~tlon-list-dead-urls-in-repo~
:PROPERTIES:
:CUSTOM_ID: h:tlon-list-dead-urls-in-repo
:END:
#+findex: tlon-list-dead-urls-in-repo

Check all URLs in the current repository for dead links. You are prompted whether to use JSON format for the output. The command determines the repository root via ~tlon-get-repo~ and runs =lychee= on the entire directory, displaying results in an =eshell= buffer.

** Fixing dead URLs
:PROPERTIES:
:CUSTOM_ID: h:fixing-dead-urls
:END:

*** ~tlon-fix-dead-urls-in-file~
:PROPERTIES:
:CUSTOM_ID: h:tlon-fix-dead-urls-in-file
:END:
#+findex: tlon-fix-dead-urls-in-file

Run =lychee= to find and interactively fix dead links in a single FILE. When called interactively, you are prompted for the file (defaulting to the current buffer's file). The command runs =lychee= in JSON mode asynchronously and, once complete, presents each dead link with the following action choices:

| Key | Action    | Description                                               |
|-----+-----------+-----------------------------------------------------------|
| =a= | Archive   | Fetch a Wayback Machine archive and replace the dead link |
| =r= | Replace   | Manually enter a replacement URL                          |
| =d= | Delete    | Remove the URL markup, keeping any link text              |
| =w= | Whitelist | Add the URL to the whitelist to skip in future runs       |
| =s= | Skip      | Skip this link and move to the next                       |
| =q= | Quit      | Stop processing remaining dead links                      |

The "Archive" option is not available for URLs that are already Wayback Machine URLs.

*** ~tlon-fix-dead-urls-in-repo~
:PROPERTIES:
:CUSTOM_ID: h:tlon-fix-dead-urls-in-repo
:END:
#+findex: tlon-fix-dead-urls-in-repo

Run =lychee= to find dead links across the entire repository and interactively fix them. This command operates on the current project's root directory (identified by ~tlon-get-repo~) and provides the same interactive fixing workflow as ~tlon-fix-dead-urls-in-file~ ([[#h:tlon-fix-dead-urls-in-file][above]]).

The process is asynchronous: =lychee= runs in the background, and when it completes, dead links are presented one at a time for your action. Each dead link is also opened in the secondary browser so you can verify its status.

** Wayback Machine archive retrieval
:PROPERTIES:
:CUSTOM_ID: h:wayback-machine
:END:

*** ~tlon-get-archived~
:PROPERTIES:
:CUSTOM_ID: h:tlon-get-archived
:END:
#+findex: tlon-get-archived

Return the latest working archived version of URL from the Wayback Machine and copy it to the kill ring. When called interactively, you are prompted for the URL. If an archived version is found, it is displayed in a message and copied to the kill ring. If no archive is found, an appropriate message is displayed.

*** ~tlon-get-earliest-archived~
:PROPERTIES:
:CUSTOM_ID: h:tlon-get-earliest-archived
:END:
#+findex: tlon-get-earliest-archived

Return the earliest working archived version of URL from the Wayback Machine and copy it to the kill ring. This is useful for determining when a page was first archived. The command uses the Wayback Machine CDX API to find the oldest snapshot with a 200 status code. When called interactively, you are prompted for the URL.

** URL propagation
:PROPERTIES:
:CUSTOM_ID: h:url-propagation
:END:

*** ~tlon-replace-url-across-projects~
:PROPERTIES:
:CUSTOM_ID: h:tlon-replace-url-across-projects
:END:
#+findex: tlon-replace-url-across-projects

Replace URL-DEAD with URL-LIVE in all files across Tlon content repositories (both originals and translations).

When called interactively, you are prompted for the dead URL (defaulting to URL at point) and the replacement URL. The command is intentionally conservative: it only replaces domain-root URLs when they appear inside Markdown link targets =(URL)= or as Markdown autolinks =<URL>=. It also matches common variations (http/https, optional ="www."=, optional trailing slash) and avoids replacing longer URLs with path slugs.

After processing, a message reports the number of replacements and affected directories.

*** ~tlon-propagate-changed-urls~
:PROPERTIES:
:CUSTOM_ID: h:tlon-propagate-changed-urls
:END:
#+findex: tlon-propagate-changed-urls

Find URL replacements in unstaged Git changes and apply them across all Tlon content repositories.

This command scans the output of =git diff -U0= in the current repository, pairs up URLs found on removed lines with URLs found on added lines within each hunk, and calls ~tlon-replace-url-across-projects~ for each unique (OLD, NEW) pair where OLD differs from NEW.

** URL extraction
:PROPERTIES:
:CUSTOM_ID: h:url-extraction
:END:

*** ~tlon-get-urls-in-file~
:PROPERTIES:
:CUSTOM_ID: h:tlon-get-urls-in-file
:END:
#+findex: tlon-get-urls-in-file

Return a list of all URL strings found in FILE. If FILE is =nil=, use the file visited by the current buffer. The extractor is chosen based on the file type:

- Files ending in =.bib= (or buffers in ~bibtex-mode~) are parsed with a BibTeX-specific extractor that reads the =url= field from each entry.
- Markdown files (=.md= or =.markdown=, or buffers in ~markdown-mode~) are parsed using ~markdown-regex-link-inline~ to extract link targets.
- Any other file type defaults to the Markdown extractor.

*** ~tlon-get-urls-in-dir~
:PROPERTIES:
:CUSTOM_ID: h:tlon-get-urls-in-dir
:END:
#+findex: tlon-get-urls-in-dir

Return a list of all URLs found in files within DIR. If DIR is =nil=, use the current repository (via ~tlon-get-repo~). If EXTENSION is =nil=, use ="md"=. Files are found recursively using ~directory-files-recursively~ and each is processed by ~tlon-get-urls-in-file~.

*** ~tlon-save-list-of-urls~
:PROPERTIES:
:CUSTOM_ID: h:tlon-save-list-of-urls
:END:
#+findex: tlon-save-list-of-urls

Save the list of URLS to a temporary file (one URL per line, with a =# LinkChecker URL list= header) and return the file path.

** URL replacement helpers
:PROPERTIES:
:CUSTOM_ID: h:url-replacement-helpers
:END:

*** ~tlon-lychee-replace-in-file~
:PROPERTIES:
:CUSTOM_ID: h:tlon-lychee-replace-in-file
:END:
#+findex: tlon-lychee-replace-in-file

Replace OLD-URL with NEW-URL in FILE-PATH. The function tries to match variations of OLD-URL (as-is, hexified, decoded) to account for different URL encoding styles. Returns =t= if a replacement was made, =nil= otherwise.

*** ~tlon-lychee-remove-url-from-file~
:PROPERTIES:
:CUSTOM_ID: h:tlon-lychee-remove-url-from-file
:END:
#+findex: tlon-lychee-remove-url-from-file

Remove URL markup from FILE-PATH, keeping the link text. For Markdown links =[text](URL)=, keeps only the text. For bare URLs, removes the URL entirely. Returns =t= if a removal was made, =nil= otherwise.

** Wayback Machine internals
:PROPERTIES:
:CUSTOM_ID: h:wayback-machine-internals
:END:

*** ~tlon--get-wayback-machine-url~
:PROPERTIES:
:CUSTOM_ID: h:tlon--get-wayback-machine-url
:END:
#+findex: tlon--get-wayback-machine-url

Fetch the latest working archived version of URL from the Wayback Machine asynchronously. Call CALLBACK with =(ARCHIVE-URL ORIGINAL-URL)=. ARCHIVE-URL is =nil= if no archive is found or an error occurs.

*** ~tlon--get-earliest-wayback-machine-url~
:PROPERTIES:
:CUSTOM_ID: h:tlon--get-earliest-wayback-machine-url
:END:
#+findex: tlon--get-earliest-wayback-machine-url

Fetch the earliest working archived version of URL from the Wayback Machine using the CDX API. Call CALLBACK with =(ARCHIVE-URL ORIGINAL-URL)=. ARCHIVE-URL is =nil= if no archive is found or an error occurs.

** Whitelist management
:PROPERTIES:
:CUSTOM_ID: h:whitelist-management
:END:

*** ~tlon-lychee-whitelist-file~
:PROPERTIES:
:CUSTOM_ID: h:tlon-lychee-whitelist-file
:END:
#+vindex: tlon-lychee-whitelist-file

File path where whitelisted URLs are stored (one per line). Whitelisted URLs are skipped during the dead link fixing workflow. The file is located at =etc/tlon-url-lychee-whitelist.txt= within the =tlon= package directory.
