This manual describes the features and customization options for the Emacs Lisp file =tlon-glossary.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

This module provides functions for managing a multilingual glossary stored in a JSON file (=glossary.json= in the =babel-core= repository). It allows you to:

- Create and edit glossary entries with translations in any supported language.
- Extract the glossary in different formats for various recipients (humans, DeepL editor, DeepL API, AI revision, and Sieve).
- Import translations in bulk from TSV files.
- Export a multilingual CSV with all existing translations.
- Use AI to generate translations for missing terms in a target language.
- Use AI to filter a glossary to only the terms relevant to a specific source document.
- Share extracted glossaries with translators via email.

* User options
:PROPERTIES:
:CUSTOM_ID: h:user-options
:END:

** ~tlon-ai-glossary-model~
:PROPERTIES:
:CUSTOM_ID: h:tlon-ai-glossary-model
:END:
#+vindex: tlon-ai-glossary-model

Model to use for AI glossary generation (~tlon-ai-create-glossary-language~). The value is a cons cell whose car is the backend and whose cdr is the model itself. See ~gptel-extras-ai-models~ for the available options. If nil, use the default ~gptel-model~. The default value is =("Gemini" . gemini-flash-latest)=.

** ~tlon-ai-glossary-verify-model~
:PROPERTIES:
:CUSTOM_ID: h:tlon-ai-glossary-verify-model
:END:
#+vindex: tlon-ai-glossary-verify-model

Model to use for verifying and cleaning AI glossary translations. This is the second-pass model used by ~tlon-ai-create-glossary-language~ to clean and validate the raw output of the generation model. See ~tlon-ai-glossary-model~ for format details. If nil, use the default ~gptel-model~. The default value is =("Gemini" . gemini-flash-latest)=.

** ~tlon-ai-glossary-max-terms~
:PROPERTIES:
:CUSTOM_ID: h:tlon-ai-glossary-max-terms
:END:
#+vindex: tlon-ai-glossary-max-terms

Maximum number of missing terms to send to the AI in one run of ~tlon-ai-create-glossary-language~. If nil, process all missing terms. When a positive integer, only the first N terms are processed. The default value is =100=.

You can also set this value from the glossary menu ([[#h:menu][transient menu]]) under "AI Actions / Options / Create language: max terms".

** Editing
:PROPERTIES:
:CUSTOM_ID: h:editing-commands
:END:

*** ~tlon-edit-glossary~
:PROPERTIES:
:CUSTOM_ID: h:tlon-edit-glossary
:END:
#+findex: tlon-edit-glossary

Create or update a glossary entry. This command provides an interactive workflow:

1. Parse the existing glossary from ~tlon-file-glossary-source~.
2. Prompt for a source language, then choose an existing term (using completion) or type a new one.
3. If the term is new, prompt for the term type (=variable= or =invariant=). For invariant terms, all target languages are automatically pre-populated with the source term.
4. If the term exists or is newly created and is not of type =invariant=, prompt for a target language and the corresponding translation.
5. Write the updated glossary back to the JSON file.

When you add or update a translation, the command also calls ~tlon-deepl-maybe-glossary-update~ to keep the DeepL glossary in sync when the language pair supports it.

** Extraction
:PROPERTIES:
:CUSTOM_ID: h:extraction-commands
:END:

*** ~tlon-extract-glossary~
:PROPERTIES:
:CUSTOM_ID: h:tlon-extract-glossary
:END:
#+findex: tlon-extract-glossary

Extract a glossary from SOURCE-LANGUAGE to TARGET-LANGUAGE for RECIPIENT. RECIPIENT can be =human=, =deepl-editor=, =deepl-api=, =ai-revision=, or =sieve=.

- =human=: includes only entries of type =variable=, saves to CSV, and can optionally be shared with translators via email.
- =deepl-editor=: includes all entries, saves to CSV with source and target language codes appended.
- =deepl-api=: includes all entries, saves to TSV.
- =ai-revision=: includes only entries of type =variable=, saves to CSV.
- =sieve=: includes entries that have a translation, saves to JSON mapping source terms to target terms. Standard apostrophes (=​'​=) in both source and target terms are replaced with typographic apostrophes (='=).

Term normalization: leading and trailing whitespace is trimmed, empty terms are skipped, and duplicate source terms are collapsed. When duplicates occur, entries of type =variable= take precedence over =invariant=.

*** ~tlon-extract-multilingual-glossary~
:PROPERTIES:
:CUSTOM_ID: h:tlon-extract-multilingual-glossary
:END:
#+findex: tlon-extract-multilingual-glossary

Extract a multilingual CSV with all existing translations for every term in the master glossary. The file detects all language keys present across entries, writes a header row with the language codes, and outputs one row per term with translations in each language (empty if missing). The file is saved to your downloads directory as =ALL.csv=.

*** ~tlon-glossary-target-path~
:PROPERTIES:
:CUSTOM_ID: h:tlon-glossary-target-path
:END:
#+findex: tlon-glossary-target-path

Return the target path for a glossary from SOURCE-LANGUAGE to TARGET-LANGUAGE. The file extension is chosen based on RECIPIENT: =.csv= for =human=, =deepl-editor=, and =ai-revision=; =.tsv= for =deepl-api=; =.json= for =sieve=.

** Importing
:PROPERTIES:
:CUSTOM_ID: h:importing-commands
:END:

*** ~tlon-glossary-import-tsv~
:PROPERTIES:
:CUSTOM_ID: h:tlon-glossary-import-tsv
:END:
#+findex: tlon-glossary-import-tsv

Import translations from a TSV FILE into the glossary. The first line must contain two tab-separated language codes (e.g., =en<TAB>ko=). Each subsequent line must contain a source term and its translation, separated by a tab.

For example:

: en	ko
: AI safety	인공지능 안전
: altruistic motivation	이타적 동기

Behavior:

1. Parse the header to determine the source and target language codes.
2. For each pair, find the corresponding existing glossary entry where the source term matches exactly.
3. Set or update the target translation in that entry.
4. Write the updated glossary back to ~tlon-file-glossary-source~.
5. Report the number of updated translations and list any source terms that were not found.

For missing source terms, the command asks whether to create a new entry. You can choose =[y]es= to create one entry (with type selection), =[n]o= to skip it, or =[a]lways-as-variable= to create all future missing terms automatically as =variable= without further confirmation.

Input is normalized for line endings and BOM. Empty or malformed lines are ignored.

This command is useful for importing entries from the multilingual glossary in Google Sheets. Select the relevant rows in the source and target languages and paste the contents into a file.

** Sharing
:PROPERTIES:
:CUSTOM_ID: h:sharing-commands
:END:

*** ~tlon-share-glossary~
:PROPERTIES:
:CUSTOM_ID: h:tlon-share-glossary
:END:
#+findex: tlon-share-glossary

Share a glossary with translators as an email attachment. Prompts for the glossary file (defaulting to the downloads directory) and the language (unless provided). The recipient email address is looked up from ~tlon-glossary-recipients~ based on the language. The email is sent using the =share-glossary.org= template via ~tlon-email-send~.

** AI glossary generation
:PROPERTIES:
:CUSTOM_ID: h:ai-generation-commands
:END:

*** ~tlon-ai-create-glossary-language~
:PROPERTIES:
:CUSTOM_ID: h:tlon-ai-create-glossary-language
:END:
#+findex: tlon-ai-create-glossary-language

Use AI to generate translations for missing terms in a target language. This command can be run iteratively to gradually complete a language's glossary.

1. Prompt for source and target language codes.
2. Parse the glossary and identify entries with a source-language term but no target-language translation.
3. If ~tlon-ai-glossary-max-terms~ ([[#h:tlon-ai-glossary-max-terms][~tlon-ai-glossary-max-terms~]]) is set, limit the batch to the first N missing terms.
4. Send the source terms to the generation AI model (~tlon-ai-glossary-model~) with a prompt requesting one translation per line.
5. Send the raw response to a verification AI model (~tlon-ai-glossary-verify-model~) to clean formatting, enforce the correct line count, and flag uncertain translations with =[TRANSLATION_UNAVAILABLE]=.
6. Merge the verified translations back into the glossary file, skipping any =[TRANSLATION_UNAVAILABLE]= placeholders.
7. On success, report the number of merged translations and delete the temporary raw response file. On failure, keep the raw response file in the downloads directory for inspection.

** AI glossary filtering
:PROPERTIES:
:CUSTOM_ID: h:ai-filtering-commands
:END:

*** ~tlon-ai-extract-relevant-glossary~
:PROPERTIES:
:CUSTOM_ID: h:tlon-ai-extract-relevant-glossary
:END:
#+findex: tlon-ai-extract-relevant-glossary

Create a filtered glossary CSV for FILE using FULL-GLOSSARY. FULL-GLOSSARY is a CSV with at least two columns: Source, Target. FILE is a source-language file (normally English). The command asks an AI model to select only the source terms relevant to FILE and writes a filtered two-column CSV (Source, Target) to the downloads directory as =SRC-TGT-FILTERED-[basename(FILE)].csv=.

When SOURCE-LANGUAGE or TARGET-LANGUAGE are nil, try to infer SOURCE-LANGUAGE from FILE and prompt for TARGET-LANGUAGE.

Workflow:

1. Parse the CSV to obtain (Source . Target) pairs.
2. Read the plain content of FILE (for Markdown files, YAML/front matter is ignored).
3. Perform a quick deterministic scan to seed likely matches (terms that appear verbatim in the document, case-insensitively).
4. Send the document text and the list of candidate source terms to an AI model, asking it to return only the relevant source terms (one per line).
5. Map the returned source terms back to their target translations and write the filtered CSV.

If the AI returns no valid terms but the deterministic scan found some, the command falls back to writing the deterministic filtered glossary. If neither AI nor the deterministic scan finds any terms, it writes an empty CSV file. Returned terms not present in the candidate list are ignored.

:PROPERTIES:
:CUSTOM_ID: h:tlon-glossary-make-file
:END:
#+findex: tlon-glossary-make-file

Make a glossary file path for SOURCE-LANGUAGE to TARGET-LANGUAGE with EXTENSION. The file is placed in ~paths-dir-downloads~ and named as =SRC-TGT.EXTENSION= (e.g., =EN-ES.csv=).

** ~tlon-insert-formatted-glossary~
:PROPERTIES:
:CUSTOM_ID: h:tlon-insert-formatted-glossary
:END:
#+findex: tlon-insert-formatted-glossary

Insert a glossary from JSON data for SOURCE-LANGUAGE to TARGET-LANGUAGE. Format depends on RECIPIENT: =human= / =ai-revision= produce two-column CSV, =deepl-editor= produces four-column CSV, =deepl-api= produces TSV, and =sieve= produces a JSON object. Terms are normalized, deduplicated, and filtered by type as appropriate for each recipient.

** ~tlon-insert-sieve-glossary~
:PROPERTIES:
:CUSTOM_ID: h:tlon-insert-sieve-glossary
:END:
#+findex: tlon-insert-sieve-glossary

Insert a sieve-formatted glossary for SOURCE-LANGUAGE to TARGET-LANGUAGE. Creates a JSON object mapping source terms to target terms. Replaces ASCII apostrophes (=​'​=) with typographic apostrophes (='=) on both sides.

* Transient menu
:PROPERTIES:
:CUSTOM_ID: h:menu
:END:
#+findex: tlon-glossary-menu

The command ~tlon-glossary-menu~ activates a =transient= menu interface for glossary operations. It provides the following groups and commands:

- *Glossary actions*:
  - =e= :: Edit entry (~tlon-edit-glossary~)
  - =x= :: Extract glossary (~tlon-extract-glossary~)
  - =i= :: Import TSV (~tlon-glossary-import-tsv~)
  - =X= :: Extract multilingual CSV (~tlon-extract-multilingual-glossary~)
  - =s= :: Share glossary (~tlon-share-glossary~)
- *AI actions*:
  - =c= :: AI create language (~tlon-ai-create-glossary-language~)
  - =f= :: AI filter glossary (~tlon-ai-extract-relevant-glossary~)
  - Options:
    - =-c g= :: Create language model (generation) (~tlon-ai-infix-select-glossary-generate-model~)
    - =-c v= :: Create language model (verification) (~tlon-ai-infix-select-glossary-verify-model~)
    - =-c n= :: Create language: max terms (~tlon-ai-infix-set-max-terms~)
