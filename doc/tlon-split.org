This manual describes the features and customization options for the Emacs Lisp file =tlon-split.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

The =tlon-split= module provides a minor mode, ~tlon-split-mode~, designed
to facilitate working with two vertically split windows, typically
containing an original document and its translation.  Its primary function
is to automatically align the content (specifically paragraphs) between
the two windows as you navigate the source window.  When the cursor moves
to a different screen line and is past a configurable threshold, the
other window is scrolled so that the corresponding paragraph is visible.

* User options
:PROPERTIES:
:CUSTOM_ID: h:user-options
:END:

** ~tlon-split-screen-line-threshold~
:PROPERTIES:
:CUSTOM_ID: h:tlon-split-screen-line-threshold
:END:
#+vindex: tlon-split-screen-line-threshold

Number of lines below which ~tlon-split-autoalign-paragraphs~ is
disabled.  This prevents alignment attempts when near the top of the
buffer where paragraph counting might be less reliable or meaningful.

- Default :: =60=
- Type :: =integer=

* Commands
:PROPERTIES:
:CUSTOM_ID: h:commands
:END:

** ~tlon-split-mode~
:PROPERTIES:
:CUSTOM_ID: h:tlon-split-mode
:END:
#+findex: tlon-split-mode

Toggle ~tlon-split-mode~ in the current buffer.  When enabled, the
function ~tlon-split-autoalign-paragraphs~ is added to
=post-command-hook= so that paragraph alignment happens automatically
after every command.  When disabled, the hook is removed.

** ~tlon-split-mode-reset~
:PROPERTIES:
:CUSTOM_ID: h:tlon-split-mode-reset
:END:
#+findex: tlon-split-mode-reset

Reset ~tlon-split-mode~.  This removes and re-adds
~tlon-split-autoalign-paragraphs~ to =post-command-hook=.  Use this when
the hook's behavior seems incorrect or needs refreshing.

** Transient menu
:PROPERTIES:
:CUSTOM_ID: h:tlon-split-menu
:END:
#+findex: tlon-split-menu

The command ~tlon-split-menu~ activates a =transient= menu interface for
split mode operations.  It provides the following options under the
"Split mode" group:

- =s= :: Toggle split mode (~tlon-split-mode~)
- =r= :: Reset split mode (~tlon-split-mode-reset~)

* Functions
:PROPERTIES:
:CUSTOM_ID: h:functions
:END:

** Alignment
:PROPERTIES:
:CUSTOM_ID: h:alignment
:END:

*** ~tlon-split-align-paragraphs~
:PROPERTIES:
:CUSTOM_ID: h:tlon-split-align-paragraphs
:END:
#+findex: tlon-split-align-paragraphs

Align the paragraphs in the current and other windows.  It calculates
the paragraph number the cursor is currently in within the source window
(using ~tlon-get-number-of-paragraphs~) and the screen line offset, then
scrolls the other window so that the corresponding paragraph is visible,
using =recenter= and =scroll-down=.

*** ~tlon-split-align-screen-lines~
:PROPERTIES:
:CUSTOM_ID: h:tlon-split-align-screen-lines
:END:
#+findex: tlon-split-align-screen-lines

Align the screen lines in the current and other windows.  The alignment
is performed by scrolling up or down the other window so that the cursor
in the other window is on the same screen line number as the cursor in
the current window.

*** ~tlon-split-autoalign-paragraphs~
:PROPERTIES:
:CUSTOM_ID: h:tlon-split-autoalign-paragraphs
:END:
#+findex: tlon-split-autoalign-paragraphs

Automatically align the paragraphs in the current and other windows.
This is the function added to =post-command-hook= when ~tlon-split-mode~
is active.  It checks three conditions before aligning: the mode is
enabled, the screen line has changed since the last check, and the cursor
is at or below ~tlon-split-screen-line-threshold~.

** State detection
:PROPERTIES:
:CUSTOM_ID: h:state-detection
:END:

*** ~tlon-split-screen-line-changed-p~
:PROPERTIES:
:CUSTOM_ID: h:tlon-split-screen-line-changed-p
:END:
#+findex: tlon-split-screen-line-changed-p

Return t iff the cursor is on a different screen line than the last
recorded position.  Updates ~tlon-split-last-screen-line-pos~ as a side
effect.

*** ~tlon-split-screen-line-offset~
:PROPERTIES:
:CUSTOM_ID: h:tlon-split-screen-line-offset
:END:
#+findex: tlon-split-screen-line-offset

Return the difference between the screen lines in the current and other
windows.  A positive value means the other window's cursor is further
down the screen.

*** ~tlon-split-top-of-buffer-visible-p~
:PROPERTIES:
:CUSTOM_ID: h:tlon-split-top-of-buffer-visible-p
:END:
#+findex: tlon-split-top-of-buffer-visible-p

Return t iff the top of the buffer is visible in the current window.  The
function considers the top of the buffer to be visible if the first screen
line of the buffer, or the first screen line after the metadata if any, is
visible.  Currently unused; ~tlon-split-screen-line-threshold~ is used as
an imperfect approximation instead.
