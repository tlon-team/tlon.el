This manual describes the features and customization options for the Emacs Lisp file =tlon-db.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

The =tlon-db= module provides integration with the EA International bibliographic database API. It handles authentication, entry retrieval and posting, entry movement between local BibTeX files and the remote database, entry deletion, author name management, publication status and citation metadata, and automatic synchronization of local changes with the remote server.

The module uses a two-file system for synchronization: =db.bib= is the main file you edit locally, and =db-upstream.bib= is a read-only mirror of the remote database's state. When automatic synchronization is enabled, changes detected in =db.bib= are diffed against =db-upstream.bib= and the resulting additions, modifications, and deletions are pushed to the API. Periodic background refreshes also pull updates from the server.

You can access all the major commands through the transient menu ~tlon-db-menu~ ([[#h:transient-menu][Transient menu]]).

* User options
:PROPERTIES:
:CUSTOM_ID: h:user-options
:END:

** Local development environment
:PROPERTIES:
:CUSTOM_ID: h:local-environment
:END:

#+vindex: tlon-db-use-local-environment
The user option ~tlon-db-use-local-environment~ controls whether API requests are sent to the local development server at =https://local-dev.ea.international= instead of the production server. When non-nil, it also adds the mkcert root certificate (=~/Library/Application Support/mkcert/RootCA.pem=) to ~gnutls-trustfiles~ so that TLS verification succeeds against the local server. The default is ~nil~.

You can toggle this interactively via the transient menu ([[#h:transient-menu][Transient menu]]) using the =-l= key.

** Automatic synchronization
:PROPERTIES:
:CUSTOM_ID: h:auto-sync-option
:END:

#+vindex: tlon-db-enable-auto-sync
The user option ~tlon-db-enable-auto-sync~ controls whether changes to =db.bib= detected by the file watch are automatically synchronized with the remote API. The file watch (installed by ~tlon-db-initialize~) is always active, but when this option is ~nil~ the callback does nothing, effectively pausing automatic synchronization. The default is ~nil~.

You can toggle this interactively via the transient menu ([[#h:transient-menu][Transient menu]]) using the =-s= key.

** Sync confirmation threshold
:PROPERTIES:
:CUSTOM_ID: h:sync-threshold
:END:

#+vindex: tlon-db-sync-confirm-threshold
The user option ~tlon-db-sync-confirm-threshold~ sets the minimum number of created or modified entries that triggers a confirmation prompt and brings the =*Db Sync Log*= buffer to the foreground after syncing. This acts as a safety net: when a large number of entries would be modified at once, you get the chance to review the changes before they are pushed. The default is 5.

** Periodic refresh interval
:PROPERTIES:
:CUSTOM_ID: h:refresh-interval
:END:

#+vindex: tlon-db-get-entries-interval
The user option ~tlon-db-get-entries-interval~ controls how many idle seconds must pass before the local BibTeX database is automatically refreshed from the remote API. The default is 10800 (three hours). Changing this value interactively (e.g., via the transient menu) restarts the underlying idle timer so the new value takes effect immediately.

* Commands
:PROPERTIES:
:CUSTOM_ID: h:commands
:END:

** Entry retrieval
:PROPERTIES:
:CUSTOM_ID: h:entry-retrieval
:END:

*** Get all entries
:PROPERTIES:
:CUSTOM_ID: h:get-entries
:END:

#+findex: tlon-db-get-entries
The command ~tlon-db-get-entries~ retrieves all BibTeX entries from the EA International API and saves them to both the local database file (=db.bib=) and the upstream tracking file (=db-upstream.bib=). Before fetching, it verifies that there are no unsaved changes in =db.bib= and that the two database files are identical (unless a prefix argument is supplied to bypass this check). If the files differ and the check is not bypassed, the command opens an Ediff session to let you resolve the discrepancies first. The files are saved with =utf-8-unix= encoding, and special characters are escaped.

#+findex: tlon-db-get-entries-no-confirm
The command ~tlon-db-get-entries-no-confirm~ is a convenience wrapper that calls ~tlon-db-get-entries~ with the confirmation bypass enabled. It is also used by the periodic refresh timer.

*** Get single entry
:PROPERTIES:
:CUSTOM_ID: h:get-entry
:END:

#+findex: tlon-db-get-entry
The command ~tlon-db-get-entry~ retrieves the details of a single entry from the API by its entry ID and displays them in the =*Db API Result*= buffer. When called interactively, it prompts for the entry ID.

** Entry posting and movement
:PROPERTIES:
:CUSTOM_ID: h:entry-posting
:END:

*** Post entry
:PROPERTIES:
:CUSTOM_ID: h:post-entry
:END:

#+findex: tlon-db-post-entry
The command ~tlon-db-post-entry~ creates or updates a BibTeX entry in the remote API. When called interactively, it posts the entry at point; from Lisp, you can pass a key. The entry text is normalized (author field whitespace is collapsed) and sent as =text/plain=.

On success (HTTP 200), the command replaces the local entry in both =db.bib= and =db-upstream.bib= with the version returned by the API (which may have reordered fields). If the API responds with a 400 error due to missing author names, the command offers to create each missing name automatically via ~tlon-db-set-name~ ([[#h:set-name][Set name]]) and retries once. During automatic synchronization, entries with missing author names are silently skipped and logged to the =*Db Sync Log*= buffer.

Returns ~t~ on success, ~nil~ otherwise.

*** Move entry to database
:PROPERTIES:
:CUSTOM_ID: h:move-entry
:END:

#+findex: tlon-db-move-entry
The command ~tlon-db-move-entry~ moves the BibTeX entry at point (or a specified key) into the local database by posting it to the API. If the entry is already in =db.bib= or =db-upstream.bib=, the command prints a message and takes no further action. After a successful post, the entry is removed from its original file and the Ebib database is reloaded.

*** Move entry pair
:PROPERTIES:
:CUSTOM_ID: h:move-entry-pair
:END:

#+findex: tlon-db-move-entry-pair
The command ~tlon-db-move-entry-pair~ moves the entry at point together with all its translations to the database. It determines the original entry from context: if point is on a translation, it finds the original via the =translation= field. Entries already in the database are skipped. This is convenient when you want to move an original work and all its translations in a single operation.

** Entry deletion
:PROPERTIES:
:CUSTOM_ID: h:entry-deletion
:END:

#+findex: tlon-db-delete-entry
The command ~tlon-db-delete-entry~ deletes a BibTeX entry from the remote API. Interactively, it uses the key at point (or prompts for one from the database entries) and asks for confirmation before proceeding. A prefix argument skips the confirmation. When the =locally= argument is non-nil (the default for interactive use), the entry is also removed from both local database files. On non-200 responses or when ~tlon-debug~ is non-nil, the server's response is displayed in the =*Db API Result*= buffer.

** Author name management
:PROPERTIES:
:CUSTOM_ID: h:name-management
:END:

*** Check name
:PROPERTIES:
:CUSTOM_ID: h:check-name
:END:

#+findex: tlon-db-check-name
The command ~tlon-db-check-name~ queries the API to check whether a given name exists in the EA International name database. It displays matching names along with their translations. When ~tlon-debug~ is nil and the request succeeds, a concise confirmation message is shown; otherwise, the full result is displayed in the =*Db API Result*= buffer.

*** Check or insert name
:PROPERTIES:
:CUSTOM_ID: h:check-or-insert-name
:END:

#+findex: tlon-db-check-or-insert-name
The command ~tlon-db-check-or-insert-name~ checks whether a name exists and, if it does not and there are no similar conflicting names, inserts it automatically. If similar names are found (HTTP 409), the conflicts are reported. This provides a convenient one-step workflow for adding new author names.

*** Set name
:PROPERTIES:
:CUSTOM_ID: h:set-name
:END:

#+findex: tlon-db-set-name
The command ~tlon-db-set-name~ inserts or updates a name in the database via the API. When a new name is provided, it renames the existing name. When the force argument is non-nil, similarity conflicts are overridden. Interactively, the command prompts for the name (in "Last, First" format), an optional new name, and whether to force insertion. Author names are normalized by replacing non-breaking spaces with regular spaces.

** Publication status and citation metadata
:PROPERTIES:
:CUSTOM_ID: h:publication-metadata
:END:

*** Set publication status
:PROPERTIES:
:CUSTOM_ID: h:set-publication-status
:END:

#+findex: tlon-db-set-publication-status
The command ~tlon-db-set-publication-status~ sets the publication status of the entry at point. It determines the entry ID from context: in BibTeX or Ebib buffers it uses the key at point, and in Markdown buffers it reads the key from the YAML front matter. It prompts for one of =production=, =testing=, or =unpublished=, defaulting to the entry's current status when available.

#+findex: tlon-db-set-publication-status-in-current-directory
The command ~tlon-db-set-publication-status-in-current-directory~ applies a chosen publication status to every Markdown file in the current directory. It reads the BibTeX key from each file's YAML front matter and calls ~tlon-db-set-publication-status~ for each. The command confirms before proceeding and reports a success/failure summary.

*** Set cite full title
:PROPERTIES:
:CUSTOM_ID: h:set-cite-full-title
:END:

#+findex: tlon-db-set-cite-full-title
The command ~tlon-db-set-cite-full-title~ sets the =cite_full_title= flag to =true= for the entry at point via the API. Like ~tlon-db-set-publication-status~, it determines the entry ID from the current buffer context. This flag controls whether the full title is used in citations generated by the Tlon platform.

** Translation queries
:PROPERTIES:
:CUSTOM_ID: h:translation-queries
:END:

*** Get translation key
:PROPERTIES:
:CUSTOM_ID: h:get-translation-key
:END:

#+findex: tlon-db-get-translation-key
The command ~tlon-db-get-translation-key~ returns the BibTeX key of the translation of an original entry in a specified language. It calls the =/api/translations/{original_id}= endpoint, parses the JSON response, and finds the matching translation by language. When called interactively, the translation key is also echoed in the minibuffer. If no target language is provided, it prompts for one.

*** Get translations
:PROPERTIES:
:CUSTOM_ID: h:get-translations
:END:

#+findex: tlon-db-get-translations
The command ~tlon-db-get-translations~ lists all translations of an original entry. It returns a list of plists, each containing =:key= and =:language=. When called interactively, the results are displayed in the =*Db API Result*= buffer.

** Synchronization
:PROPERTIES:
:CUSTOM_ID: h:synchronization
:END:

*** Manual sync
:PROPERTIES:
:CUSTOM_ID: h:sync-now
:END:

#+findex: tlon-db-sync-now
The command ~tlon-db-sync-now~ runs the same synchronization process used by automatic sync, regardless of whether ~tlon-db-enable-auto-sync~ ([[#h:auto-sync-option][Automatic synchronization]]) is enabled. It saves =db.bib= if modified, diffs it against =db-upstream.bib=, and processes the resulting additions, modifications, and deletions. A summary message is displayed, and details are logged to the =*Db Sync Log*= buffer. If there are no differences, it reports that and takes no action.

*** Automatic sync
:PROPERTIES:
:CUSTOM_ID: h:auto-sync
:END:

When ~tlon-db-enable-auto-sync~ is non-nil, the file watch installed by ~tlon-db-initialize~ detects changes to =db.bib= and triggers synchronization automatically. The sync process computes a unified diff between =db.bib= and =db-upstream.bib=, classifies entries as added, modified, or deleted, and pushes each change to the API. Additions and modifications are posted via ~tlon-db-post-entry~ ([[#h:post-entry][Post entry]]); deletions use ~tlon-db-delete-entry~ ([[#h:entry-deletion][Entry deletion]]). On success, both local files are updated to reflect the API's response.

The sync log buffer (=*Db Sync Log*=) records timestamps, created/modified/deleted keys, and colored diffs for modifications. When the number of changes meets or exceeds ~tlon-db-sync-confirm-threshold~ ([[#h:sync-threshold][Sync confirmation threshold]]), a confirmation prompt is shown and the log buffer is brought to the foreground.

** Authentication
:PROPERTIES:
:CUSTOM_ID: h:authentication
:END:

#+findex: tlon-db-authenticate
The command ~tlon-db-authenticate~ performs authentication against the EA International API using credentials fetched via =auth-source=. It stores the obtained access token in ~tlon-db-auth-token~ and sets its expiry time (30 minutes). On failure, it signals an error and displays the server response. Most commands that require authentication call ~tlon-db-ensure-auth~ internally, so you rarely need to invoke this manually.

** Transient menu
:PROPERTIES:
:CUSTOM_ID: h:transient-menu
:END:

#+findex: tlon-db-menu
The command ~tlon-db-menu~ activates a transient menu providing quick access to all commands and options in this module. It is organized into two columns:

- *Actions*: get entries (=g=, =G=), get single entry (=e=), get translation key (=k=), post entry (=p=), move entry (=m=, =M=), delete entry (=d=), name management (=c=, =i=, =n=), publication status (=s=, =S=), cite full title (=t=), sync now (=y=), and authenticate (=a=).
- *Options*: toggle local environment (=-l=), toggle auto sync (=-s=), and set refresh interval (=-i=).

* Functions
:PROPERTIES:
:CUSTOM_ID: h:functions
:END:

** Initialization
:PROPERTIES:
:CUSTOM_ID: h:initialization
:END:

#+findex: tlon-db-initialize
The function ~tlon-db-initialize~ sets up the =tlon-db= package. It installs the file watch for automatic synchronization, adds =db.bib= to the global bibliography list, and refreshes the citar bibliography cache. This function is called automatically when the module is loaded.

** Authentication helpers
:PROPERTIES:
:CUSTOM_ID: h:auth-helpers
:END:

#+findex: tlon-db-ensure-auth
The function ~tlon-db-ensure-auth~ checks whether the current authentication token is valid and not expired. If it is missing or expired, it calls ~tlon-db-authenticate~ to refresh it. Returns the valid token or signals an error. This is the function called internally by commands that require authentication.

** Database entry selection
:PROPERTIES:
:CUSTOM_ID: h:db-entry-selection
:END:

#+findex: tlon-db-get-db-entries
The function ~tlon-db-get-db-entries~ prompts you to select a work from the database using citar's completion interface, scoped to =db.bib= only. It returns the selected key.

** File watch management
:PROPERTIES:
:CUSTOM_ID: h:file-watch
:END:

#+findex: tlon-db-remove-file-notify-watch
The function ~tlon-db-remove-file-notify-watch~ removes the file notification watch on =db.bib=. It is added to ~kill-emacs-hook~ automatically during initialization to ensure clean shutdown.
