This manual describes the features and customization options for the Emacs Lisp file =tlon-tts.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

The =tlon-tts= module provides comprehensive text-to-speech (TTS) functionality, integrating with five cloud-based TTS engines: Microsoft Azure, Google Cloud, Amazon Polly, OpenAI, and ElevenLabs.  Its core workflow consists of three stages: staging content (preparing Markdown text for narration by applying SSML tags, listener cues, abbreviation expansions, and other transformations), generating audio (sending prepared text chunks to a TTS engine via its API), and finalizing the output (optionally normalizing, appending silence between paragraphs, and joining all chunks into a single audio file).

The module also ships a transient menu ([[#h:transient-menu][~tlon-tts-menu~]]) that surfaces every command and option, a quality report generator that flags potential issues before narration, and editing facilities for global and local abbreviation, replacement, and phonetic transcription dictionaries.

External tools required: =ffmpeg= (audio processing), =curl= (API requests), =aws= CLI (Amazon Polly), =gcloud= CLI (Google Cloud token), =jq= (Google Cloud response parsing), =mp3splt= (audio truncation).

* User options
:PROPERTIES:
:CUSTOM_ID: h:user-options
:END:

** General TTS settings
:PROPERTIES:
:CUSTOM_ID: h:general-tts-settings
:END:

#+vindex: tlon-tts-global-engine
~tlon-tts-global-engine~ controls which TTS engine is selected when you create a new staging buffer.  Accepted values are ="Microsoft Azure"=, ="Google Cloud"=, ="Amazon Polly"=, ="OpenAI"=, and ="ElevenLabs"= (the default).  You can override this per session from the transient menu ([[#h:transient-menu][~tlon-tts-menu~]]).

#+vindex: tlon-tts-use-alternate-voice
~tlon-tts-use-alternate-voice~ determines whether elements such as notes, asides, blockquotes, and other listener-cued sections are read with an alternate voice.  When non-nil, the alternate voice configured for the current engine is used for those sections.  When nil (the default), all text is read with the main voice.

#+vindex: tlon-tts-delete-file-chunks
~tlon-tts-delete-file-chunks~ controls whether individual audio chunk files are automatically deleted after they have been joined into the final output file during finalization.  When nil (the default), chunk files are preserved so you can re-finalize or inspect them.

#+vindex: tlon-tts-narrate-sidenotes
~tlon-tts-narrate-sidenotes~ controls whether sidenotes are read aloud.  When non-nil, the content of sidenotes is repositioned to the end of the sentence containing the reference and narrated with a listener cue.  When nil (the default), sidenotes are removed along with footnotes.

#+vindex: tlon-tts-generate-missing-chunks-only
~tlon-tts-generate-missing-chunks-only~ controls whether ~tlon-tts-narrate-staged-buffer~ ([[#h:narrate-staged-buffer][described below]]) skips chunks whose audio files already exist on disk.  When non-nil (the default), narration resumes from the first missing chunk, which is useful for continuing an interrupted run.  When nil, all chunks are regenerated.  This option does not affect ~tlon-tts-narrate-staged-chunks~, which always regenerates the selected chunks.

#+vindex: tlon-tts-normalize-audio
~tlon-tts-normalize-audio~ controls whether an =ffmpeg= loudness normalization filter is applied to each chunk during finalization.  When nil (the default), the normalization step is skipped.

#+vindex: tlon-tts-loudness-target
~tlon-tts-loudness-target~ sets the target integrated loudness in LUFS for the normalization filter.  Common values are =-16= (loud, the default), =-18= (moderate), =-20= (quiet), and =-23= (broadcast standard).  To determine a suitable value, analyze a reference clip with =ffmpeg -i reference.mp3 -af loudnorm=print_format=summary -f null -= and use the "Input Integrated" LUFS value.

#+vindex: tlon-tts-paragraph-break-duration
~tlon-tts-paragraph-break-duration~ is a string specifying the duration of the SSML =<break>= tag inserted after each paragraph (e.g., ="0.8s"=, the default).  Note that ElevenLabs replaces the narrator's natural pause with the specified duration, whereas other engines add the specified duration on top of the natural pause.

#+vindex: tlon-tts-listener-cue-break-duration
~tlon-tts-listener-cue-break-duration~ is a string specifying the duration of the pause inserted before and after listener cues (e.g., ="0.5s"=, the default).

#+vindex: tlon-tts-prompt
~tlon-tts-prompt~ is a generic prompt string to include in TTS requests.  Selection candidates for each language are listed in the constant ~tlon-tts-prompts~.  This option is currently unused.

** Engine-specific audio settings
:PROPERTIES:
:CUSTOM_ID: h:engine-audio-settings
:END:

Each engine has a =defcustom= that selects the default output audio format as a cons cell =("FORMAT_NAME" . "extension")=.

#+vindex: tlon-microsoft-azure-audio-settings
~tlon-microsoft-azure-audio-settings~ defaults to =("audio-24khz-160kbitrate-mono-mp3" . "mp3")=.  The docstring of this variable lists the main format options and links to the full Microsoft documentation.

#+vindex: tlon-google-cloud-audio-settings
~tlon-google-cloud-audio-settings~ defaults to =("MP3" . "mp3")=.  Other choices include ="FLAC"=, ="OGG_OPUS"=, ="WEBM_OPUS"=, and more.

#+vindex: tlon-amazon-polly-audio-settings
~tlon-amazon-polly-audio-settings~ defaults to =("mp3" . "mp3")=.  Admissible values are ="ogg_vorbis"=, ="pcm"=, and ="mp3"=.

#+vindex: tlon-openai-audio-settings
~tlon-openai-audio-settings~ defaults to =("wav" . "wav")=.  Other choices are ="mp3"= and ="opus"=.

#+vindex: tlon-elevenlabs-audio-settings
~tlon-elevenlabs-audio-settings~ defaults to =("mp3_44100_192" . "mp3")=.  The 192 kbps option requires at least a Creator tier subscription.  Several PCM and mu-law formats are also available.

** Engine-specific model settings
:PROPERTIES:
:CUSTOM_ID: h:engine-model-settings
:END:

#+vindex: tlon-openai-model
~tlon-openai-model~ selects the OpenAI TTS model.  Options include ="gpt-4o-mini-tts"= (the default, latest model), ="tts-1"= (lowest latency), and ="tts-1-hd"= (higher quality).

#+vindex: tlon-elevenlabs-model
~tlon-elevenlabs-model~ selects the ElevenLabs model.  The default is ="eleven_multilingual_v2"=, the most life-like and emotionally rich model.  Other options include ="eleven_v3"= (latest, 74 languages), ="eleven_flash_v2_5"= (ultra-low latency, 32 languages), and several older models.  The docstring lists all models with their characteristics.

** ElevenLabs pronunciation dictionaries
:PROPERTIES:
:CUSTOM_ID: h:elevenlabs-pronunciation-dictionaries
:END:

#+vindex: tlon-elevenlabs-attach-dictionaries
~tlon-elevenlabs-attach-dictionaries~ controls whether =pronunciation_dictionary_locators= are included in ElevenLabs requests.  When non-nil, up to three dictionary locators for the current language are read from the JSON file specified by ~tlon-file-elevenlabs-dictionary-locators~ and sent with each API request.  IPA rules require a compatible model such as ="eleven_turbo_v2"=.

** Quick TTS settings
:PROPERTIES:
:CUSTOM_ID: h:quick-tts-settings
:END:

#+vindex: tlon-tts-quick-voice
~tlon-tts-quick-voice~ specifies the voice to use with ~tlon-tts-quick~ ([[#h:quick-tts][described below]]).  It defaults to ="sage"=.  Consult ~tlon-openai-voices~ for available options.

* Commands
:PROPERTIES:
:CUSTOM_ID: h:commands
:END:

** Core workflow
:PROPERTIES:
:CUSTOM_ID: h:core-workflow
:END:

*** Stage content
:PROPERTIES:
:CUSTOM_ID: h:stage-content
:END:
#+findex: tlon-tts-stage-content

You call ~tlon-tts-stage-content~ to prepare content for narration.  It reads the active region, a specified file, or the file visited by the current buffer; creates a dedicated staging buffer (named like =*TTS: filename*=); inserts the content; sets file-local variables (source, language, engine, voice, audio format, locale); runs the full preparation pipeline (abbreviation expansion, SSML tagging, listener cue insertion, formatting removal, numeral processing, and more); calculates chunk boundaries; inserts chunk comments; and positions the cursor at the paragraph corresponding to your position in the source buffer.

*** Narrate staged buffer
:PROPERTIES:
:CUSTOM_ID: h:narrate-staged-buffer
:END:
#+findex: tlon-tts-narrate-staged-buffer

You call ~tlon-tts-narrate-staged-buffer~ from a staging buffer to narrate the entire prepared text.  It ensures chunks are calculated and comments are present, then iterates through chunks asynchronously, sending each to the selected TTS engine's API.  If ~tlon-tts-generate-missing-chunks-only~ ([[#h:general-tts-settings][described above]]) is non-nil, it skips chunks whose audio files already exist.  After all chunks finish, you must call ~tlon-tts-finalize-audio-processing~ ([[#h:finalize-audio][described below]]) to normalize (if enabled), append silence, and join the chunks into the final output file.

*** Narrate staged chunks
:PROPERTIES:
:CUSTOM_ID: h:narrate-staged-chunks
:END:
#+findex: tlon-tts-narrate-staged-chunks

You call ~tlon-tts-narrate-staged-chunks~ from a staging buffer to regenerate specific chunks.  If a region is active, it generates audio for all chunks starting within that region.  Otherwise, it generates audio for the chunk at point.  This command always overwrites existing audio files for the selected chunks, regardless of the ~tlon-tts-generate-missing-chunks-only~ setting.  It does not perform finalization; use ~tlon-tts-finalize-audio-processing~ afterwards.

*** Go to chunk
:PROPERTIES:
:CUSTOM_ID: h:goto-chunk
:END:
#+findex: tlon-tts-goto-chunk

You call ~tlon-tts-goto-chunk~ (interactively prompted for a 1-based chunk number) to jump to the beginning of that chunk in the staging buffer.

** Audio file management
:PROPERTIES:
:CUSTOM_ID: h:audio-file-management
:END:

*** Finalize audio processing
:PROPERTIES:
:CUSTOM_ID: h:finalize-audio
:END:
#+findex: tlon-tts-finalize-audio-processing

You call ~tlon-tts-finalize-audio-processing~ to trigger the final audio processing steps for a given base audio file.  It normalizes chunks asynchronously if ~tlon-tts-normalize-audio~ is non-nil, appends silence between paragraphs if the engine requires it (currently ElevenLabs), and joins all chunks into the final output file using =ffmpeg= concatenation.  It deletes original chunk files if ~tlon-tts-delete-file-chunks~ is set, opens a Dired buffer at the output directory, and plays the final file.

*** Delete chunks of file
:PROPERTIES:
:CUSTOM_ID: h:delete-chunks
:END:
#+findex: tlon-tts-delete-chunks-of-file

You call ~tlon-tts-delete-chunks-of-file~ to delete all audio chunk files associated with a base filename (matching the pattern =basename-chunk-*.ext=).  It also kills the corresponding staging buffer if it exists.  When invoked in Dired, it operates on the file at point; otherwise it prompts for a file.

*** Truncate audio file
:PROPERTIES:
:CUSTOM_ID: h:truncate-audio
:END:
#+findex: tlon-tts-truncate-audio-file

You call ~tlon-tts-truncate-audio-file~ to remove a specified number of milliseconds from the end of an audio file using =mp3splt=.  It prompts for the file and the duration to remove.

*** Normalize audio files
:PROPERTIES:
:CUSTOM_ID: h:normalize-audio-files
:END:
#+findex: tlon-tts-normalize-audio-files

You call ~tlon-tts-normalize-audio-files~ to normalize one or more audio files in place using the =ffmpeg= loudnorm filter.  When invoked from Dired with marked files, those files are used; otherwise you are prompted for a single file.  Each file is normalized to a temporary file and then replaced.

*** Move file to audio server
:PROPERTIES:
:CUSTOM_ID: h:move-to-server
:END:
#+findex: tlon-tts-move-file-to-audio-server

You call ~tlon-tts-move-file-to-audio-server~ to move a generated audio file to the =uqbar-audio= repository, under a path like =uqbar-audio/lang/bare-dir/filename.ext=.  The language and bare directory (e.g., "articulos", "temas") are inferred from the source file's path.  It then prompts whether to stage, commit, and push the change via Magit.  If chunk files exist for the moved file, it also prompts whether to delete them.

*** Open audio directory
:PROPERTIES:
:CUSTOM_ID: h:open-audio-dir
:END:
#+findex: tlon-tts-open-audio-directory

You call ~tlon-tts-open-audio-directory~ to open the audio repository directory for the current language (or a prompted one) in Dired.

** Reporting and data editing
:PROPERTIES:
:CUSTOM_ID: h:reporting-and-editing
:END:

*** Generate report
:PROPERTIES:
:CUSTOM_ID: h:generate-report
:END:
#+findex: tlon-tts-generate-report

You call ~tlon-tts-generate-report~ from a staging buffer to generate a =*TTS Report*= buffer highlighting potential issues that might affect narration quality.  It checks for missing acronym definitions (acronyms present in the buffer but not defined in any abbreviation list), potential chemical symbols, unprocessed emphasis markers, unprocessed en dashes, and numbers lacking thousands separators.

*** Edit global abbreviations
:PROPERTIES:
:CUSTOM_ID: h:edit-global-abbreviations
:END:
#+findex: tlon-tts-edit-global-abbreviations

You call ~tlon-tts-edit-global-abbreviations~ to add or edit a global abbreviation.  You are prompted for a language (including the special value ="default"=, which applies to all languages), an abbreviation, and its spoken form.  The mapping is stored in the JSON file for the selected language under the =babel-core= repository.

*** Edit global phonetic replacements
:PROPERTIES:
:CUSTOM_ID: h:edit-global-phonetic-replacements
:END:
#+findex: tlon-tts-edit-global-phonetic-replacements

You call ~tlon-tts-edit-global-phonetic-replacements~ to add or edit a global phonetic replacement.  Like abbreviations, you are prompted for a language, a term to replace, and its replacement text.

*** Edit global phonetic transcriptions
:PROPERTIES:
:CUSTOM_ID: h:edit-global-phonetic-transcriptions
:END:
#+findex: tlon-tts-edit-global-phonetic-transcriptions

You call ~tlon-tts-edit-global-phonetic-transcriptions~ to add or edit a global phonetic (IPA) transcription.  When adding a new term, the command asks an AI model for a suggested IPA transcription and pre-fills the prompt with it.  The transcription is stored in the corresponding JSON file.

*** Add local abbreviation
:PROPERTIES:
:CUSTOM_ID: h:add-local-abbreviation
:END:
#+findex: tlon-add-local-abbreviation

You call ~tlon-add-local-abbreviation~ to add a file-local abbreviation to the =tlon-local-abbreviations= variable in the current buffer's local variables section.  You are prompted for the abbreviation and its expanded form.

*** Add local replacement
:PROPERTIES:
:CUSTOM_ID: h:add-local-replacement
:END:
#+findex: tlon-add-local-replacement

You call ~tlon-add-local-replacement~ to add a file-local phonetic replacement to the =tlon-local-replacements= variable in the current buffer's local variables section.  You are prompted for the text to replace and its replacement.

** Quick TTS
:PROPERTIES:
:CUSTOM_ID: h:quick-tts
:END:
#+findex: tlon-tts-quick

You call ~tlon-tts-quick~ to quickly generate an audio file from text using the OpenAI TTS engine.  When called interactively, it uses the active region or prompts for text, then prompts for the output file path.  The voice used is specified by ~tlon-tts-quick-voice~ ([[#h:quick-tts-settings][described above]]).  The JSON payload is written to a temporary file to avoid shell quoting issues.

** ElevenLabs voice listing
:PROPERTIES:
:CUSTOM_ID: h:elevenlabs-voices
:END:
#+findex: tlon-tts-elevenlabs-get-voices

You call ~tlon-tts-elevenlabs-get-voices~ to retrieve a JSON listing of all available ElevenLabs voices and display them in a formatted buffer.

** Transient menu
:PROPERTIES:
:CUSTOM_ID: h:transient-menu
:END:
#+findex: tlon-tts-menu

The command ~tlon-tts-menu~ activates a =transient= menu interface that surfaces all TTS operations and options in three columns:

- *Narration*: stage content (=s=), narrate buffer (=b=), narrate chunks (=c=), generate report (=e=), plus infixes for paragraph break duration (=-b=), prompt (=-p=), narrate sidenotes (=-s=), alternate voice (=-v=), engine (=-e=), and engine settings (=-t=).

- *File processing*: finalize audio (=f=), move to audio dir (=m=), delete chunks (=d=), normalize files (=n=), truncate file (=x=), open audio dir (=o=), plus infixes for normalize audio (=-n=), loudness target (=-l=), delete chunks after finalizing (=-d=), and generate missing chunks only (=-k=).

- *Edit*: global abbreviation (=a=), replacement (=r=), transcription (=t=); local abbreviation (=A=), replacement (=R=).

* Functions
:PROPERTIES:
:CUSTOM_ID: h:functions
:END:

This section documents noteworthy public functions that are not interactive commands.

** Staging and preparation
:PROPERTIES:
:CUSTOM_ID: h:staging-functions
:END:

#+findex: tlon-tts-get-staging-buffer-name
~tlon-tts-get-staging-buffer-name~ returns the name of the staging buffer for a given file (e.g., =*TTS: filename*=).

#+findex: tlon-tts-set-file-local-vars
~tlon-tts-set-file-local-vars~ sets all file-local variables in the staging buffer: source, language, engine, audio format, voice, and locale.  Each argument defaults to an auto-detected value if not provided.

#+findex: tlon-tts-prepare-staging-buffer
~tlon-tts-prepare-staging-buffer~ orchestrates the full text preparation pipeline in the staging buffer.  It processes notes, removes unwanted sections, replaces BibTeX keys with citations, adds listener cues, processes links, removes formatting, adds paragraph breaks, processes currencies and numerals, expands abbreviations, handles phonetic replacements and transcriptions, removes unsupported SSML tags, cleans up newlines, and generates the quality report.

#+findex: tlon-tts-staging-buffer-p
~tlon-tts-staging-buffer-p~ returns non-nil if the current buffer is a TTS staging buffer (i.e., the buffer-local variable ~tlon-tts-staging-buffer-p~ is bound and true).

** Chunking
:PROPERTIES:
:CUSTOM_ID: h:chunking-functions
:END:

#+findex: tlon-tts-calculate-chunks
~tlon-tts-calculate-chunks~ analyzes the staging buffer content and determines how to split it into chunks based on the engine's character limit and voice changes.  When the character limit is nil (the case for ElevenLabs by default), each paragraph becomes its own chunk, which helps mitigate voice degradation on longer texts.  Voice changes specified in ~tlon-tts-voice-chunks~ always force a chunk break.  The function populates the buffer-local ~tlon-tts-chunks~ list.

#+findex: tlon-tts-break-into-chunks
~tlon-tts-break-into-chunks~ is the core function that splits text in the current buffer into chunks.  It respects paragraph boundaries, engine character limits, and voice change markers.  Each chunk stores a 1-based chunk number, the text, voice parameters, the output filename, processing status, markers, and other metadata.

#+findex: tlon-tts-insert-chunk-comments
~tlon-tts-insert-chunk-comments~ inserts =<!-- Chunk N -->= comments before each chunk in the staging buffer, using the markers and chunk numbers stored in ~tlon-tts-chunks~.

#+findex: tlon-tts-get-chunk-name
~tlon-tts-get-chunk-name~ returns the filename for a chunk, formatted as =basename-chunk-NNN.ext= where NNN is the zero-padded 1-based chunk number.

#+findex: tlon-tts-get-list-of-chunks
~tlon-tts-get-list-of-chunks~ returns a sorted list of all existing chunk files matching a base filename pattern.

** Audio processing
:PROPERTIES:
:CUSTOM_ID: h:audio-processing-functions
:END:

#+findex: tlon-tts-finish-processing
~tlon-tts-finish-processing~ is the internal function that finalizes the TTS process.  It normalizes asynchronously (if enabled), appends silence (if the engine requires it), and joins chunks into the final output file.

#+findex: tlon-tts-append-silence-to-chunks
~tlon-tts-append-silence-to-chunks~ appends silence of the duration specified by ~tlon-tts-paragraph-break-duration~ to each chunk file, creating temporary files.  It returns the list of temporary file paths.

#+findex: tlon-tts-append-silence-to-chunks-p
~tlon-tts-append-silence-to-chunks-p~ returns non-nil if silence should be appended to chunks for the given file, which is currently the case only when the engine is ElevenLabs.

#+findex: tlon-tts-get-original-filename
~tlon-tts-get-original-filename~ returns the base filename from which a chunk file derives by removing the =-chunk-NNN= suffix.

** Voice selection
:PROPERTIES:
:CUSTOM_ID: h:voice-functions
:END:

#+findex: tlon-tts-select-voice
~tlon-tts-select-voice~ prompts you to select from available voices in the current language for the current engine, displaying the voice name, gender, and role.

#+findex: tlon-tts-get-voice-at-point
~tlon-tts-get-voice-at-point~ returns the voice ID of the =voice= SSML tag enclosing point, or the default voice (~tlon-tts-voice~) if point is not inside a =voice= tag.

#+findex: tlon-tts-get-voice-of-role
~tlon-tts-get-voice-of-role~ returns the voice ID associated with a given role (="main"=, ="alternate"=, ="male"=, ="female"=, ="alternate-gender"=, or ="inherit"=) for the current engine and language.
