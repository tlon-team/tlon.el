This manual describes the features and customization options for the Emacs Lisp file =tlon-clock.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

The =tlon-clock= module provides functions for interacting with Org mode's clocking mechanism (=org-clock=), tailored to the Tlon workflow.  It extracts metadata from the currently clocked heading -- BibTeX keys, associated files, forge issues, workflow actions and labels -- and provides commands to open the relevant resources.  It also includes utility functions for copying buffer or region contents and for setting up side-by-side editing windows for original and translation files.

* Commands
:PROPERTIES:
:CUSTOM_ID: h:commands
:END:

** ~tlon-open-clock-file~
:PROPERTIES:
:CUSTOM_ID: h:tlon-open-clock-file
:END:
#+findex: tlon-open-clock-file

Open the file associated with the currently clocked task.  The command extracts the BibTeX key from the clocked heading (via ~tlon-get-clock-key~), resolves it to a file path (via ~tlon-get-file-from-key~), and visits that file with ~find-file~.  You use this when you want quick access to the document you are currently working on.

** ~tlon-open-clock-issue~
:PROPERTIES:
:CUSTOM_ID: h:tlon-open-clock-issue
:END:
#+findex: tlon-open-clock-issue

Open the Forge issue linked in the currently clocked heading.  The command navigates to the clocked heading, searches for an =orgit-forge= link (matching =orgit-topic:GID=), extracts the issue GID, and calls ~forge-visit-issue~ to open it.  This requires the =forge= package and an =orgit-topic= link in the heading.  If no clock is running, the command signals an error.

** ~tlon-copy-dwim~
:PROPERTIES:
:CUSTOM_ID: h:tlon-copy-dwim
:END:
#+findex: tlon-copy-dwim

Copy text to the kill ring using a "Do What I Mean" approach.  If a region is active, copy the region; otherwise, copy the entire contents of the current buffer.  This is a general-purpose utility command.

* Functions
:PROPERTIES:
:CUSTOM_ID: h:functions
:END:

** Retrieving clock information
:PROPERTIES:
:CUSTOM_ID: h:clock-getters
:END:

#+findex: tlon-get-clock
- ~tlon-get-clock~ :: Return the raw string of the currently clocked heading (=org-clock-current-task=), with text properties stripped.  Signals an error if no clock is running.

#+findex: tlon-get-clock-key
- ~tlon-get-clock-key~ :: Extract and return the BibTeX key from the clocked heading.  The key is expected to be enclosed in backticks; the regular expression ~tlon-key-regexp~ handles an optional trailing =.md= extension.  Signals an error if no key is found.  This function is autoloaded.

#+findex: tlon-get-clock-file
- ~tlon-get-clock-file~ :: Return the file path corresponding to the BibTeX key in the clocked heading.  Calls ~tlon-get-clock-key~ to obtain the key and ~tlon-get-file-from-key~ to resolve it to a path.

#+findex: tlon-get-clock-issue
- ~tlon-get-clock-issue~ :: Return the issue GID from the first =orgit-forge= link found in the clocked heading's subtree.  Navigates to the clocked heading with ~org-clock-goto~, narrows to the subtree, and searches for an =orgit-topic:GID= link.  Returns the GID string or =nil= if none is found.  Signals an error if no clock is running.

#+findex: tlon-get-clock-action
- ~tlon-get-clock-action~ :: Return the workflow action word from the clocked heading.  The action is assumed to be the second word in the heading string.  The function validates it against the list of known actions from ~tlon-label-lookup-all~.  As a side effect it also verifies that a BibTeX key is present in the heading.

#+findex: tlon-get-clock-phase
- ~tlon-get-clock-phase~ :: Return the phase associated with the action in the clocked heading.  Uses ~tlon-label-lookup~ to map the action (from ~tlon-get-clock-action~) to its corresponding phase.

#+findex: tlon-get-clock-next-phase
- ~tlon-get-clock-next-phase~ :: Return the phase that follows the current clock phase in ~tlon-job-labels~.  This is used to determine the next step in the workflow sequence.

** Utility functions
:PROPERTIES:
:CUSTOM_ID: h:utility-functions
:END:

#+findex: tlon-next-value
- ~tlon-next-value~ :: Return the "next" value of PROPERTY after the item with VALUE in ALIST.  This is a generic helper used by ~tlon-get-clock-next-phase~ to walk a plist-based alist.

#+findex: tlon-copy-buffer
- ~tlon-copy-buffer~ :: Copy the entire contents of FILE (or the current buffer if FILE is =nil=) to the kill ring.

#+findex: tlon-copy-region
- ~tlon-copy-region~ :: Copy the contents between BEG and END to the kill ring.

#+findex: tlon-set-paths-from-clock
- ~tlon-set-paths-from-clock~ :: Return the file paths for the original document, its translation, and the BibTeX key, based on the clocked task.  Uses ~tlon-get-clock-key~ to find the key, looks up the translation file in repository metadata, and derives the original via ~tlon-get-counterpart~.  Returns multiple values via ~cl-values~.

#+findex: tlon-set-windows
- ~tlon-set-windows~ :: Open ORIGINAL-PATH and TRANSLATION-PATH in windows 1 and 2, respectively.  Splits the frame if it is not already split (via =window-extras-split-if-unsplit=) and uses =winum= to select the target windows.

** Variables
:PROPERTIES:
:CUSTOM_ID: h:variables
:END:

#+vindex: tlon-key-regexp
- ~tlon-key-regexp~ :: Regular expression used to match BibTeX keys in clocked headings.  It matches text enclosed in backticks, with an optional trailing =.md= extension handled by a second capture group.
