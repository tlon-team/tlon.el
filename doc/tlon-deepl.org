This manual describes the features and customization options for the Emacs Lisp file =tlon-deepl.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

This module provides functions for interacting with the DeepL API, enabling text translation and management of DeepL glossaries. It relies on an API key stored securely via =auth-source= (using the entry =tlon/babel/deepl.com/[WORK_EMAIL]=).

Key capabilities:

- Translate text via the DeepL API, with automatic glossary lookup when a matching glossary exists.
- Create, update, delete, and retrieve DeepL glossaries.
- Refresh the list of supported glossary language pairs from the DeepL API.
- Compare a local translation with its DeepL-generated counterpart using =ediff=.
- Work around newline handling issues in the =quality_optimized= model.

* User options
:PROPERTIES:
:CUSTOM_ID: h:user-options
:END:

** ~tlon-deepl-model-type~
:PROPERTIES:
:CUSTOM_ID: h:tlon-deepl-model-type
:END:
#+vindex: tlon-deepl-model-type

Model type for DeepL translation. The default value is ="quality_optimized"=. Possible values are:

- ="latency_optimized"=: uses lower latency "classic" translation models, which support all language pairs; this is the DeepL default.
- ="quality_optimized"=: uses higher latency, improved quality "next-gen" translation models, which support only a subset of language pairs; if a language pair that is not supported by next-gen models is included in the request, it will fail.
- ="prefer_quality_optimized"=: prioritizes use of higher latency, improved quality "next-gen" translation models; if a request includes a language pair not supported by next-gen models, the request will fall back to =latency_optimized= classic models.

See [[https://developers.deepl.com/docs/api-reference/translate#request-body-descriptions][DeepL API reference]] for details.

** Translation
:PROPERTIES:
:CUSTOM_ID: h:translation-commands
:END:

*** ~tlon-deepl-translate~
:PROPERTIES:
:CUSTOM_ID: h:tlon-deepl-translate
:END:
#+findex: tlon-deepl-translate

Translate TEXT from SOURCE-LANG into TARGET-LANG and execute CALLBACK. If SOURCE-LANG is nil, prompt for a source language. If CALLBACK is nil, execute ~tlon-deepl-print-translation~, which copies the translation to the kill ring and displays it. If NO-GLOSSARY is non-nil, do not ask for confirmation when no glossary is found. If NO-GLOSSARY is the symbol =skip=, skip the translation and return nil without signaling an error.

When called interactively, the command prompts for:

1. A source language code.
2. A target language code (excluding the source language).
3. The text to translate, defaulting to the active region or the word at point.

Glossary behavior:

- If both source and target languages support DeepL glossaries and a matching glossary exists in the local cache (~tlon-deepl-glossaries~), the glossary ID is included in the request.
- If the language pair supports glossaries but no matching glossary is found, you are prompted for confirmation before proceeding (unless NO-GLOSSARY is non-nil).

Returns the translated text as a string, or nil if skipped.

*** ~tlon-deepl-diff~
:PROPERTIES:
:CUSTOM_ID: h:tlon-deepl-diff
:END:
#+findex: tlon-deepl-diff

Run an =ediff= session to compare a TRANSLATION file with the DEEPL translation of it. If TRANSLATION is nil, use the current buffer's file. If DEEPL is nil, prompt for a file.

** Glossaries
:PROPERTIES:
:CUSTOM_ID: h:glossary-commands
:END:

*** ~tlon-deepl-select-glossary~
:PROPERTIES:
:CUSTOM_ID: h:tlon-deepl-select-glossary
:END:
#+findex: tlon-deepl-select-glossary

Interactively select an existing DeepL glossary from the locally cached list (~tlon-deepl-glossaries~). Uses ~completing-read~ with formatted glossary names showing name, languages, entry count, and ID. Returns the selected glossary ID.

*** ~tlon-deepl-get-glossaries~
:PROPERTIES:
:CUSTOM_ID: h:tlon-deepl-get-glossaries
:END:
#+findex: tlon-deepl-get-glossaries

Retrieve and display the list of glossaries from the DeepL API. The callback updates the ~tlon-deepl-glossaries~ variable. This command is also called automatically after glossary creation or deletion.

*** ~tlon-deepl-glossary-create~
:PROPERTIES:
:CUSTOM_ID: h:tlon-deepl-glossary-create
:END:
#+findex: tlon-deepl-glossary-create

Create a new DeepL glossary for SOURCE-LANGUAGE to TARGET-LANGUAGE. When called interactively, prompts for the source and target language codes. Extracts the glossary data for that pair in TSV format using ~tlon-extract-glossary~, then uploads it to the DeepL API. The callback refreshes the local glossary list and messages the API response.

*** ~tlon-deepl-glossary-update~
:PROPERTIES:
:CUSTOM_ID: h:tlon-deepl-glossary-update
:END:
#+findex: tlon-deepl-glossary-update

Update a DeepL glossary for SOURCE-LANGUAGE to TARGET-LANGUAGE by deleting it (if it exists) and then recreating it with the current local glossary data. When called interactively, prompts for the source and target language codes.

*** ~tlon-deepl-glossary-delete~
:PROPERTIES:
:CUSTOM_ID: h:tlon-deepl-glossary-delete
:END:
#+findex: tlon-deepl-glossary-delete

Delete a DeepL glossary. When called interactively without arguments, prompts you to choose a glossary. Programmatically, you can supply LANGUAGE-OR-ID as:

- A target language code (three characters or fewer, e.g., ="es"=), which is resolved against the current source language.
- A cons pair =(SOURCE . TARGET)= of language codes.
- A glossary ID string.

After successful deletion, execute CALLBACK (if non-nil).

*** ~tlon-deepl-maybe-glossary-update~
:PROPERTIES:
:CUSTOM_ID: h:tlon-deepl-maybe-glossary-update
:END:
#+findex: tlon-deepl-maybe-glossary-update

Update a DeepL glossary when supported. If LANGUAGE-OR-PAIR is a cons =(SRC . TGT)=, update that pair when both languages support glossaries. If it is a string code, update =EN= to that language for backward compatibility. Does nothing when the language pair does not support glossaries.

*** ~tlon-deepl-refresh-supported-glossary-languages~
:PROPERTIES:
:CUSTOM_ID: h:tlon-deepl-refresh-supported-glossary-languages
:END:
#+findex: tlon-deepl-refresh-supported-glossary-languages

Refresh supported glossary language pairs from DeepL and update the local caches (~tlon-deepl-supported-glossary-pairs~ and ~tlon-deepl-supported-glossary-languages~). Reports how many pairs and language codes were retrieved.

* Functions
:PROPERTIES:
:CUSTOM_ID: h:functions
:END:

** ~tlon-deepl-get-key~
:PROPERTIES:
:CUSTOM_ID: h:tlon-deepl-get-key
:END:
#+findex: tlon-deepl-get-key

Return the DeepL API key, initializing it lazily if needed. The key is retrieved from =auth-source= using the entry =tlon/babel/deepl.com/[WORK_EMAIL]=.

** ~tlon-deepl-print-translation~
:PROPERTIES:
:CUSTOM_ID: h:tlon-deepl-print-translation
:END:
#+findex: tlon-deepl-print-translation

Default callback for translation requests. Parses the JSON response from the current buffer, extracts the translated text, copies it to the kill ring, and displays it. If the API returns an error object inside the translations array, it displays the error and returns nil. Returns the translation string or nil if parsing fails.

** ~tlon-deepl-get-language-glossary~
:PROPERTIES:
:CUSTOM_ID: h:tlon-deepl-get-language-glossary
:END:
#+findex: tlon-deepl-get-language-glossary

Return the glossary ID for LANGUAGE paired with SOURCE. Looks up ~tlon-deepl-glossaries~ based on SOURCE (defaults to ~tlon-translate-source-language~) and target LANGUAGE. Returns nil if no matching glossary is found.

** ~tlon-deepl-get-list-of-glossaries~
:PROPERTIES:
:CUSTOM_ID: h:tlon-deepl-get-list-of-glossaries
:END:
#+findex: tlon-deepl-get-list-of-glossaries

Return a list of cons cells =(DISPLAY-STRING . ID)= from the cached glossary data in ~tlon-deepl-glossaries~. Each display string includes the glossary name, source/target languages, entry count, and ID. Used by ~tlon-deepl-select-glossary~ for ~completing-read~.

* Transient menu
:PROPERTIES:
:CUSTOM_ID: h:menu
:END:
#+findex: tlon-deepl-menu

The command ~tlon-deepl-menu~ activates a =transient= menu interface for DeepL operations. It provides the following groups and commands:

- *Translate*:
  - =e= :: Ediff translation (~tlon-deepl-diff~)
- *Glossaries*:
  - =s= :: Select (~tlon-deepl-select-glossary~)
  - =r= :: Retrieve (~tlon-deepl-get-glossaries~)
  - =c= :: Create (~tlon-deepl-glossary-create~)
  - =u= :: Update (~tlon-deepl-glossary-update~)
  - =d= :: Delete (~tlon-deepl-glossary-delete~)
- *Options*:
  - =-m= :: Model type (~tlon-deepl-model-type-infix~)
