This manual describes the features and customization options for the Emacs Lisp file =tlon-images.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

The =tlon-images= module provides image processing functionality for the Tlon workflow.  It supports reducing brightness, inverting colors, and removing transparency using ImageMagick; automatically choosing between inversion and darkening via the =invertornot.com= API; generating AI-powered alt text descriptions; and downloading remote images referenced in Markdown files into structured local directories with automatic conversion to =Figure= tags.

The module depends on =tlon-core=, =tlon-ai=, =tlon-md=, =transient=, =dired=, and =url-parse=.  ImageMagick must be installed for image processing commands.

* User options
:PROPERTIES:
:CUSTOM_ID: h:user-options
:END:

** ~tlon-images-default-brightness-reduction~
:PROPERTIES:
:CUSTOM_ID: h:tlon-images-default-brightness-reduction
:END:
#+vindex: tlon-images-default-brightness-reduction

Default percentage (integer) by which to reduce image brightness when using ~tlon-images-reduce-brightness~ without specifying a percentage.  The default is =20=.

** ~tlon-images-open-after-processing~
:PROPERTIES:
:CUSTOM_ID: h:tlon-images-open-after-processing
:END:
#+vindex: tlon-images-open-after-processing

Whether to open the original (light theme copy) and processed (dark theme) images in separate windows after processing.  The default is =t=.  When using ~tlon-images-download-from-markdown~, a non-nil value also opens the target directory in Dired.

** ~tlon-images-read-file-without-asking~
:PROPERTIES:
:CUSTOM_ID: h:tlon-images-read-file-without-asking
:END:
#+vindex: tlon-images-read-file-without-asking

Whether to process the image at point (in Dired or the current buffer) without prompting for confirmation.  The default is =nil=, meaning the user is always prompted.

** ~tlon-images-overwrite-alt-text~
:PROPERTIES:
:CUSTOM_ID: h:tlon-images-overwrite-alt-text
:END:
#+vindex: tlon-images-overwrite-alt-text

Whether to overwrite existing alt text when using ~tlon-images-set-image-alt-text-in-buffer~.  The default is =nil=, preserving existing alt text.  This option does not affect ~tlon-images-set-image-alt-text-in-file~, which always overwrites.

** ~tlon-images-preserve-markdown-alt-text~
:PROPERTIES:
:CUSTOM_ID: h:tlon-images-preserve-markdown-alt-text
:END:
#+vindex: tlon-images-preserve-markdown-alt-text

Whether to preserve Markdown alt text when replacing Markdown images with =Figure= tags (e.g., during ~tlon-images-download-from-markdown~).  If non-nil, the Markdown alt text is copied into the =alt= attribute of the =Figure= tag.  The default is =nil=, meaning the =alt= attribute is omitted.

* Commands
:PROPERTIES:
:CUSTOM_ID: h:commands
:END:

** Image processing for dark themes
:PROPERTIES:
:CUSTOM_ID: h:dark-theme-processing
:END:

*** ~tlon-images-auto-process~
:PROPERTIES:
:CUSTOM_ID: h:tlon-images-auto-process
:END:
#+findex: tlon-images-auto-process

Automatically process an IMAGE for a dark theme.  The command prompts for the image file, checks whether its colors can be safely inverted (via the =invertornot.com= API using ~tlon-images-can-invert-p~), and either inverts the colors or reduces the brightness accordingly.  The output is saved to a dark-themed filename (e.g., =image-dark.png=).

*** ~tlon-images-reduce-brightness~
:PROPERTIES:
:CUSTOM_ID: h:tlon-images-reduce-brightness
:END:
#+findex: tlon-images-reduce-brightness

Reduce the brightness of a SOURCE image by PERCENT (defaulting to ~tlon-images-default-brightness-reduction~) and save the result to TARGET (defaulting to the dark-themed filename).  Uses ImageMagick's =-modulate= flag.  The original is also copied to a light-themed filename.  If ~tlon-images-open-after-processing~ is non-nil, both versions are opened in separate windows.

*** ~tlon-images-invert-colors~
:PROPERTIES:
:CUSTOM_ID: h:tlon-images-invert-colors
:END:
#+findex: tlon-images-invert-colors

Invert the RGB colors of a SOURCE image and save the result to TARGET (defaulting to the dark-themed filename).  Uses ImageMagick's =-channel RGB -negate= flags.  The original is also copied to a light-themed filename.

*** ~tlon-images-make-nontransparent~
:PROPERTIES:
:CUSTOM_ID: h:tlon-images-make-nontransparent
:END:
#+findex: tlon-images-make-nontransparent

Remove the transparent background of a SOURCE image and replace it with a solid BACKGROUND color (defaulting to ="white"=).  The result is saved to TARGET, which defaults to the source file itself (overwriting it).  Uses ImageMagick's =-background= and =-flatten= flags.

** Alt text generation
:PROPERTIES:
:CUSTOM_ID: h:alt-text
:END:

*** ~tlon-images-describe-image~
:PROPERTIES:
:CUSTOM_ID: h:tlon-images-describe-image
:END:
#+findex: tlon-images-describe-image

Generate a concise description of an image FILE using an AI model.  The description language is determined from the file's context, and a language-specific prompt (from ~tlon-images-describe-image-prompt~) requests a one-to-two-sentence summary of at most 50 words.  By default the description is displayed in the minibuffer; if CALLBACK is non-nil, it is called with the response and info instead.

*** ~tlon-images-set-image-alt-text-in-file~
:PROPERTIES:
:CUSTOM_ID: h:tlon-images-set-image-alt-text-in-file
:END:
#+findex: tlon-images-set-image-alt-text-in-file

Generate and insert an alt text description for the =Figure= or =OurWorldInData= tag at point.  Extracts the =src= attribute, resolves it to a local file, calls ~tlon-images-describe-image~, and inserts the result into the =alt= attribute.  Always overwrites existing alt text.  Signals an error if no recognized image tag is found at point.

*** ~tlon-images-set-image-alt-text-in-buffer~
:PROPERTIES:
:CUSTOM_ID: h:tlon-images-set-image-alt-text-in-buffer
:END:
#+findex: tlon-images-set-image-alt-text-in-buffer

Iterate through all =Figure= and =OurWorldInData= tags in the current buffer and set their alt text.  Whether existing alt text is overwritten depends on the value of ~tlon-images-overwrite-alt-text~.

** Image downloading
:PROPERTIES:
:CUSTOM_ID: h:image-downloading
:END:

*** ~tlon-images-download-from-markdown~
:PROPERTIES:
:CUSTOM_ID: h:tlon-images-download-from-markdown
:END:
#+findex: tlon-images-download-from-markdown

Scan a Markdown FILE for all image URLs, download them, and store them in a local directory derived from the file's path relative to the repository root.  For example, for =<repo>/articles/my-post.md=, images are saved in =<repo>/<image-dir>/articles/my-post/=.

The command performs the following steps:

1. If the target directory already exists and contains files, prompt you to confirm deletion.
2. Create the target directory.
3. Download each image, determining the file extension from magic numbers, then the =Content-Type= header, then the URL.  Skip unrecognized formats.
4. Save images with sequential names (e.g., =figure-01.png=, =figure-02.jpg=).
5. Replace each Markdown image (=![alt](url)=) with a =Figure= tag pointing to the local file.  Whether the =alt= attribute is populated depends on ~tlon-images-preserve-markdown-alt-text~.
6. If ~tlon-images-open-after-processing~ is non-nil, open the target directory in Dired.

** Image directory
:PROPERTIES:
:CUSTOM_ID: h:image-directory
:END:

*** ~tlon-images-get-dir~
:PROPERTIES:
:CUSTOM_ID: h:tlon-images-get-dir
:END:
#+findex: tlon-images-get-dir

Return the images directory for FILE.  The path is constructed as =<repo-root>/<image-dir-name>/<slug>=, where =<image-dir-name>= is language-specific (looked up from ~tlon-image-dirs~) and =<slug>= is the relative path of FILE inside the repository, without extension.

** Transient menu
:PROPERTIES:
:CUSTOM_ID: h:tlon-images-menu
:END:
#+findex: tlon-images-menu

The command ~tlon-images-menu~ activates a =transient= menu for image processing.

The menu provides the following groups:

- *Light/Dark*:
  - =a= :: auto process (~tlon-images-auto-process~)
  - =r= :: reduce brightness (~tlon-images-reduce-brightness~)
  - =i= :: invert colors (~tlon-images-invert-colors~)
  - =n= :: make nontransparent (~tlon-images-make-nontransparent~)
  - Options: toggle open after processing (=-o=), set brightness reduction percentage (=-r=)
- *Alt text*:
  - =d= :: display alt text (~tlon-images-describe-image~)
  - =t= :: set alt text in tag (~tlon-images-set-image-alt-text-in-file~)
  - =b= :: set alt text in buffer (~tlon-images-set-image-alt-text-in-buffer~)
  - Options: toggle overwrite alt text (=-l=)
- *Other*:
  - =w= :: download images in file (~tlon-images-download-from-markdown~)
  - Options: toggle preserve alt text (=-w=)
- *General options*:
  - =-f= :: toggle read file without asking

** Image reading and processing helpers
:PROPERTIES:
:CUSTOM_ID: h:image-helpers
:END:

#+findex: tlon-images-read-image
- ~tlon-images-read-image~ :: Prompt for an image file path.  In Dired, default to the file at point.  If the current buffer visits an image, default to that.  If ~tlon-images-read-file-without-asking~ is non-nil, skip the prompt and use the default directly.

#+findex: tlon-images-process-image
- ~tlon-images-process-image~ :: Execute the ImageMagick COMMAND for SOURCE and TARGET.  Copy SOURCE to a light-themed filename, optionally open both versions, and display MESSAGE-FMT with the source file name.

#+findex: tlon-images-handle-output
- ~tlon-images-handle-output~ :: Check the OUTPUT string from a shell command.  If empty, display the success MESSAGE.  If non-empty, signal a user error with the output content.

#+findex: tlon-images-get-themed-file-name
- ~tlon-images-get-themed-file-name~ :: Return a filename with a theme suffix appended before the extension.  For example, for FILE =image.png= and THEME =dark=, return =image-dark.png=.

#+findex: tlon-images-maybe-open-after-processing
- ~tlon-images-maybe-open-after-processing~ :: If ~tlon-images-open-after-processing~ is non-nil, open the ORIGINAL and PROCESSED images in separate windows, splitting the frame if necessary.

** invertornot.com integration
:PROPERTIES:
:CUSTOM_ID: h:invertornot
:END:

#+findex: tlon-images-post-file-to-invertornot
- ~tlon-images-post-file-to-invertornot~ :: Send an image FILE to the =invertornot.com/api/file= endpoint via a multipart POST request.  Return the parsed JSON response.

#+findex: tlon-images-post-url-to-invertornot
- ~tlon-images-post-url-to-invertornot~ :: Send an IMAGE-URL to the =invertornot.com/api/url= endpoint via a JSON POST request.  Return the parsed JSON response.

#+findex: tlon-images-handle-synchronous-response
- ~tlon-images-handle-synchronous-response~ :: Process the buffer from a synchronous HTTP request, extract the JSON payload, parse it, and return the result as an alist.

#+findex: tlon-images-can-invert-p
- ~tlon-images-can-invert-p~ :: Return =t= if the colors of IMAGE can be safely inverted (the API returns =1=), =nil= if they cannot (=0=), and signal an error for any other response.

** Alt text helpers
:PROPERTIES:
:CUSTOM_ID: h:alt-text-helpers
:END:

#+findex: tlon-images-describe-image-callback
- ~tlon-images-describe-image-callback~ :: Handle the AI response from ~tlon-images-describe-image~.  If ORIGINAL-CALLBACK is non-nil, call it; otherwise display RESPONSE as a message.  Restore =gptel-context= to PREVIOUS-CONTEXT.

#+findex: tlon-images-read-image-file
- ~tlon-images-read-image-file~ :: Read an image FILE from multiple sources: the provided FILE argument, the =src= attribute of a =Figure= tag at point, the current buffer (if it is an image), the file at point in Dired, or a user prompt.

#+findex: tlon-images-get-image-file-from-src
- ~tlon-images-get-image-file-from-src~ :: Resolve SRC to a local image file.  If SRC is an Our World in Data URL, download the thumbnail and return a temporary file path.  Otherwise, construct a local path relative to the repository root.
