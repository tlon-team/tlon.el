This manual describes the features and customization options for the Emacs Lisp file =tlon-dub.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

The =tlon-dub= module provides functionality for dubbing audio and video files into different languages.  It integrates with the ElevenLabs Dubbing API for cloud-based dubbing, uses WhisperX for local transcription and diarization, and includes tools for splitting/joining audio and video files, SRT manipulation (cleaning, resegmenting, converting to CSV), and audio extraction/replacement -- all using =ffmpeg=.

The module depends on =tlon-core=, =tlon-tts=, =tlon-whisperx=, =json=, =threads=, =transient=, and =cl-lib=.  External tools: =curl= (for API requests), =ffmpeg= and =ffprobe= (for audio/video manipulation), and =whisperx= (for local transcription).

** ElevenLabs API
:PROPERTIES:

*** ~tlon-dub-start-project~
:PROPERTIES:
:CUSTOM_ID: h:tlon-dub-start-project
:END:
#+findex: tlon-dub-start-project

Start a new ElevenLabs dubbing project.  Prompts for the SOURCE-FILE (audio or video), SOURCE-LANG (ISO code), TARGET-LANG (ISO code), and PROJECT-NAME.  Optionally accepts VOICE-ID (an ElevenLabs voice), DUBBING-STUDIO (whether to create in Dubbing Studio mode for web-based adjustments), and NUM-SPEAKERS (expected number of speakers; 0 means auto-detect).

The command validates that the source file has a supported media extension (e.g., mp3, wav, mp4, webm) before proceeding.  After successful creation, it automatically attempts to share the project with the user associated with the current API key (via ~tlon-dub--share-project-with-self~) with ="editor"= permissions.  Returns the parsed JSON response, which typically contains the =dubbing_id=.  Signals an error if the file type is unsupported or creation fails; logs a warning if sharing fails.

*** ~tlon-dub-get-project-metadata~
:PROPERTIES:
:CUSTOM_ID: h:tlon-dub-get-project-metadata
:END:
#+findex: tlon-dub-get-project-metadata

Retrieve metadata for an existing ElevenLabs dubbing project.  Prompts for the DUBBING-ID.  Returns the parsed JSON response containing project details (status, languages, etc.).

*** ~tlon-dub-get-dubbing~
:PROPERTIES:
:CUSTOM_ID: h:tlon-dub-get-dubbing
:END:
#+findex: tlon-dub-get-dubbing

Retrieve and parse the dubbing transcript (VTT format) for a specific language within an ElevenLabs dubbing project.  Prompts for the DUBBING-ID and LANGUAGE-CODE (ISO code).  The VTT response is parsed into a list of segment plists (=:start=, =:end=, =:text=) and displayed in a new buffer.  If parsing fails, the raw transcript is displayed instead.  Returns the segment list or the raw transcript string.

*** ~tlon-dub-get-resource-data~
:PROPERTIES:
:CUSTOM_ID: h:tlon-dub-get-resource-data
:END:
#+findex: tlon-dub-get-resource-data

Retrieve the detailed resource structure for an ElevenLabs dubbing project.  Prompts for the DUBBING-ID.  The response typically contains resource, speaker, and segment IDs needed for modifications.  Displays the parsed JSON (or raw response) in a new buffer.

*** ~tlon-dub-add-speaker-segment~
:PROPERTIES:
:CUSTOM_ID: h:tlon-dub-add-speaker-segment
:END:
#+findex: tlon-dub-add-speaker-segment

Add a new segment for a specific speaker within an ElevenLabs dubbing project.  Prompts for DUBBING-ID, SPEAKER-ID, START-TIME (seconds, float), END-TIME (seconds, float), and TEXT.

** Transcription and diarization
:PROPERTIES:
:CUSTOM_ID: h:transcription
:END:

*** ~tlon-dub-transcribe-with-whisperx~
:PROPERTIES:
:CUSTOM_ID: h:tlon-dub-transcribe-with-whisperx
:END:
#+findex: tlon-dub-transcribe-with-whisperx

Generate a transcript for AUDIO-FILE asynchronously using WhisperX.  FORMAT overrides ~tlon-dub-transcription-format~ (which can be set via the transient menu).  This is a wrapper around ~tlon-whisperx-transcribe~.

*** ~tlon-dub-diarize-with-whisperx~
:PROPERTIES:
:CUSTOM_ID: h:tlon-dub-diarize-with-whisperx
:END:
#+findex: tlon-dub-diarize-with-whisperx

Diarize AUDIO-FILE using WhisperX.  Optional LANGUAGE defaults to ="en"=.  If SPEAKERS is greater than 0, WhisperX is instructed to use exactly that many speakers.  This is a wrapper around ~tlon-whisperx-diarize~.

** SRT manipulation
:PROPERTIES:
:CUSTOM_ID: h:srt-manipulation
:END:

*** ~tlon-dub-resegment-srt~
:PROPERTIES:
:CUSTOM_ID: h:tlon-dub-resegment-srt
:END:
#+findex: tlon-dub-resegment-srt

Resegment SRT-FILE into larger segments.  A new segment starts whenever the speaker changes, and each segment is at least MIN-DURATION seconds (default 5).  Subject to these constraints, segments are kept as short as possible (closing when a sentence-ending punctuation mark is found).  The resegmented file is written next to the original with the suffix =-resegmented.srt=.  Returns the list of merged segments.

*** ~tlon-dub-clean-diarized-srt~
:PROPERTIES:
:CUSTOM_ID: h:tlon-dub-clean-diarized-srt
:END:
#+findex: tlon-dub-clean-diarized-srt

Convert a diarized SRT file (with =[SPEAKER_xx]= prefixes) into a cleaner SRT.  Interactively prompts for a human-readable name for each distinct speaker.  SPEAKER-ALIST may be supplied programmatically as =((\"00\" . \"Alice\") (\"01\" . \"Bob\"))=.  The first subtitle block of each speaker run is prefixed with ="Name: "=; subsequent contiguous blocks by the same speaker omit the prefix.  The cleaned file is written with the suffix =-cleaned.srt=.

*** ~tlon-dub-convert-srt-to-csv~
:PROPERTIES:
:CUSTOM_ID: h:tlon-dub-convert-srt-to-csv
:END:
#+findex: tlon-dub-convert-srt-to-csv

Convert a pair of timestamped English and translated SRT files into a CSV file formatted for ElevenLabs.  Prompts for the ENGLISH-SRT-FILE and TRANSLATED-SRT-FILE.  The output CSV (named after the English SRT file) has columns: =speaker,start_time,end_time,transcription,translation=.  The function validates that both SRT files have the same number of segments and that corresponding segments share identical timestamps; signals an error otherwise.

** Video splitting and joining
:PROPERTIES:
:CUSTOM_ID: h:video-splitting-joining
:END:

*** ~tlon-dub-split-video-at-timestamps~
:PROPERTIES:
:CUSTOM_ID: h:tlon-dub-split-video-at-timestamps
:END:
#+findex: tlon-dub-split-video-at-timestamps

Split VIDEO-FILE into parts based on a TIMESTAMPS-FILE (one =HH:MM:SS.mmm= per line).  Each part starts at its timestamp and ends 40 ms before the next timestamp (or at the end of the video for the final part).  Parts are saved as =<base>-partN.<ext>= in OUTPUT-DIR (defaulting to the video's directory).  When QUICK is non-nil, the video stream is copied without re-encoding (faster but cuts only at keyframes).

*** ~tlon-dub-split-video-at-timestamps-quick~
:PROPERTIES:
:CUSTOM_ID: h:tlon-dub-split-video-at-timestamps-quick
:END:
#+findex: tlon-dub-split-video-at-timestamps-quick

Convenience wrapper that calls ~tlon-dub-split-video-at-timestamps~ with QUICK set to non-nil.

*** ~tlon-dub-join-video-files~
:PROPERTIES:
:CUSTOM_ID: h:tlon-dub-join-video-files
:END:
#+findex: tlon-dub-join-video-files

Join video files whose paths are listed line-by-line in LIST-FILE into a single OUTPUT-FILE.  This is a convenience wrapper around ~tlon-dub-join-files~ intended for recombining segments created by ~tlon-dub-split-video-at-timestamps~.

** Audio manipulation
:PROPERTIES:
:CUSTOM_ID: h:audio-manipulation
:END:

*** ~tlon-dub-extract-audio~
:PROPERTIES:
:CUSTOM_ID: h:tlon-dub-extract-audio
:END:
#+findex: tlon-dub-extract-audio

Extract the audio track from a single VIDEO-FILE and save it as a WAV file alongside the original.  Uses =ffmpeg= with uncompressed =pcm_s16le= encoding.  Returns the path of the created WAV file.

*** ~tlon-dub-extract-audio-from-parts~
:PROPERTIES:
:CUSTOM_ID: h:tlon-dub-extract-audio-from-parts
:END:
#+findex: tlon-dub-extract-audio-from-parts

Extract audio (WAV) from all =<base>-partN= video files derived from VIDEO-FILE.  Prompts for the master video file, finds all matching part files in the same directory, and runs =ffmpeg= to create corresponding =.wav= files.  Returns the list of generated WAV filenames.  Signals an error if no part files are found.

*** ~tlon-dub-replace-audio~
:PROPERTIES:
:CUSTOM_ID: h:tlon-dub-replace-audio
:END:
#+findex: tlon-dub-replace-audio

Replace the audio track in VIDEO-FILE with AUDIO-FILE.  The video stream is copied without re-encoding; the audio is re-encoded as AAC.  If OUTPUT-FILE is =nil=, the result is saved as =<base>-replaced.<ext>=.  Returns the output path.

*** ~tlon-dub-split-audio-at-timestamps~
:PROPERTIES:
:CUSTOM_ID: h:tlon-dub-split-audio-at-timestamps
:END:
#+findex: tlon-dub-split-audio-at-timestamps

Split AUDIO-FILE into parts based on a TIMESTAMPS-FILE (one =HH:MM:SS.mmm= per line).  Each part is saved alongside the original as =<base>-partN.<ext>=.  Uses =ffmpeg= with stream copy (no re-encoding).

*** ~tlon-dub-join-audio-files~
:PROPERTIES:
:CUSTOM_ID: h:tlon-dub-join-audio-files
:END:
#+findex: tlon-dub-join-audio-files

Join audio files whose paths are listed in LIST-FILE into a single output.  This is a convenience wrapper around ~tlon-dub-join-files~.

** File joining
:PROPERTIES:
:CUSTOM_ID: h:file-joining
:END:

*** ~tlon-dub-join-files~
:PROPERTIES:
:CUSTOM_ID: h:tlon-dub-join-files
:END:
#+findex: tlon-dub-join-files

Join files listed line-by-line in LIST-FILE into OUTPUT-FILE.  If all listed files have a recognized media extension (e.g., mp4, wav, mp3), concatenation is performed with =ffmpeg= using the concat demuxer.  Otherwise, the files are treated as plain text and their contents are concatenated.  Blank lines and lines starting with =#= in LIST-FILE are ignored.  Signals an error if any listed file cannot be read or if =ffmpeg= fails.  Returns the path of OUTPUT-FILE.

** Transient menu
:PROPERTIES:
:CUSTOM_ID: h:tlon-dub-menu
:END:
#+findex: tlon-dub-menu

The command ~tlon-dub-menu~ activates a =transient= menu for dubbing functionality.

The menu provides the following groups:

- *Transcription & Timestamps (srt)*:
  - =t t= :: transcribe with WhisperX (~tlon-dub-transcribe-with-whisperx~)
  - =t i= :: diarize with WhisperX (~tlon-dub-diarize-with-whisperx~)
  - =t x= :: clean diarized SRT (~tlon-dub-clean-diarized-srt~)
  - =t c= :: convert SRTs to CSV (~tlon-dub-convert-srt-to-csv~)
  - =t r= :: resegment SRT (~tlon-dub-resegment-srt~)
  - Options: =t -f= for transcription format (~tlon-dub-infix-select-transcription-format~)
- *Audio & video manipulation*:
  - Split: =s v= (video, quick), =s V= (video, precise), =s a= (audio)
  - Join: =j v= (video), =j a= (audio)
  - Extract: =e a= (audio from video), =e A= (audio from parts)
  - Replace: =r a= (replace video audio)
- *ElevenLabs API*:
  - =a s= :: start new dubbing project (~tlon-dub-start-project~)
  - =a d= :: get project metadata (~tlon-dub-get-project-metadata~)
  - =a g= :: get dubbing transcript (~tlon-dub-get-dubbing~)
  - =a r= :: get resource data (~tlon-dub-get-resource-data~)
  - =a k= :: add speaker segment (~tlon-dub-add-speaker-segment~)
