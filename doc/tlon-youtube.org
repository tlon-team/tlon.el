This manual describes the features and customization options for the Emacs Lisp file =tlon-youtube.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=tlon-youtube= provides functionality for processing and uploading podcast episodes to YouTube. It covers the full workflow: generating animated wavelength videos from audio files, creating thumbnails with ImageMagick, uploading videos and thumbnails via the YouTube Data API v3 (using OAuth 2.0 with resumable uploads), and managing playlist assignments. Authentication is handled by the =oauth2-auto= library.

* User options
:PROPERTIES:
:CUSTOM_ID: h:user-options
:END:

** ~tlon-youtube-video-resolution~
:PROPERTIES:
:CUSTOM_ID: h:tlon-youtube-video-resolution
:END:
#+vindex: tlon-youtube-video-resolution

Video resolution =(WIDTH . HEIGHT)= for generated videos.

The default value is =(1280 . 720)= (720p). The type is a cons cell of two integers. Common resolutions:

- 720p: =(1280 . 720)=
- 1080p: =(1920 . 1080)=
- 1440p: =(2560 . 1440)=
- 4K (2160p): =(3840 . 2160)=

You can change this value via the transient menu ([[#h:tlon-youtube-menu-command][~tlon-youtube-menu~]]) using the "Video Resolution" option, which presents a predefined list of choices.

** ~tlon-youtube-api-key~
:PROPERTIES:
:CUSTOM_ID: h:tlon-youtube-api-key
:END:
#+vindex: tlon-youtube-api-key

YouTube Data API v3 key for API requests.

The default value is =nil=. The type is a string or =nil=.

Get this from the Google Cloud Console. This key is used for read-only API operations.

** ~tlon-youtube-client-id~
:PROPERTIES:
:CUSTOM_ID: h:tlon-youtube-client-id
:END:
#+vindex: tlon-youtube-client-id

OAuth 2.0 client ID for YouTube API authentication.

The default value is retrieved from =auth-source-pass= using the shared Tlon email. The type is a string or =nil=.

This must be a *Desktop app* client ID from the Google Cloud Console. Web application client IDs will not work with the =oauth2-auto= authorization flow. See [[#h:authentication-setup][authentication setup]] for details.

** ~tlon-youtube-client-secret~
:PROPERTIES:
:CUSTOM_ID: h:tlon-youtube-client-secret
:END:
#+vindex: tlon-youtube-client-secret

OAuth 2.0 client secret for YouTube API authentication.

The default value is retrieved from =auth-source-pass= using the shared Tlon email. The type is a string or =nil=.

Must correspond to the same Desktop app client as ~tlon-youtube-client-id~.

** ~tlon-youtube-default-privacy~
:PROPERTIES:
:CUSTOM_ID: h:tlon-youtube-default-privacy
:END:
#+vindex: tlon-youtube-default-privacy

Default privacy setting for uploaded videos.

The default value is ="private"=. Valid values are ="private"=, ="unlisted"=, and ="public"=.

This value is used as the default selection when you are prompted for a privacy setting during video upload.

** Video generation

*** ~tlon-youtube-generate-wavelength-video~
:PROPERTIES:
:CUSTOM_ID: h:tlon-youtube-generate-wavelength-video
:END:
#+findex: tlon-youtube-generate-wavelength-video

Generate a video with an animated wavelength visualization from an audio file.

The command uses =seewav= to generate the animation. You are prompted to select an audio file from the ="uqbar-audio"= repository. The output video is saved in ~paths-dir-downloads~ with a =.mp4= extension, using the original audio file's base name. The resolution is determined by ~tlon-youtube-video-resolution~ ([[#h:tlon-youtube-video-resolution][above]]).

The process runs asynchronously. Upon successful completion, the generated video is opened externally via ~tlon-browse-file~. If the process fails, the output buffer is displayed for debugging.

Requires the =seewav= command-line tool to be installed and in your =PATH=.

*** ~tlon-youtube-generate-thumbnail~
:PROPERTIES:
:CUSTOM_ID: h:tlon-youtube-generate-thumbnail
:END:
#+findex: tlon-youtube-generate-thumbnail

Generate a thumbnail image for a video using ImageMagick.

You are prompted for a video title and author(s). The command creates a thumbnail with:

- A gradient background.
- The title rendered in the center using GilliusADF-Regular font.
- The author name(s) below the title.
- The Tlon/EA logo (from the ="tlon.team-content"= repository) in the bottom-right corner.
- The URL ="altruismoeficaz.net"= in the bottom-left corner.

The thumbnail dimensions match ~tlon-youtube-video-resolution~ and the image is rendered at 300 DPI for quality, then scaled down to the target resolution. The output is saved as =thumbnail.png= in ~paths-dir-downloads~.

Requires the =magick= (ImageMagick) command-line tool and the specified font files in =~/Library/Fonts/=.

** Video upload
:PROPERTIES:
:CUSTOM_ID: h:video-upload
:END:

*** ~tlon-youtube-upload-video~
:PROPERTIES:
:CUSTOM_ID: h:tlon-youtube-upload-video
:END:
#+findex: tlon-youtube-upload-video

Upload a video file to YouTube using the YouTube Data API v3 resumable upload method.

You are prompted for:
- A video file (filtered to =.mp4= files, defaulting to ~paths-dir-downloads~).
- A title for the YouTube video.
- A description.
- A privacy setting (="private"=, ="unlisted"=, or ="public"=; defaults to ~tlon-youtube-default-privacy~).
- An optional playlist to add the video to after upload (from ~tlon-youtube-playlists~).

The upload uses a two-step process: an initialization request to obtain a resumable upload URL, followed by the actual file upload. Both steps use =curl=. If a playlist is selected, the video is automatically added to it after a successful upload.

Requires ~tlon-youtube-client-id~ and ~tlon-youtube-client-secret~ to be configured.

*** ~tlon-youtube-prepare-upload-command~
:PROPERTIES:
:CUSTOM_ID: h:tlon-youtube-prepare-upload-command
:END:
#+findex: tlon-youtube-prepare-upload-command

Prepare the upload command for debugging without executing the final upload step.

This command runs the initialization step automatically to obtain an upload URL, then displays the exact =curl= command needed for the video upload in a =*YouTube Upload Commands*= buffer. You can copy and run this command manually in a terminal for testing. It uses the same prompts as ~tlon-youtube-upload-video~.

:PROPERTIES:
:CUSTOM_ID: h:tlon-youtube-upload-thumbnail

#+findex: tlon-youtube-upload-thumbnail

Upload a thumbnail image to an existing YouTube video.

*** ~tlon-youtube-menu~
:PROPERTIES:
:CUSTOM_ID: h:tlon-youtube-menu-command
:END:
#+findex: tlon-youtube-menu

Display a =transient= menu interface for YouTube operations. The menu provides the following groups:

- *Generate*: Generate wavelength video (~tlon-youtube-generate-wavelength-video~), generate thumbnail (~tlon-youtube-generate-thumbnail~).
- *Upload*: Upload video (~tlon-youtube-upload-video~), upload thumbnail (~tlon-youtube-upload-thumbnail~), add video to playlist (~tlon-youtube-add-to-playlist~).
- *Debug*: Prepare upload command (~tlon-youtube-prepare-upload-command~).
- *Options*: Video resolution (~tlon-youtube-video-resolution~), authorize YouTube API (~tlon-youtube-authorize~).

* Authentication setup
:PROPERTIES:
:CUSTOM_ID: h:authentication-setup
:END:

To upload videos and thumbnails to YouTube, you must authorize the application to access your Google Account using OAuth 2.0. The authorization is handled by the =oauth2-auto= library.

You *must* use an OAuth 2.0 Client ID of type *Desktop app*. Web application client IDs will not work with this flow.

** Creating credentials
:PROPERTIES:
:CUSTOM_ID: h:creating-credentials
:END:

1. Go to the [[https://console.cloud.google.com/apis/credentials][Credentials page]] in the Google Cloud Console.
2. Select the project you are using for Tlon.
3. Click *+ CREATE CREDENTIALS* and select *OAuth client ID*.
4. For *Application type*, choose *Desktop app*.
5. Give it a name (e.g., "Tlon Emacs Desktop App").
6. Click *CREATE* and copy the *Client ID* and *Client Secret*.
7. Configure ~tlon-youtube-client-id~ and ~tlon-youtube-client-secret~ with these values.

** Authorization flow
:PROPERTIES:
:CUSTOM_ID: h:authorization-flow
:END:

Once credentials are configured, the first time you run a command that requires authentication (such as ~tlon-youtube-upload-video~), =oauth2-auto= automatically initiates the authorization flow:

1. Your browser opens to Google's authorization page.
2. Sign in and grant the requested permissions.
3. Google redirects your browser to a local server that =oauth2-auto= is listening on.
4. The library captures the authorization code, exchanges it for tokens, and stores them securely.

You should only need to authorize once. The system automatically refreshes tokens as they expire. If you need to re-authorize (e.g., after revoking access), use ~tlon-youtube-authorize~ ([[#h:tlon-youtube-authorize][above]]).
