This manual describes the features and customization options for the Emacs Lisp file =tlon-jobs.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

The =tlon-jobs= module manages the workflow for processing "Babel jobs," which typically involve translating documents through multiple phases: processing, translation, revision, checking, and review. It integrates Org mode's clocking mechanism, GitHub issues (via =orgit-forge=), and Git version control (via =magit=) to track each job through its lifecycle.

Each phase has an associated label, action verb, and default assignee. The module provides commands to initialize and finalize each phase, create job records, manage =jobs.org= headings, and handle section and URL correspondences.

* User options
:PROPERTIES:
:CUSTOM_ID: h:user-options
:END:

This module does not define ~defcustom~ user options. Behavior is driven by the constants described below and by user options in related modules (=tlon-forg=, =tlon-core=).

* Commands
:PROPERTIES:
:CUSTOM_ID: h:commands
:END:

** Starting and finishing job phases
:PROPERTIES:
:CUSTOM_ID: h:start-finish-phase
:END:

#+findex: tlon-jobs-start-or-finish-phase
- ~tlon-jobs-start-or-finish-phase~ :: The central command for advancing the job workflow. It determines the current context and the action associated with the currently clocked task.

  - *Initialization (in Org mode):* Clocks into the task, saves the buffer, checks the repository's label and assignee, ensures the =main= branch is checked out, pulls the latest changes from upstream, sets up windows with original and translation files, opens the associated GitHub issue, copies the original buffer content, and then calls the phase-specific initialization function (e.g., ~tlon-jobs-initialize-processing~ for the processing phase, ~tlon-jobs-initialize-translation~ for translation, etc.).

  - *Finalization (in Markdown mode):* Closes split mode if active, saves the buffer, checks the branch and label/assignee, performs phase-specific checks (e.g., prompts for Jinx/Flycheck confirmation after translation), saves the modified files, commits and pushes changes, updates the GitHub issue with the next label and assignee (closing it after the review phase), and marks relevant TODO items as done.

** Job creation
:PROPERTIES:
:CUSTOM_ID: h:job-creation
:END:

#+findex: tlon-create-record-for-job
- ~tlon-create-record-for-job~ :: Create the tracking records for a job based on a BibTeX KEY. If KEY is not provided, it is extracted from the current buffer (Markdown via ~tlon-get-key-in-buffer~ or Ebib via ~ebib--get-key-at-point~). Creates a GitHub issue titled =Job: `KEY`= and a corresponding heading in =jobs.org=.

#+findex: tlon-create-heading-for-job
- ~tlon-create-heading-for-job~ :: Add a new heading to the =jobs.org= file for tracking a specific job based on its BibTeX KEY. If KEY is not provided, it is extracted from the current buffer. The heading is formatted as =[cite:@KEY]=. If the heading does not already exist, inserts it under the main jobs section, sets the TODO state, adds the repository abbreviation as a tag, sorts the headings, and saves the buffer. If COMMIT is non-nil, commits the change.

** Section correspondences
:PROPERTIES:
:CUSTOM_ID: h:section-correspondences
:END:

#+findex: tlon-section-correspondence-dwim
- ~tlon-section-correspondence-dwim~ :: Add a new section correspondence or modify an existing one. Prompts you to select a BibTeX reference, enter values for =sectionOriginal= and =sectionSpanish=, and writes the data to =tlon-file-section-correspondences= in JSON format. Before proceeding, checks that the selected BibTeX entry is associated with the original work (not a translation).

** URL correspondences
:PROPERTIES:
:CUSTOM_ID: h:url-correspondences
:END:

#+findex: tlon-edit-url-correspondences
- ~tlon-edit-url-correspondences~ :: Add or edit a URL correspondence for a given LANGUAGE. When LANGUAGE is nil, prompts for one from ~tlon-project-languages~. Delegates to ~tlon-edit-json-mapping~.

#+findex: tlon-add-url-correspondence
- ~tlon-add-url-correspondence~ :: Add a URL correspondence mapping SOURCE-URL to TARGET-URL for LANGUAGE. When called interactively, SOURCE-URL defaults to the URL at point. LANGUAGE is inferred from the current buffer or prompted for. If the source URL is already mapped, prompts for confirmation before overwriting.

** Transient menu
:PROPERTIES:
:CUSTOM_ID: h:transient-menu
:END:

#+findex: tlon-jobs-menu
- ~tlon-jobs-menu~ :: Activate the =transient= menu interface for job-related operations.

  + *Job phases*:
    - =j= :: start or finish phase (~tlon-jobs-start-or-finish-phase~)
  + *Job creation*:
    - =1= :: add original BibTeX entry (~tlon-create-bibtex-original~)
    - =2= :: add translation BibTeX entry (~tlon-create-bibtex-translation~)
    - =3= :: move entries to db (~tlon-db-move-entry-pair~)
    - =4= :: import original document (~tlon-import-document~)
    - =5= :: translate original document (~tlon-translate-file~)
  + *Add or modify*:
    - =a s= :: section correspondence (~tlon-section-correspondence-dwim~)
    - =a u= :: URL correspondence (~tlon-edit-url-correspondences~)
    - =a U= :: add URL correspondence (~tlon-add-url-correspondence~)
  + *jobs.org*:
    - =r= :: create record (~tlon-create-record-for-job~)
    - =h= :: create heading (~tlon-create-heading-for-job~)
    - =t= :: sort headings (~tlon-sort-headings~)

* Functions
:PROPERTIES:
:CUSTOM_ID: h:functions
:END:

** Phase logic
:PROPERTIES:
:CUSTOM_ID: h:phase-logic
:END:

#+findex: tlon-jobs-get-action-in-phase
- ~tlon-jobs-get-action-in-phase~ :: Extract the action verb (e.g., =Process=, =Translate=) from a job phase string (e.g., =Awaiting processing=). Returns the second word of the phase string.

#+findex: tlon-jobs-initialize
- ~tlon-jobs-initialize~ :: Perform common setup tasks when starting any job phase: check the repository's label and assignee, ensure the =main= branch, pull from upstream, set up windows with original and translation files, open the GitHub issue, copy the original buffer content, and call the specific phase initialization function FUN.

#+findex: tlon-jobs-finalize
- ~tlon-jobs-finalize~ :: Perform common cleanup and update tasks when finishing any job phase: check the branch and label/assignee, run phase-specific checks, save and commit the translation file (and the original if processing), update the GitHub issue with the next label and assignee, and mark relevant TODO items as done.

#+findex: tlon-jobs-get-next-assignee
- ~tlon-jobs-get-next-assignee~ :: Determine the assignee for the next phase. If the current user is the designated assignee for the current phase, returns the designated assignee for the next phase. Otherwise, returns the substitute assignee for the next phase.

#+findex: tlon-jobs-act-on-issue
- ~tlon-jobs-act-on-issue~ :: Update the GitHub issue associated with ORIGINAL-KEY: find the issue by title (=Job: `KEY`=), set the specified LABEL and ASSIGNEE, and optionally close the issue if CLOSE is non-nil.

#+findex: tlon-jobs-finalize-set-todos
- ~tlon-jobs-finalize-set-todos~ :: Mark the relevant TODO items as =DONE= during finalization. Marks the clocked task itself as done. For =Review= and =Check= phases, also marks the parent TODO. For the =Review= phase, additionally marks the =jobs.org= heading as done, sorts headings, and commits.

** Phase-specific initialization
:PROPERTIES:
:CUSTOM_ID: h:phase-initialization
:END:

#+findex: tlon-jobs-initialize-processing
- ~tlon-jobs-initialize-processing~ :: Setup for the =Process= phase: open the original file and the relevant section in =manual.org= (using ~tlon-jobs-manual-processing-id~), narrow the view, and open the issue.

#+findex: tlon-jobs-initialize-translation
- ~tlon-jobs-initialize-translation~ :: Setup for the =Translate= phase: open the DeepL application using ~macos-open-app~.

#+findex: tlon-jobs-initialize-revision
- ~tlon-jobs-initialize-revision~ :: Setup for the =Revise= phase: activate split mode (~tlon-split-mode~).

#+findex: tlon-jobs-initialize-check
- ~tlon-jobs-initialize-check~ :: Setup for the =Check= phase: rearrange windows, start Markdown preview, and initiate text-to-speech using ~read-aloud-buf~.

#+findex: tlon-jobs-initialize-review
- ~tlon-jobs-initialize-review~ :: Setup for the =Review= phase: open an ediff session showing changes since the last user commit, set up =jinx= for spell/grammar checking in the target language, and move to the beginning of the buffer.

** jobs.org helpers
:PROPERTIES:
:CUSTOM_ID: h:jobs-org-helpers
:END:

#+findex: tlon-jobs-get-file
- ~tlon-jobs-get-file~ :: Retrieve the path to the =jobs.org= file, using the Org ID ~tlon-jobs-id~ and caching the result in ~tlon-jobs-file~ via ~tlon-get-or-set-org-var~.

#+findex: tlon-jobs-get-key-in-heading
- ~tlon-jobs-get-key-in-heading~ :: Extract the BibTeX key from the Org heading at point, matching either the =[cite:@KEY]= or =Job: `KEY`.md`= format.

#+findex: tlon-jobs-goto-heading
- ~tlon-jobs-goto-heading~ :: Move point to the heading corresponding to a given BibTeX KEY within the =jobs.org= file.

** URL correspondence helpers
:PROPERTIES:
:CUSTOM_ID: h:url-correspondence-helpers
:END:

#+findex: tlon-highlight-url-correspondences
- ~tlon-highlight-url-correspondences~ :: Highlight source URLs from the URL correspondences file for LANGUAGE in the current buffer. When LANGUAGE is nil, prompts for one. Strips protocol prefixes and applies a green background highlight to matching text.

#+findex: tlon-section-correspondence-check
- ~tlon-section-correspondence-check~ :: Check that the selected BibTeX KEY is associated with the original work rather than a translation. If the entry's =langid= is not English, warns and prompts for confirmation.

* Constants and variables
:PROPERTIES:
:CUSTOM_ID: h:constants
:END:

#+vindex: tlon-job-labels
- ~tlon-job-labels~ :: List of plists defining the phases a job goes through. Each plist contains =:label= (the display label, e.g., =Awaiting processing=), =:action= (the action verb, e.g., =Process=), and =:assignee= (the default GitHub username for that phase). This structure drives the workflow logic, determining the next step and assignee after a phase is completed.

#+vindex: tlon-jobs-manual-processing-id
- ~tlon-jobs-manual-processing-id~ :: The Org ID of the =processing= heading in =manual.org=, used by ~tlon-jobs-initialize-processing~.

#+vindex: tlon-jobs-id
- ~tlon-jobs-id~ :: The Org ID of the main =jobs= heading in =jobs.org=. This file serves as the primary tracker for all jobs.

#+vindex: tlon-jobs-file
- ~tlon-jobs-file~ :: Variable holding the path to the =jobs.org= file. Set dynamically by ~tlon-jobs-get-file~; do not set manually.
