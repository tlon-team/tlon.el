This manual describes the features and customization options for the Emacs Lisp file =tlon-yaml.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

The =tlon-yaml= module handles every aspect of YAML front-matter management
for content files in the Tlon project.  It can parse YAML into association
lists, retrieve individual field values, insert or delete fields, reorder
metadata according to canonical key lists, and perform batch operations
across entire directories.  The module also integrates with AI backends to
suggest tags for articles and guess English counterpart filenames for
translated tag/author files.  Finally, it provides a transient menu
(~tlon-yaml-menu~) that exposes the most common operations.

*** ~tlon-yaml-insert-field~
:PROPERTIES:
:CUSTOM_ID: h:tlon-yaml-insert-field
:END:
#+findex: tlon-yaml-insert-field

Insert a YAML field with KEY and VALUE in the metadata section of the
current buffer.  If KEY or VALUE are nil, you are prompted to select from
a list of suitable candidates (the valid keys for the file's type).  When
KEY is ="html_title"= and the field is not already present, the default
suggestion depends on the file type: for articles, it is the BibTeX title
of the entry referenced by the YAML =key= field; for authors and tags, it
is the value of the YAML =title= field.

If the field already exists, you are asked whether to overwrite it.  When
OVERWRITE is non-nil, the existing field is replaced without prompting.

The field is inserted in the canonical order defined by the key list for
the file type.  Special handling applies to the ="meta"= key, which is
inserted as a YAML block scalar just before the closing delimiter.

*** ~tlon-yaml-insert-original-path~
:PROPERTIES:
:CUSTOM_ID: h:tlon-yaml-insert-original-path
:END:
#+findex: tlon-yaml-insert-original-path

Insert the =original_path= YAML field.  This is a convenience wrapper
around ~tlon-yaml-insert-field~ that passes ="original_path"= as the key,
prompting you for the value from a list of filenames in the counterpart
directory.

*** ~tlon-yaml-copy-key~
:PROPERTIES:
:CUSTOM_ID: h:tlon-yaml-copy-key
:END:
#+findex: tlon-yaml-copy-key

Copy the YAML =key= field value to the kill ring.  When in a Markdown
buffer, operate on its file.  When point is on a Markdown file in Dired,
operate on that file.  Signal an error when the file is not Markdown or
the =key= field is missing.  Return the copied value.

*** ~tlon-yaml-set-publication-status~
:PROPERTIES:
:CUSTOM_ID: h:tlon-yaml-set-publication-status
:END:
#+findex: tlon-yaml-set-publication-status

Set the YAML =publication_status= of the Markdown file at point to
STATUS.  If STATUS is nil, you are prompted for a status from
~tlon-db-publication-statuses~, defaulting to the current value.  Signal
an error for non-Markdown files.  When the file is an article, signal an
error suggesting you use ~tlon-db-set-publication-status~ instead, because
article publication status is managed through the database.

*** ~tlon-yaml-normalize-tags-field~
:PROPERTIES:
:CUSTOM_ID: h:tlon-yaml-normalize-tags-field
:END:
#+findex: tlon-yaml-normalize-tags-field

Convert a multiline =tags= YAML field (one where the list spans multiple
lines) into a single-line bracketed format.  When a conversion occurs,
return non-nil.  This is useful after manual editing that may have
introduced unwanted line breaks inside the tags list.

** Tag management
:PROPERTIES:
:CUSTOM_ID: h:tag-management
:END:

*** ~tlon-yaml-suggest-tags~
:PROPERTIES:
:CUSTOM_ID: h:tlon-yaml-suggest-tags
:END:
#+findex: tlon-yaml-suggest-tags

Suggest tags for an article file using AI.  When called interactively,
default to the current article or prompt for one inside the
=uqbar-en/articles= directory.  The command collects all candidate tag
files (excluding those under an =excluded/= subdirectory), builds a prompt
containing the full article Markdown and the candidate tag titles with
their first paragraphs, and sends it to the AI model specified by
~tlon-yaml-suggest-tags-model~.  The AI returns a JSON array of tag titles
ordered by relevance (at most six).  The callback normalises casing
against the canonical tag titles and inserts the result into the =tags=
YAML field of the article file.

*** ~tlon-yaml-suggest-tags-in-dir~
:PROPERTIES:
:CUSTOM_ID: h:tlon-yaml-suggest-tags-in-dir
:END:
#+findex: tlon-yaml-suggest-tags-in-dir

Suggest tags via AI for article files in DIR that lack a =tags= field.
When N is non-nil (prefix argument), only process the first N articles
missing tags.  When DIR is nil, use =default-directory=.  Each file is
processed by calling ~tlon-yaml-suggest-tags~.

*** ~tlon-yaml-insert-translated-tags~
:PROPERTIES:
:CUSTOM_ID: h:tlon-yaml-insert-translated-tags
:END:
#+findex: tlon-yaml-insert-translated-tags

Insert translated tag titles into FILE's YAML metadata.  The function
looks at the English counterpart of FILE, reads its =tags= field, maps
every tag slug to its tag file in English, locates the tag translation
into the language of FILE, retrieves the translated tag's =title=, and
writes the list of titles back to FILE as the value of the =tags= field.

When the target-language tag file cannot be found, the command falls back
to the glossary: it looks up the English tag title and uses its
translation into the target language.  It also reports which tags were
translated via the glossary.

*** ~tlon-yaml-insert-translated-tags-in-dir~
:PROPERTIES:
:CUSTOM_ID: h:tlon-yaml-insert-translated-tags-in-dir
:END:
#+findex: tlon-yaml-insert-translated-tags-in-dir

Walk the current directory (non-recursively) and call
~tlon-yaml-insert-translated-tags~ for each article file that lacks a
=tags= field.  With a numeric prefix argument, only process the first N
such files.

*** ~tlon-yaml-guess-english-counterpart~
:PROPERTIES:
:CUSTOM_ID: h:tlon-yaml-guess-english-counterpart
:END:
#+findex: tlon-yaml-guess-english-counterpart

Use an AI model to guess the English counterpart filename for the tag or
author at point.  The command sends the current YAML =title= and the list
of candidate filenames from the English repository's tags or authors
folder, expects the model to return the exact matching filename, and
inserts it into the YAML under =original_path=.

If =original_path= already exists, the command asks for confirmation when
called interactively and does nothing when called programmatically.

*** ~tlon-yaml-guess-english-counterpart-in-dir~
:PROPERTIES:
:CUSTOM_ID: h:tlon-yaml-guess-english-counterpart-in-dir
:END:
#+findex: tlon-yaml-guess-english-counterpart-in-dir

Walk the current directory (non-recursively) and call
~tlon-yaml-guess-english-counterpart~ for each tag or author file.  With a
numeric argument, only process the first N files that do not already have
an =original_path= field.

** Transient menu
:PROPERTIES:
:CUSTOM_ID: h:tlon-yaml-menu
:END:
#+findex: tlon-yaml-menu

The command ~tlon-yaml-menu~ activates a =transient= menu interface for
YAML operations.  It provides the following groups:

- *General*:
  - =i= :: Insert or edit field (~tlon-yaml-insert-field~)
  - =m= :: List files missing field in dir (~tlon-yaml-list-files-missing-field-in-dir~)
  - =s= :: Set publication status (~tlon-yaml-set-publication-status~)
  - =k= :: Copy key (~tlon-yaml-copy-key~)
- *Tags*:
  - =t s= :: Suggest tags (~tlon-yaml-suggest-tags~)
  - =t S= :: Suggest tags in dir (~tlon-yaml-suggest-tags-in-dir~)
  - =t t= :: Insert translated tags (~tlon-yaml-insert-translated-tags~)
  - =t T= :: Insert translated tags in dir (~tlon-yaml-insert-translated-tags-in-dir~)
  - =t g= :: Guess English counterpart (~tlon-yaml-guess-english-counterpart~)
  - =t G= :: Guess counterparts in dir (~tlon-yaml-guess-english-counterpart-in-dir~)
- *AI models*:
  - =-t= :: Suggest tags model
  - =-g= :: Guess counterpart model

*** ~tlon-get-bibtex-key~
:PROPERTIES:
:CUSTOM_ID: h:tlon-get-bibtex-key
:END:
#+findex: tlon-get-bibtex-key

Get the BibTeX key of the selected work.  INITIAL-INPUT is used as the
initial input for the =citar= completion.

*** ~tlon-yaml-get-bibtex-key-values~
:PROPERTIES:
:CUSTOM_ID: h:tlon-yaml-get-bibtex-key-values
:END:
#+findex: tlon-yaml-get-bibtex-key-values

Return the BibTeX key of the work at point appropriate for FIELD.  If
FIELD is =original_key=, return the BibTeX key of the original work.  If
FIELD is =translation_key=, return the BibTeX key of the translation.

*** ~tlon-yaml-select-key-values~
:PROPERTIES:
:CUSTOM_ID: h:tlon-yaml-select-key-values
:END:
#+findex: tlon-yaml-select-key-values

Select a value for a YAML field with KEY from a list of VALUES.  The
prompt and input method vary depending on the key type (list-valued keys
use =completing-read=, string-valued keys use =read-string=, BibTeX keys
use ~tlon-yaml-get-bibtex-key-values~).
