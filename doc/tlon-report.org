This manual describes the features and customization options for the Emacs Lisp file =tlon-report.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

The =tlon-report= module provides functions for creating, submitting, and generating reports from Org mode clock files, tailored for the Tlon team's time tracking workflow. It assumes each user has a dedicated clock repository managed with Git (named =clock-NICKNAME= under =paths-dir-tlon-repos=).

The workflow is: create daily clock entry files from your Org agenda data, submit them to your clock repository via Git, and generate aggregated reports across date ranges.

* User options
:PROPERTIES:
:CUSTOM_ID: h:user-options
:END:

This module does not define ~defcustom~ user options. Behavior is driven by the user's Org agenda configuration and the Tlon user/repository lookup system in =tlon-core=.

** Creating clock entries
:PROPERTIES:
:CUSTOM_ID: h:creating-entries
:END:

#+findex: tlon-clock-entry-create
- ~tlon-clock-entry-create~ :: Create a clock entry file for a specific DATE in your clock repository.

  If DATE is nil, you are prompted for one, defaulting to today. The file is named =YYYY-MM-DD.org= in the directory returned by ~tlon-clock-get-repo~. If the file already exists and contains data, you are asked whether to overwrite it.

  The command inserts a =#+TITLE:= line with the date and an Org clock report (via =org-extras-clock-report-insert=) covering that single day from your agenda files.

  The SUBMIT argument controls post-creation behavior:
  - =nil= (default when interactive) :: prompt you whether to submit the entry.
  - =never= :: do not submit.
  - Any other non-nil value :: submit without prompting.

#+findex: tlon-clock-entry-create-in-range
- ~tlon-clock-entry-create-in-range~ :: Create clock entry files for all dates between START and END (inclusive).

  Interactively prompts for both dates using the Org date selector. Iterates through each date and calls ~tlon-clock-entry-create~ with =submit= set to =never=. After creating all files, prompts whether to submit all entries at once via ~tlon-clock-entry-submit-all~.

** Submitting clock entries
:PROPERTIES:
:CUSTOM_ID: h:submitting-entries
:END:

#+findex: tlon-clock-entry-submit
- ~tlon-clock-entry-submit~ :: Submit a specific clock entry FILE to your clock Git repository.

  If FILE is nil, you are prompted for one, defaulting to today's entry. The command ensures the file exists, tracks it with =magit-extras-track-file= if untracked, then stages, commits (with a message like =Add clock entry: YYYY-MM-DD.org=), and pushes the changes via =magit-extras-stage-commit-and-push=. Set NO-PUSH to non-nil to suppress the push.

#+findex: tlon-clock-entry-submit-all
- ~tlon-clock-entry-submit-all~ :: Submit all unstaged and untracked =.org= files in your clock repository.

  Retrieves the lists of unstaged and untracked files, calls ~tlon-clock-entry-submit~ for each (suppressing individual pushes), and performs a single =magit-push-current-to-pushremote= at the end to push all committed changes.

** Generating reports
:PROPERTIES:
:CUSTOM_ID: h:generating-reports
:END:

#+findex: tlon-clock-report-create
- ~tlon-clock-report-create~ :: Generate an aggregated clock report for a date range (START-DATE to END-DATE, both in =YYYY-MM-DD= format).

  Collects all relevant =.org= files within the date range from your clock repository using ~tlon-clock-collect-clock-files~. Signals an error if no files are found.

  For each file, searches for =#+BEGIN: clocktable= blocks and parses them with ~tlon-clock-parse-clocktable-block~, aggregating time per headline in a hash table.

  Produces a report in a buffer named =*Aggregated Clock Report*= containing:
  - An Org table with one row per activity and its total time (=H:MM= format), sorted alphabetically.
  - A footer row with the grand total.
  - Summary statistics: total days, total hours, average hours per day, average hours per week, and average hours per workday (assuming a 5-day work week).

  The report buffer is set to =org-mode= and displayed.

#+findex: tlon-clock-get-repo
- ~tlon-clock-get-repo~ :: Return the absolute path to your clock repository. Constructs the name as =clock-= followed by your nickname (obtained via ~tlon-user-lookup~ based on =user-full-name=) and concatenates it with =paths-dir-tlon-repos=.

#+findex: tlon-clock-get-file-for-date

- ~tlon-clock-collect-clock-files~ :: Return a list of =.org= files in your clock repository whose filenames match the =YYYY-MM-DD.org= pattern and fall within the START-DATE to END-DATE range (inclusive).

** Parsing and formatting helpers
:PROPERTIES:
:CUSTOM_ID: h:parsing-helpers
:END:

#+findex: tlon-clock-parse-clocktable-block
- ~tlon-clock-parse-clocktable-block~ :: Parse the text content of an Org clocktable block. Processes table rows, identifies detail rows (empty first column, not containing =Total time= or =File time=), extracts headline names and time strings, converts time to minutes, and aggregates into the provided ACTIVITIES hash table.

#+findex: tlon-clock-time-string-to-minutes
- ~tlon-clock-time-string-to-minutes~ :: Convert a time string in =H:MM= format to the total number of minutes. Returns =0= if the string does not match the expected format.

#+findex: tlon-clock-minutes-to-time-string
- ~tlon-clock-minutes-to-time-string~ :: Convert a total number of MINUTES to a formatted time string =H:MM= (e.g., 90 becomes =1:30=).

#+findex: tlon-clock-insert-summary-stats
- ~tlon-clock-insert-summary-stats~ :: Calculate and insert summary statistics into the current buffer. Computes total hours from TOTAL minutes, determines the number of days in the range (inclusive), and inserts formatted lines for total days, total hours, average hours per day, average hours per week (7 days), and average hours per workday (5 days).
