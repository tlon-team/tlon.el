This manual describes the features and customization options for the Emacs Lisp file =tlon-forg.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

The =tlon-forg= module provides bidirectional integration between Forge (the Emacs interface to GitHub) and Org mode. It allows you to capture GitHub issues as Org TODO items, keep their state synchronized, navigate between an issue and its corresponding TODO, and manage related metadata like assignees, labels, status, tags, and effort estimates.

TODO items are stored in a main generic file, a jobs-specific file, or in repository-specific Org files. Status, tags, and estimates are kept in sync with GitHub Project fields. The module relies on =forge-extras= for GitHub Project (V2) interactions.

* User options
:PROPERTIES:
:CUSTOM_ID: h:user-options
:END:

#+vindex: tlon-when-assignee-is-nil
- ~tlon-when-assignee-is-nil~ :: Controls what to do when capturing an issue that has no assignee.

  - =prompt= :: Prompt you to confirm that the issue will be assigned to you.
  - =change= :: Assign the issue to you without prompting.
  - =warn= :: Capture the issue as-is, logging a warning in the =*Messages*= buffer.
  - =capture= :: Capture the issue as-is, without logging a warning.
  - =no-capture= (or any other value) :: Do not capture the issue. This is the default.

  You can also set this temporarily via ~tlon-forg-menu~; the value will persist only for the current session.

#+vindex: tlon-when-assignee-is-someone-else
- ~tlon-when-assignee-is-someone-else~ :: Controls what to do when capturing an issue assigned to someone other than you. The options are the same as ~tlon-when-assignee-is-nil~. Defaults to =no-capture=.

  Note that this option has no effect when tasks are captured via ~tlon-capture-all-issues-in-project~ ([[#h:capture-all-issues-in-project][capture all issues in project]]). That command always treats this variable as =no-capture=. You can set this temporarily via ~tlon-forg-menu~.

#+vindex: tlon-forg-when-syncing
- ~tlon-forg-when-syncing~ :: Controls what to do when ~tlon-sync-issue-and-todo~ ([[#h:sync-issue-and-todo][sync issue and todo]]) finds discrepancies between the issue and its TODO.

  - =prompt= (default) :: Ask you whether to keep the issue value or the TODO value.
  - =issue= :: Automatically update the TODO to match the issue.
  - =todo= :: Automatically update the issue to match the TODO.

  You can set this temporarily via ~tlon-forg-menu~.

#+vindex: tlon-forg-include-archived
- ~tlon-forg-include-archived~ :: When non-nil, include archived issues and TODOs in capture and sync operations. Defaults to =nil=. You can toggle this temporarily via ~tlon-forg-menu~.

#+vindex: tlon-forg-archive-todo-on-close
- ~tlon-forg-archive-todo-on-close~ :: When non-nil, ~tlon-close-issue-and-todo~ ([[#h:close-issue-and-todo][close issue and todo]]) will archive the corresponding Org TODO subtree after marking it as =DONE=. Defaults to =nil=. You can toggle this temporarily via ~tlon-forg-menu~.

#+vindex: tlon-forg-sort-after-sync-or-capture
- ~tlon-forg-sort-after-sync-or-capture~ :: When non-nil, bulk capture and sync operations on a project will sort each TODO buffer by status and project order after completing their operations. Defaults to =nil=. You can toggle this temporarily via ~tlon-forg-menu~.

#+vindex: tlon-forg-enforce-user
- ~tlon-forg-enforce-user~ :: If non-nil, this string is treated as the current user's GitHub login. Intended for testing purposes. Defaults to =nil=. Type: string.

#+vindex: tlon-forg-sync-after-capture-project
- ~tlon-forg-sync-after-capture-project~ :: Controls whether to automatically sync all issues after capturing them in a project.

  - =t= :: Always sync after capturing all issues in a project.
  - =nil= :: Never sync after capturing.
  - =prompt= (default) :: Prompt you whether to sync after capturing.

  The sync operation reuses the cached project item list when available. You can set this temporarily via ~tlon-forg-menu~.

** Capturing issues as TODOs
:PROPERTIES:
:CUSTOM_ID: h:capturing-issues
:END:

#+findex: tlon-capture-issue
- ~tlon-capture-issue~ :: Create a new Org TODO based on a GitHub issue.

  If called interactively from a Forge buffer, the issue at point is used. If called from an Org buffer, you are prompted to select a repository and then an issue. The command checks that the issue is open, handles the assignee according to ~tlon-when-assignee-is-nil~ and ~tlon-when-assignee-is-someone-else~, and then captures the TODO.

  Job issues (titles starting with =Job: =) are captured with phase handling and refiled under a master job heading in the jobs file. Generic issues are captured to the repository-specific Org file (=REPO-NAME.org= in =paths-dir-tlon-todos=) if it exists, or to the main generic TODOs file otherwise.

  If a TODO for the issue already exists, the command returns immediately. Accepts optional arguments ISSUE, INVOKED-FROM-ORG-FILE, and OVERRIDE-STATUS.

#+findex: tlon-capture-all-issues-in-repo
- ~tlon-capture-all-issues-in-repo~ :: Capture all open issues in a selected repository that do not already have a corresponding TODO and are not assigned to another user.

  If a repository cannot be inferred from the current context, you are prompted to select one. Before capturing, the command performs a full pull from Forge to ensure local data is up to date (skip with a prefix argument). If any TODOs are captured, the target Org file is opened and point is moved to the last captured TODO. Clears the =org-refile= cache upon completion.

#+findex: tlon-capture-all-issues-in-project
- ~tlon-capture-all-issues-in-project~ :: Capture all open issues in the GitHub Project configured in =forge-extras.el= that do not already have a corresponding TODO.

  Fetches all items from the configured GitHub Project, groups them by repository, pulls from Forge for each repository (skip with a prefix argument), and captures issues. After all repositories are processed, optionally runs ~tlon-sync-all-issues-in-project~ based on ~tlon-forg-sync-after-capture-project~ ([[#h:user-options][user options]]). Restores the original window configuration and clears the =org-refile= cache upon completion.

#+findex: tlon-capture-all-issues-in-repo-after-pull-by-name
- ~tlon-capture-all-issues-in-repo-after-pull-by-name~ :: Run ~tlon-capture-all-issues-in-repo-after-pull~ for a repository specified by name. Interactively, you are prompted to choose from Tlon-configured repository names. This command can be useful for debugging since, unlike ~tlon-capture-all-issues-in-repo~, it uses =gh= rather than =forge= to get the issues.

** Syncing issues and TODOs
:PROPERTIES:
:CUSTOM_ID: h:syncing-issues
:END:

#+findex: tlon-sync-issue-and-todo
- ~tlon-sync-issue-and-todo~ :: Synchronize the state between the current issue and its associated Org TODO, or vice versa. Works whether point is in the issue buffer or on the TODO heading.

  For title, status, and tags, the command compares the issue and TODO values element by element. When they differ, the action taken is governed by ~tlon-forg-when-syncing~ ([[#h:user-options][user options]]): you may be prompted, or the issue or TODO may be updated automatically.

  For estimates (GitHub Project =Estimate= field vs. Org =Effort= property): if only one side has a value, the other is updated silently; if both have values and they differ, the behavior follows ~tlon-forg-when-syncing~.

#+findex: tlon-sync-all-issues-in-repo
- ~tlon-sync-all-issues-in-repo~ :: Sync all Org TODOs with their corresponding GitHub issues in a selected repository.

  If a repository cannot be inferred from context, you are prompted to select one. Performs a full pull from Forge first (skip with a prefix argument). Iterates through all headings in all detected TODO files (generic, jobs, and repository-specific), finds the associated issue, and syncs title, status, tags, and estimate. Project item data is fetched in batch per repository for efficiency. Clears the =org-refile= cache upon completion.

#+findex: tlon-sync-all-issues-in-project
- ~tlon-sync-all-issues-in-project~ :: Sync all Org TODOs with their corresponding GitHub issues across the entire GitHub Project.

  Fetches all items from the configured GitHub Project once, groups them by repository, pulls from Forge for each repository (skip with a prefix argument), and syncs each issue. Status and estimate data from the project is passed to individual sync calls to avoid redundant API calls. Restores window configuration and clears the =org-refile= cache upon completion.

** Creating issues and TODOs
:PROPERTIES:
:CUSTOM_ID: h:creating-issues
:END:

#+findex: tlon-create-new-issue
- ~tlon-create-new-issue~ :: Create a GitHub issue from scratch, then capture it as an Org TODO.

  Prompts for a repository, title, status (Org TODO keyword), optional effort estimate in hours, and assignee. Creates the issue on GitHub, labels it with the chosen status, sets the project estimate when given, captures the issue as an Org TODO, and leaves point on the resulting heading. Silently adds the issue to the project and updates its project status.

#+findex: tlon-create-issue-from-todo
- ~tlon-create-issue-from-todo~ :: Create a new GitHub issue based on the Org heading at point. Enable verbose logging with a prefix argument.

  The command ensures the heading does not already link to an issue, determines the repository (from the buffer filename, heading tag, or by prompting), extracts the title, status, tags, and effort from the heading, creates the issue on GitHub, sets the assignee, labels, and project status, and replaces the heading text with an =orgit-topic= link to the new issue.

#+findex: tlon-create-issue-or-todo
- ~tlon-create-issue-or-todo~ :: Create the counterpart if it does not exist. If on an Org heading without an issue, call ~tlon-create-issue-from-todo~. If in an issue buffer without a TODO, call ~tlon-capture-issue~.

** Visiting issues and TODOs
:PROPERTIES:
:CUSTOM_ID: h:visiting-issues
:END:

#+findex: tlon-visit-issue
- ~tlon-visit-issue~ :: Visit the GitHub issue associated with the current context. If point is on an =orgit-topic:= link, follow it. Otherwise, extract the issue number and repository from the current Org heading.

#+findex: tlon-visit-todo
- ~tlon-visit-todo~ :: Visit an Org TODO item. If called with a POS-FILE argument (a cons cell of =POSITION . FILE-PATH=), navigate directly. Otherwise, use the current issue to find the associated TODO via ~tlon-get-todo-position-from-issue~. Signals an error if no TODO is found.

#+findex: tlon-visit-counterpart
- ~tlon-visit-counterpart~ :: Navigate between an issue and its corresponding TODO. If in an issue buffer, visit the TODO. If on a TODO heading, visit the issue.

#+findex: tlon-visit-counterpart-or-capture
- ~tlon-visit-counterpart-or-capture~ :: Like ~tlon-visit-counterpart~, but if navigating from an issue to a non-existent TODO, capture the issue first.

** Closing issues and TODOs
:PROPERTIES:
:CUSTOM_ID: h:close-issue-and-todo
:END:

#+findex: tlon-close-issue-and-todo
- ~tlon-close-issue-and-todo~ :: Close both the GitHub issue and its associated Org TODO. Works from either context (issue buffer or TODO heading).

  Finds the issue, closes it on GitHub, visits the corresponding TODO, sets its state to =DONE=, optionally archives the subtree (if ~tlon-forg-archive-todo-on-close~ is non-nil), and saves the Org buffer.

** Sorting org entries
:PROPERTIES:
:CUSTOM_ID: h:sorting
:END:

#+findex: tlon-forg-sort-by-tag
- ~tlon-forg-sort-by-tag~ :: Sort Org entries in the current buffer based on the first tag that matches the pattern defined in ~tlon-forg-sort-by-tags-regexp~. Entries without a matching tag are sorted to the end. Uses ~org-sort-entries~ with a custom sorter.

#+findex: tlon-forg-sort-by-status-and-project-order
- ~tlon-forg-sort-by-status-and-project-order~ :: Sort Org entries first by their TODO status and then by their order in the GitHub Project.

  Status order is based on the sequence in ~tlon-todo-statuses~ (=DOING=, =NEXT=, =LATER=, =SOMEDAY=, etc.), followed by other active Org TODO states, then =DONE=, and finally items with no status. Project order is determined by ~forge-extras-list-project-items-ordered~. With a prefix argument, use the cached project items list instead of fetching a fresh one.

- ~tlon-forg-menu~ :: Activate the =transient= menu interface for =tlon-forg= operations.

  + *Actions*:
    - =n= :: new issue (~tlon-create-new-issue~)
    - =p= :: post (create issue from todo) (~tlon-create-issue-from-todo~)

    - =c c= :: capture issue (~tlon-capture-issue~)

    - =s s= :: sync issue (~tlon-sync-issue-and-todo~)

  + *Options*:
    - =-d= :: when issue and todo differ (~tlon-forg-when-syncing-infix~)
    - =-n= :: when assignee is nil (~tlon-when-assignee-is-nil-infix~)

    - =-s= :: sync after project capture (~tlon-forg-sync-after-capture-project-infix~)
