This manual describes the features and customization options for the Emacs Lisp file =tlon-newsletter.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=tlon-newsletter= provides functionality for creating and publishing newsletter issues for the Tlon "boletin" (Spanish-language newsletter). It uses AI to transform a list of URLs and notes into a structured Markdown newsletter draft, leveraging web browsing tools (DuckDuckGo search, content fetching) and Slack integration to gather information. The module also assists in publishing finished issues to Substack.

** ~tlon-newsletter-model~
:PROPERTIES:
:CUSTOM_ID: h:tlon-newsletter-model
:END:
#+vindex: tlon-newsletter-model

Model to use for creating newsletter issues.

The default value is =("Gemini" . gemini-pro-latest)=. The type is a cons cell whose car is the backend name (a string) and whose cdr is the model symbol. See ~gptel-extras-ai-models~ for available options.

When set, this model overrides the default =gptel= model for newsletter creation. Set to =nil= to use the default model. You can also change this value via the transient menu ([[#h:tlon-newsletter-menu-command][~tlon-newsletter-menu~]]).

#+begin_src emacs-lisp
(setq tlon-newsletter-model nil) ; Use default gptel model
#+end_src

*** ~tlon-newsletter-create-issue~
:PROPERTIES:
:CUSTOM_ID: h:tlon-newsletter-create-issue
:END:
#+findex: tlon-newsletter-create-issue

Create a draft for a new newsletter issue using AI.

This command automates the drafting of a new newsletter issue. It works as follows:

1. Locates the ="boletin"= repository (as configured in ~tlon-repos~) and targets the next month's issue file (named =YYYY-MM.md=) within the =numeros/= subdirectory. If the file does not exist, it is created.
2. Reads the content of this file. The file is expected to contain a list of items (URLs or plain text), one per line.
3. If the file is empty, you are prompted for confirmation before proceeding.
4. Validates that the prompt file ([[#h:tlon-newsletter-prompt-file][~tlon-newsletter-prompt-file~]]) and sample issue file ([[#h:tlon-newsletter-sample-issue-file][~tlon-newsletter-sample-issue-file~]]) exist.
5. Sends the content to the AI with a prompt that instructs it to:
   - Process each line: if a URL, fetch its content and write a one-paragraph summary in Spanish; if plain text, summarize it directly.
   - Structure the output as a Spanish newsletter in Markdown with sections such as =## Eventos=, =## Anuncios=, =## Articulos=, and =## Oportunidades=.
   - Use the sample issue for formatting guidance.
6. The AI has access to DuckDuckGo search and content fetching tools, as well as Slack channel history for the =ae-racionalidad= workspace.
7. The resulting draft overwrites the original input file, which is then opened for review.

The model used for the request is controlled by ~tlon-newsletter-model~.

*** ~tlon-newsletter-publish-issue~
:PROPERTIES:
:CUSTOM_ID: h:tlon-newsletter-publish-issue
:END:
#+findex: tlon-newsletter-publish-issue

Assist in publishing the current newsletter issue. This command performs several steps to streamline the publishing workflow:

1. Copies the issue title (e.g., ="Boletin de noviembre de 2025"=) to the kill ring so you can paste it into Substack.
2. Opens the Substack publish page in your browser.
3. Opens the current issue's Markdown file and runs ~markdown-preview~ to generate an HTML preview.
4. After a short delay, opens the EWW preview buffer in your external browser so you can copy the formatted content.

** Date handling
:PROPERTIES:
:CUSTOM_ID: h:date-handling
:END:

*** ~tlon-newsletter-get-issue-month~
:PROPERTIES:
:CUSTOM_ID: h:tlon-newsletter-get-issue-month
:END:
#+findex: tlon-newsletter-get-issue-month

Return the lowercase Spanish month name for a date string DATE in ="yyyy-mm"= format. For example, ="2025-08"= returns ="agosto"=. Uses ~format-time-string~ with the =es_ES.UTF-8= locale.

*** ~tlon-newsletter-get-content-year~
:PROPERTIES:
:CUSTOM_ID: h:tlon-newsletter-get-content-year
:END:
#+findex: tlon-newsletter-get-content-year

Return the year of the content period from a date string DATE in ="yyyy-mm"= format. Since the newsletter is published on the first day of the month, the content period is the previous month. For example, if DATE is ="2025-01"=, the content period is December 2024, so this function returns =2024=.

*** ~tlon-newsletter-get-previous-month~
:PROPERTIES:
:CUSTOM_ID: h:tlon-newsletter-get-previous-month
:END:
#+findex: tlon-newsletter-get-previous-month

Return the Spanish name of the month preceding MONTH-NAME. For example, given ="marzo"=, this returns ="febrero"=.

*** ~tlon-newsletter--next-year-month~
:PROPERTIES:
:CUSTOM_ID: h:tlon-newsletter--next-year-month
:END:
#+findex: tlon-newsletter--next-year-month

Return a cons cell =(YEAR . MONTH)= for the next calendar month. If TIME is non-nil, compute relative to TIME; otherwise use the current time. For example, in November 2025 this returns =(2025 . 12)=; in December 2025 it returns =(2026 . 1)=.

:PROPERTIES:
:CUSTOM_ID: h:tlon-newsletter-repo-dir
:END:
#+vindex: tlon-newsletter-repo-dir

Path to the local clone of the ="boletin"= newsletter repository. This constant is derived from ~tlon-repo-lookup~.

** ~tlon-newsletter-numeros-subdir~
:PROPERTIES:
:CUSTOM_ID: h:tlon-newsletter-numeros-subdir
:END:
#+vindex: tlon-newsletter-numeros-subdir

Path to the =numeros/= subdirectory within the newsletter repository. This is where individual issue files (named =YYYY-MM.md=) are stored.

** ~tlon-newsletter-prompt-file~
:PROPERTIES:
:CUSTOM_ID: h:tlon-newsletter-prompt-file
:END:
#+vindex: tlon-newsletter-prompt-file

Path to the prompt file used for creating newsletter issues. Located in the =etc/= subdirectory of the =tlon= package directory. This file contains the AI prompt template with format specifiers for the content month, issue month, timestamp, sample issue, and input content.

:PROPERTIES:
:CUSTOM_ID: h:tlon-newsletter-sample-issue-file
:END:
#+vindex: tlon-newsletter-sample-issue-file

Path to a sample newsletter issue file used as a formatting reference for the AI. Currently points to =2025-08.md= in the =numeros/= subdirectory.
