This manual describes the features and customization options for the Emacs Lisp file =tlon-bib.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

The =tlon-bib= module provides a comprehensive suite of tools for working with BibTeX bibliographic data. It covers abstract fetching and management (from CrossRef, Google Books, Zotra, and AI), batch abstract processing, entry movement between fluid and stable BibTeX files, field manipulation (database tags, language identifiers, URL population, journal title fields), citation key generation, citation conversion and replacement (including AI-powered workflows), abstract translation via DeepL, reporting on entry quality, and entry creation from Markdown YAML front matter.

The module integrates with several external packages including =ebib=, =citar=, =zotra-extras=, =gptel=, and =tlon-ai= to provide a seamless bibliography management experience. Most commands work in both =bibtex-mode= and Ebib modes, dispatching to the appropriate field accessor functions automatically.

You can access all the major commands through the transient menu ~tlon-bib-menu~ ([[#h:transient-menu][Transient menu]]).

* User options
:PROPERTIES:
:CUSTOM_ID: h:user-options
:END:

** Abstract overwrite behavior
:PROPERTIES:
:CUSTOM_ID: h:abstract-overwrite
:END:

#+vindex: tlon-abstract-overwrite
The user option ~tlon-abstract-overwrite~ controls what happens when you fetch an abstract for an entry that already has one. Its default value is =ask=, which prompts you interactively each time. You can set it to =always= to silently overwrite existing abstracts, or to =never= to skip entries that already have an abstract without prompting.

This option affects ~tlon-fetch-and-set-abstract~ ([[#h:fetch-and-set-abstract][Fetch and set abstract]]) and the batch abstract processing command ~tlon-batch-set-abstracts~ ([[#h:batch-set-abstracts][Batch set abstracts]]).

** AI model for citation replacement
:PROPERTIES:
:CUSTOM_ID: h:replace-citations-model
:END:

#+vindex: tlon-bib-replace-citations-model
The user option ~tlon-bib-replace-citations-model~ specifies the AI model used by ~tlon-bib-replace-citations-in-file~ ([[#h:replace-citations-in-file][Replace citations in file]]) to identify and replace bibliographic citations with structured =<Cite>= tags. The value is a cons cell whose car is the backend name (a string) and whose cdr is the model symbol. For example:

#+begin_src emacs-lisp
(setq tlon-bib-replace-citations-model '("Gemini" . gemini-pro-latest))
#+end_src

When set to ~nil~, the default ~gptel-model~ is used. See ~gptel-extras-ai-models~ for available backend and model combinations.

** AI model for adding missing citations
:PROPERTIES:
:CUSTOM_ID: h:add-missing-citations-model
:END:

#+vindex: tlon-bib-add-missing-citations-model
The user option ~tlon-bib-add-missing-citations-model~ specifies the AI model used by ~tlon-bib-add-missing-citations~ ([[#h:add-missing-citations][Add missing citations]]) to locate and add bibliographic entries for citations marked as missing. Its format and semantics are the same as ~tlon-bib-replace-citations-model~ ([[#h:replace-citations-model][AI model for citation replacement]]). By default, it inherits the value of ~tlon-bib-replace-citations-model~.

** Maximum concurrent citation replacements
:PROPERTIES:
:CUSTOM_ID: h:max-concurrent
:END:

#+vindex: tlon-bib-replace-citations-max-concurrent
The user option ~tlon-bib-replace-citations-max-concurrent~ controls how many files are processed simultaneously when running ~tlon-bib-replace-citations-in-org-files-in-directory~ ([[#h:replace-citations-in-org-dir][Replace citations in Org directory]]). The default is 5. Increase it if your system can handle more parallel AI requests; decrease it if you encounter rate limits or resource constraints.

* Commands
:PROPERTIES:
:CUSTOM_ID: h:commands
:END:

** Abstract management
:PROPERTIES:
:CUSTOM_ID: h:abstract-management
:END:

*** Fetch and set abstract
:PROPERTIES:
:CUSTOM_ID: h:fetch-and-set-abstract
:END:

#+findex: tlon-fetch-and-set-abstract
The command ~tlon-fetch-and-set-abstract~ attempts to fetch an abstract for the BibTeX entry at point and set it in the =abstract= field. It works in both =bibtex-mode= and =ebib-entry-mode=.

The command first checks whether proceeding is allowed based on ~tlon-abstract-overwrite~ ([[#h:abstract-overwrite][Abstract overwrite behavior]]) and any existing abstract. It then tries three data sources in sequence: CrossRef (using the entry's DOI), Google Books (using its ISBN), and Zotra (using its URL or DOI). If an abstract is found, it is cleaned up (XML/HTML tags and extraneous prefixes are removed, and a trailing period is ensured) before being set. If no abstract is found, a message is displayed.

When invoked as part of a batch operation via ~tlon-ai-batch-fun~, the command automatically triggers ~tlon-ai-batch-continue~ to process the next entry.

*** Batch set abstracts
:PROPERTIES:
:CUSTOM_ID: h:batch-set-abstracts
:END:

#+findex: tlon-batch-set-abstracts
The command ~tlon-batch-set-abstracts~ adds abstracts to all entries in the current BibTeX buffer that lack one. It operates in two passes. In the first pass, it tries non-AI sources (CrossRef, Google Books, Zotra) for each entry. In the second pass, it queues entries that have an associated text file for AI-powered abstract generation using ~tlon-make-gptel-request~. The AI pass processes entries one at a time, using the language-specific prompt from ~tlon-ai-get-abstract-prompts~.

This command requires the buffer to be in =bibtex-mode=. It saves the buffer after each pass and reports statistics on how many abstracts were set, skipped, or failed.

*** Translate abstract
:PROPERTIES:
:CUSTOM_ID: h:translate-abstract
:END:

#+findex: tlon-translate-abstract
The command ~tlon-translate-abstract~ translates the abstract of the BibTeX entry at point from its source language (determined by the =langid= field) to a target language using DeepL. If no target language is provided, you are prompted to select one. The translation is stored in the per-language JSON file at =babel-refs/json/<code>/abstract-translations.json=. You can optionally supply a variable to accumulate translations in memory rather than writing to disk after each one.

#+findex: tlon-translate-abstracts
The function ~tlon-translate-abstracts~ extends this to multiple entries. It iterates through up to 2000 entries in the current BibTeX buffer, translating each one that does not already have a translation (unless the overwrite argument is set). When doing large batch translations, use the variable argument to avoid repeated file I/O.

** Entry management
:PROPERTIES:
:CUSTOM_ID: h:entry-management
:END:

*** Move entry to fluid
:PROPERTIES:
:CUSTOM_ID: h:move-entry-to-fluid
:END:

#+findex: tlon-move-entry-to-fluid
The command ~tlon-move-entry-to-fluid~ moves the BibTeX entry at point (or a specified key) from its current file to the fluid BibTeX file (~tlon-file-fluid~). After moving, it adds the =database= field set to "Tlon" and copies the citation key to the kill ring for convenient pasting.

*** Move all fluid entries to stable
:PROPERTIES:
:CUSTOM_ID: h:move-all-to-stable
:END:

#+findex: tlon-move-all-fluid-entries-to-stable
The command ~tlon-move-all-fluid-entries-to-stable~ transfers all entries from =fluid.bib= to =stable.bib= in a single operation. It requires both files to be saved before proceeding. After the move, =fluid.bib= is emptied. You will typically want to commit these changes to version control afterward.

*** Add database field
:PROPERTIES:
:CUSTOM_ID: h:add-database-field
:END:

#+findex: tlon-add-database-field
The command ~tlon-add-database-field~ iterates over every entry in a specified BibTeX file and adds or updates the =database= field to "Tlon". This is useful for tagging entries that belong to the Tlon bibliography.

#+findex: tlon-auto-add-database-field
The function ~tlon-auto-add-database-field~ is intended for use in =after-save-hook=. Whenever the fluid BibTeX file is saved, it automatically runs ~tlon-add-database-field~ on that file.

*** Auto-clean entry
:PROPERTIES:
:CUSTOM_ID: h:auto-clean-entry
:END:

#+findex: tlon-auto-clean-entry
The command ~tlon-auto-clean-entry~ is a hook function for =after-save-hook= in BibTeX mode. On each save, it removes trailing spaces in fields, escapes special characters (=$=, =%=, =#=, =&=), and runs ~bibtex-clean-entry~ if point is at the start of an entry. It skips cleaning on the upstream database file (~tlon-file-db-upstream~).

*** Add language identifiers
:PROPERTIES:
:CUSTOM_ID: h:add-lang-id
:END:

#+findex: tlon-add-lang-id-to-entries
The command ~tlon-add-lang-id-to-entries~ iterates through entries in the current BibTeX buffer. For each entry missing a =langid= field, it prompts you to select a language using ~tlon-select-language~ and sets the field accordingly.

*** Swap colon character in title
:PROPERTIES:
:CUSTOM_ID: h:swap-colon
:END:

#+findex: tlon-bib-swap-colon-in-title
The command ~tlon-bib-swap-colon-in-title~ toggles between the standard colon character (=:=) and the Unicode modified letter colon (=꞉=, U+A789) in the =title= field of a BibTeX entry. This is useful because BibTeX treats the colon as a separator between title and subtitle, so the modified letter colon can be used to avoid unintended splitting. The command prompts for a BibTeX key, jumps to the entry, performs the swap, and saves the file.

*** Unescape characters
:PROPERTIES:
:CUSTOM_ID: h:unescape-characters
:END:

#+findex: tlon-bib-unescape-escaped-characters
The command ~tlon-bib-unescape-escaped-characters~ reverses the escaping performed by ~bibtex-extras-escape-special-characters~. For each character in ~bibtex-extras-escaped-characters~, it replaces the backslash-escaped form (e.g., =\$=) with the literal character (e.g., =$=). It operates on a specified file or the current buffer, and reports the number of replacements made.

*** Remove URL fields
:PROPERTIES:
:CUSTOM_ID: h:remove-url-fields
:END:

#+findex: tlon-bib-remove-url-fields-with-doi
The command ~tlon-bib-remove-url-fields-with-doi~ removes the =url= field from every entry in the current BibTeX buffer that also has a =doi= field. This is useful for cleaning up redundant URL fields when a DOI is available. With a prefix argument, the buffer is saved after modification.

#+findex: tlon-bib-remove-url-fields-with-isbn
The command ~tlon-bib-remove-url-fields-with-isbn~ does the same but for entries that have an =isbn= field instead of a DOI.

*** Create BibTeX original
:PROPERTIES:
:CUSTOM_ID: h:create-original
:END:

#+findex: tlon-create-bibtex-original
The command ~tlon-create-bibtex-original~ adds a new original BibTeX entry to the fluid file by calling ~zotra-extras-add-entry~.

** URL management
:PROPERTIES:
:CUSTOM_ID: h:url-management
:END:

*** Prompt to add missing URLs
:PROPERTIES:
:CUSTOM_ID: h:add-missing-urls
:END:

#+findex: tlon-prompt-to-add-missing-urls
The command ~tlon-prompt-to-add-missing-urls~ scans the current Markdown buffer for URLs that are not present in the Tlon bibliography. For each missing URL, it highlights it and prompts you to decide whether to add it. The URLs you approve are saved to a file for subsequent import via ~zotra-extras-add-multiple-urls~.

*** Populate URL field
:PROPERTIES:
:CUSTOM_ID: h:populate-url-field
:END:

#+findex: tlon-bib-populate-url-field
The command ~tlon-bib-populate-url-field~ constructs and sets the =url= field for the BibTeX entry at point. The URL is built from three components: the base URL for the entry's language from the "uqbar" subproject, the articles path for that language, and a slugified version of the title. If the entry already has a =url= field, you are prompted before overwriting.

#+findex: tlon-bib-populate-url-fields-in-language
The command ~tlon-bib-populate-url-fields-in-language~ extends this to all entries in the language of the entry at point. It collects all article files from the corresponding site directory, reads their BibTeX keys from YAML front matter, and populates the =url= field for each matching entry.

*** Populate journal title fields
:PROPERTIES:
:CUSTOM_ID: h:populate-journaltitle
:END:

#+findex: tlon-bib-populate-journaltitle-fields-in-language
The command ~tlon-bib-populate-journaltitle-fields-in-language~ sets the =journaltitle= field for all BibTeX entries in the language of the entry at point. The journal title value is computed once by looking up the site's homepage title for the appropriate language. Existing values trigger a confirmation prompt before overwriting.

** Translation entry creation
:PROPERTIES:
:CUSTOM_ID: h:translation-entries
:END:

*** Create BibTeX translation
:PROPERTIES:
:CUSTOM_ID: h:create-translation
:END:

#+findex: tlon-create-bibtex-translation
The command ~tlon-create-bibtex-translation~ creates a new BibTeX entry representing a translation of the entry at point. It works in =bibtex-mode=, =ebib-entry-mode=, and =ebib-index-mode=.

The command prompts for a target language (or accepts a two-letter code programmatically). It then translates the title and abstract via DeepL, generates a new key by appending the capitalized language code to the original key, and populates fields including =langid=, =title=, =author=, =journaltitle=, =translation= (the original key), =abstract=, and =date=. In Ebib, it switches to the translations database (database 3) and creates the entry there. The =url= field is then populated via ~tlon-bib-populate-url-field~ ([[#h:populate-url-field][Populate URL field]]).

*** Create translations from directory
:PROPERTIES:
:CUSTOM_ID: h:create-translations-from-dir
:END:

#+findex: tlon-bib-create-translations-from-dir
The command ~tlon-bib-create-translations-from-dir~ iterates over all Markdown files in a directory (defaulting to the current directory). For each file, it reads the YAML =original_key= and =title=, jumps to the corresponding BibTeX entry, calls ~tlon-create-bibtex-translation~, overrides the new entry's =title= with the YAML title (if present), and regenerates the =url= field based on the updated title.

** Entry creation from Markdown
:PROPERTIES:
:CUSTOM_ID: h:entry-from-markdown
:END:

*** Create entry from Markdown
:PROPERTIES:
:CUSTOM_ID: h:create-entry-from-markdown
:END:

#+findex: tlon-bib-create-entry-from-markdown
The command ~tlon-bib-create-entry-from-markdown~ creates a BibTeX entry in the database file (=db.bib=) from the YAML front matter of the current Markdown buffer. It populates the =title=, =author= (from =authors=, reversed to "Last, First" format), =translator= (from =translators=), =langid= (from the repository's language), =date=, and =translation= (from =original_key=). The BibTeX key is constructed by appending the capitalized two-letter language code to the =original_key=. After insertion, the =url= field is populated via ~tlon-bib-populate-url-field~ ([[#h:populate-url-field][Populate URL field]]).

#+findex: tlon-bib-create-entries-in-dir
The command ~tlon-bib-create-entries-in-dir~ applies ~tlon-bib-create-entry-from-markdown~ to every Markdown file in a directory. It continues on errors and reports a summary of created, skipped, and failed entries.

** Citation conversion and replacement
:PROPERTIES:
:CUSTOM_ID: h:citation-commands
:END:

*** Convert links to citations
:PROPERTIES:
:CUSTOM_ID: h:convert-links-to-cite
:END:

#+findex: tlon-convert-links-to-cite
The command ~tlon-convert-links-to-cite~ iterates through all Markdown links in the current buffer. When a link's URL matches the =url= field of an entry in the Tlon bibliography, it prompts you to convert the link to a =<Cite>= tag using the corresponding BibTeX key.

*** Convert bibliography to citations
:PROPERTIES:
:CUSTOM_ID: h:convert-bibliography-to-cite
:END:

#+findex: tlon-convert-bibliography-to-cite
The command ~tlon-convert-bibliography-to-cite~ converts plain-text bibliography entries (in the "Author, Title (Year)" format) to =<Cite>= tags. It looks up the BibTeX key based on the title. This command should be run with the buffer narrowed to the bibliography section, as it assumes one work per line and does not handle locators.

*** Replace keys with formatted citations
:PROPERTIES:
:CUSTOM_ID: h:replace-keys-with-citations
:END:

#+findex: tlon-bib-replace-keys-with-citations
The command ~tlon-bib-replace-keys-with-citations~ replaces all =<Cite>= tags in a file (or the current buffer) with formatted citations fetched from the Tlon API. The citation style ("short" or "long") is determined by the =length= attribute in each =<Cite>= tag. When the optional audio argument is non-nil, the corresponding audio citation styles are used instead.

#+findex: tlon-bib-replace-keys-with-citations-in-dir
The command ~tlon-bib-replace-keys-with-citations-in-dir~ applies this recursively to all Markdown files in the current directory.

*** Check BibTeX keys
:PROPERTIES:
:CUSTOM_ID: h:check-bibkeys
:END:

#+findex: tlon-bib-check-bibkeys
The command ~tlon-bib-check-bibkeys~ validates all BibTeX keys inside =<Cite>= tags in the current buffer, a specified file, or an active region. A key is valid if it exists in the bibliography (as determined by ~tlon-bibliography-lookup~). The command reports any invalid keys found.

** AI-powered citation tools
:PROPERTIES:
:CUSTOM_ID: h:ai-citation-tools
:END:

*** Replace citations in file
:PROPERTIES:
:CUSTOM_ID: h:replace-citations-in-file
:END:

#+findex: tlon-bib-replace-citations-in-file
The command ~tlon-bib-replace-citations-in-file~ uses an AI agent to identify and replace all bibliographic citations in a file with structured =<Cite>= tags. The agent is provided with tools to search the bibliography, fetch web content, perform web searches, and edit the file. Citations that cannot be found are enclosed in ={!= and =!}= markers for later processing.

When called from Dired with marked files, it processes all marked files. When called with an active region, it processes the current file. Otherwise, it prompts for a file. Files are processed in batches of ~tlon-bib-replace-citations-max-concurrent~ ([[#h:max-concurrent][Maximum concurrent citation replacements]]).

The AI model used is controlled by ~tlon-bib-replace-citations-model~ ([[#h:replace-citations-model][AI model for citation replacement]]).

*** Replace citations in Org directory
:PROPERTIES:
:CUSTOM_ID: h:replace-citations-in-org-dir
:END:

#+findex: tlon-bib-replace-citations-in-org-files-in-directory
The command ~tlon-bib-replace-citations-in-org-files-in-directory~ runs ~tlon-bib-replace-citations-in-file~ on every =.org= file in a specified directory (non-recursively). Files are processed in batches; the next batch starts only after all files in the current batch have finished.

*** Add missing citations
:PROPERTIES:
:CUSTOM_ID: h:add-missing-citations
:END:

#+findex: tlon-bib-add-missing-citations
The command ~tlon-bib-add-missing-citations~ instructs an AI agent to process a file or region and find all citations marked as missing (enclosed in ={!= and =!}=). For each, the agent uses search tools to find an identifier (URL, DOI, or ISBN), adds a corresponding entry to the fluid bibliography file via the =add_bib_entry= tool, and replaces the marked citation with a proper =<Cite>= tag.

The AI model used is controlled by ~tlon-bib-add-missing-citations-model~ ([[#h:add-missing-citations-model][AI model for adding missing citations]]).

** Reporting
:PROPERTIES:
:CUSTOM_ID: h:reporting
:END:

*** Entries report
:PROPERTIES:
:CUSTOM_ID: h:entries-report
:END:

#+findex: tlon-bib-entries-report
The command ~tlon-bib-entries-report~ generates a report buffer listing BibTeX entries in the current buffer that have problems with their abstracts. It categorizes entries into four groups: those with no abstract, those with abstracts shorter than ~tlon-min-abstract-length~ words, those with abstracts longer than ~tlon-max-abstract-length~ words, and those with abstracts of fewer than 10 words. Entries in ~tlon-bib-excluded-keys~ ([[#h:excluded-keys][Excluded keys management]]), entries with a =crossref= field, and translation entries (those with a =translation= field) are filtered out.

*** Excluded keys management
:PROPERTIES:
:CUSTOM_ID: h:excluded-keys
:END:

#+findex: tlon-bib-add-to-excluded-keys
The command ~tlon-bib-add-to-excluded-keys~ adds one or more BibTeX keys (as a space-separated string) to the ~tlon-bib-excluded-keys~ list and persists them to ~tlon-bib-excluded-keys-file~. These keys are excluded from the entries report ([[#h:entries-report][Entries report]]).

*** Find entry with missing field
:PROPERTIES:
:CUSTOM_ID: h:find-missing-field
:END:

#+findex: tlon-bib-find-next-entry-with-missing-field
The command ~tlon-bib-find-next-entry-with-missing-field~ moves point to the next BibTeX entry in the buffer that is missing a specified field. It prompts for the field name with completion from ~bibtex-extras-biblatex-fields~ and skips entries that have a =crossref= field.

** Transient menu
:PROPERTIES:
:CUSTOM_ID: h:transient-menu
:END:

#+findex: tlon-bib-menu
The command ~tlon-bib-menu~ activates a transient menu providing quick access to the commands in this module. It is organized into three columns:

- *Markdown*: URL management (=m f=, =m z=), citation conversion (=m b=, =m l=), key checking (=m c=), AI citation replacement (=m r=, =m R=, =m a=), and model selection (=m -r=, =m -a=).
- *Ebib*: abstract fetching (=e a=), translation entry creation (=e c=), URL population (=e u=, =e U=), and journal title population (=e J=).
- *BibTeX*: reporting (=b g=, =b x=), entry movement (=b t=, =b s=), URL field removal (=b d=, =b i=), and colon swapping (=b :=).

* Functions
:PROPERTIES:
:CUSTOM_ID: h:functions
:END:

** Mode guards
:PROPERTIES:
:CUSTOM_ID: h:mode-guards
:END:

#+findex: tlon-ensure-bib
The function ~tlon-ensure-bib~ signals a =user-error= unless the current buffer is in =bibtex-mode=, =ebib-entry-mode=, or =ebib-index-mode=. Use it at the start of commands that require a BibTeX-related context.

#+findex: tlon-ensure-bibtex
The function ~tlon-ensure-bibtex~ is a stricter variant that signals an error unless the buffer is specifically in =bibtex-mode=.

** Key retrieval
:PROPERTIES:
:CUSTOM_ID: h:key-retrieval
:END:

#+findex: tlon-get-key-at-point
The function ~tlon-get-key-at-point~ returns the BibTeX key at point, dispatching to the appropriate backend function depending on whether you are in Ebib or =bibtex-mode=.

** Bibliography lookup
:PROPERTIES:
:CUSTOM_ID: h:bibliography-lookup
:END:

#+findex: tlon-bibliography-lookup
The function ~tlon-bibliography-lookup~ searches the cached Tlon bibliography (via =citar-cache--bibliographies=) for an entry where a specified field matches a given value. It can optionally perform substring matching and return a different field from the matching entry. This is the central lookup mechanism used by many commands in this module.

#+findex: tlon-get-field-in-bibliography
The function ~tlon-get-field-in-bibliography~ returns a list of all values for a specified field across all entries in the cached bibliography. It signals an error if no bibliographies are cached.

#+findex: tlon-bib-get-keys-in-file
The function ~tlon-bib-get-keys-in-file~ returns a list of all BibTeX keys in a specified file by accessing the citar cache. It ensures the file is cached if not already present.

** Abstract translation helpers
:PROPERTIES:
:CUSTOM_ID: h:abstract-translation-helpers
:END:

#+findex: tlon-read-abstract-translations
The function ~tlon-read-abstract-translations~ reads and parses the per-language JSON abstract translations file for a given language code or name, returning an alist of =(KEY . TEXT)= pairs.

#+findex: tlon-write-abstract-translations
The function ~tlon-write-abstract-translations~ writes an alist of =(KEY . TEXT)= pairs to the per-language JSON file. It creates the directory if needed.

#+findex: tlon-add-abstract-translation
The function ~tlon-add-abstract-translation~ adds or updates a translation for a given key in the per-language store. When a variable is provided instead of a language, the alist is updated in memory rather than written to disk.

#+findex: tlon-get-abstract-translation
The function ~tlon-get-abstract-translation~ retrieves the translated abstract for a specific BibTeX key and language code from the per-language JSON file.

** Autokey generation
:PROPERTIES:
:CUSTOM_ID: h:autokey-generation
:END:

#+findex: tlon-generate-autokey
The function ~tlon-generate-autokey~ generates a BibTeX citation key from author, year, and title strings. It respects the standard BibTeX autokey settings (separators, name count, year length, title word count) and optionally applies ~bibtex-autokey-before-presentation-function~.

** Reporting helpers
:PROPERTIES:
:CUSTOM_ID: h:reporting-helpers
:END:

#+findex: tlon-bib-load-excluded-keys
The function ~tlon-bib-load-excluded-keys~ loads the excluded keys list from ~tlon-bib-excluded-keys-file~ into ~tlon-bib-excluded-keys~.

#+findex: tlon-bib-save-excluded-keys
The function ~tlon-bib-save-excluded-keys~ persists the current ~tlon-bib-excluded-keys~ list back to its file.

** Miscellaneous helpers
:PROPERTIES:
:CUSTOM_ID: h:misc-helpers
:END:

#+findex: tlon-abstract-cleanup
The function ~tlon-abstract-cleanup~ cleans a raw abstract string by removing XML/HTML tags, LaTeX formatting artifacts, and extraneous leading words like "summary" or "abstract". It also ensures the string ends with a period.

#+findex: tlon-bib-remove-braces
The function ~tlon-bib-remove-braces~ strips curly braces from a string, which is useful before sending BibTeX text to external services like DeepL.

#+findex: tlon-fetch-url-from-doi
The function ~tlon-fetch-url-from-doi~ resolves a DOI to its final redirect URL using =curl=.

#+findex: tlon-bib-add-multiple-urls-from-file
The function ~tlon-bib-add-multiple-urls-from-file~ reads a file containing URLs (one per line), filters out those already in the bibliography, and calls ~zotra-extras-add-multiple-urls~ to add the rest to the fluid file.
