This manual describes the features and customization options for the Emacs Lisp file =tlon-repos.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

The =tlon-repos= module provides functions for managing Tlon's Git repositories. It integrates with =vc=, =vc-extras=, =forge=, =forge-extras=, and =pass-extras= to offer commands for creating, cloning, deleting, tracking, searching, and interacting with repositories, both locally and on GitHub under the =tlon-team= organization.

The module also provides a custom Forge dispatch menu that overrides the default =forge-dispatch=, adding Tlon-specific search capabilities alongside standard Forge operations.

* User options
:PROPERTIES:
:CUSTOM_ID: h:user-options
:END:

This module does not define ~defcustom~ user options. It defines a customization group ~tlon-repos~ under the parent group ~tlon~.

* Commands
:PROPERTIES:
:CUSTOM_ID: h:commands
:END:

** Version control operations
:PROPERTIES:
:CUSTOM_ID: h:vc-operations
:END:

#+findex: tlon-create-repo
- ~tlon-create-repo~ :: Create a new repository on GitHub under the =tlon-team= organization. Prompts for the repository NAME if not provided. Delegates to ~vc-extras-create-repo~.

#+findex: tlon-clone-repo
- ~tlon-clone-repo~ :: Clone an existing Tlon repository from the =tlon-team= organization into the directory specified by =paths-dir-tlon-repos=. Prompts for the repository NAME if not provided. If NO-FORGE is non-nil, do not prompt to add the repo to the Forge database. Delegates to ~vc-extras-clone-repo~.

#+findex: tlon-clone-missing-repos
- ~tlon-clone-missing-repos~ :: Clone all Tlon repositories defined in the configuration (via ~tlon-repo-lookup-all :dir~) that do not exist locally. Clones are performed asynchronously and do not automatically add the repos to the Forge database. To track these repos in Forge, use ~tlon-forge-track-missing-repos~ afterwards.

#+findex: tlon-delete-local-repo
- ~tlon-delete-local-repo~ :: Delete a local copy of a Tlon repository. If NAME is nil, prompts you to select from the intersection of configured Tlon repos and locally existing ones. Delegates to ~vc-extras-delete-local-repo~.

** Forge operations
:PROPERTIES:
:CUSTOM_ID: h:forge-operations
:END:

#+findex: tlon-forge-track-repo
- ~tlon-forge-track-repo~ :: Track a Tlon repository NAME in the Forge database. If NAME is nil, prompts you to select from local Tlon repos. Delegates to ~forge-extras-track-repository~.

#+findex: tlon-forge-track-missing-repos
- ~tlon-forge-track-missing-repos~ :: Add all missing Tlon repos to the Forge database. Only processes repos that exist locally; to clone missing repos first, use ~tlon-clone-missing-repos~. For each untracked repo, calls ~forge-extras-track-repo-all-topics~ and waits for the tracking operation to complete.

#+findex: tlon-pull-issues-in-repo
- ~tlon-pull-issues-in-repo~ :: Pull issues in the repository located at DIR. If DIR is nil, use the current directory. Only pulls if the repository is tracked in Forge.

#+findex: tlon-pull-issues-in-all-repos
- ~tlon-pull-issues-in-all-repos~ :: Pull issues in all Tlon repositories. Iterates through all directories returned by ~tlon-repo-lookup-all :dir~ and calls ~tlon-pull-issues-in-repo~ for each.

** Search
:PROPERTIES:
:CUSTOM_ID: h:search
:END:

#+findex: tlon-forge-search
- ~tlon-forge-search~ :: Search for STRING across Tlon repositories. If REPOS is nil, search in all tracked repos. For each repository, sets up a Magit status buffer, runs =forge-search=, and collects results. Reports the number of repos with matches and the elapsed time.

** Git-crypt
:PROPERTIES:
:CUSTOM_ID: h:git-crypt
:END:

#+findex: tlon-git-crypt-unlock
- ~tlon-git-crypt-unlock~ :: Unlock a Tlon repository using =git-crypt=. Prompts you to select a repository from the list derived from entries under =tlon/core/git-crypt/= in the password store. Delegates to ~pass-extras-git-crypt-unlock~.

** Issues-to-projects configuration
:PROPERTIES:
:CUSTOM_ID: h:issues-to-projects
:END:

#+findex: tlon-repos-configure-issues-to-projects
- ~tlon-repos-configure-issues-to-projects~ :: Configure the =issues-to-projects= workflow for REPO-NAME. This command performs the equivalent of the manual steps described in the =issues-to-projects= repository README:

  1. Write REPO-NAME into =config.sh= at line 24.
  2. Run =automate-github-secrets.sh= to set the GitHub secret.
  3. Run =batch-deploy-add-new-issues-workflow.sh= to deploy the workflow.

  When called interactively, prompts for a local Tlon repository name. The original =config.sh= is backed up and restored after the scripts complete.

** Transient menus
:PROPERTIES:
:CUSTOM_ID: h:transient-menus
:END:

#+findex: tlon-repos-menu
- ~tlon-repos-menu~ :: Activate the =transient= menu interface for repository management.

  + *Git*:
    - =l= :: clone repo (~tlon-clone-repo~)
    - =L= :: clone all missing repos (~tlon-clone-missing-repos~)
    - =s= :: split local repo (~vc-extras-split-local-repo~)
    - =d= :: delete local repo (~tlon-delete-local-repo~)
    - =c= :: create remote repo (~tlon-create-repo~)
    - =u= :: unlock git-crypt (~tlon-git-crypt-unlock~)
    - =?= :: check authentication (~vc-extras-check-gh-authenticated~)
    - =I= :: configure issues-to-projects (~tlon-repos-configure-issues-to-projects~)
  + *Forge*:
    - =a= :: track repo (~tlon-forge-track-repo~)
    - =A= :: track all missing repos (~tlon-forge-track-missing-repos~)
    - =r= :: untrack repo (~forge-remove-repository~)
    - =R= :: untrack all repos / db reset (~forge-reset-database~)
    - =t= :: list tracked repos (~forge-list-repositories~)
    - =p= :: pull issues in repo (~tlon-pull-issues-in-repo~)
    - =P= :: pull issues in all repos (~tlon-pull-issues-in-all-repos~)
    - =U= :: unlock git-crypt (~tlon-git-crypt-unlock~)

#+findex: tlon-forge-menu
- ~tlon-forge-menu~ :: A custom Forge dispatch menu that overrides the default =forge-dispatch= via =advice-add=. Extends the standard Forge menu with Tlon-specific search (=s a= for searching all repos via ~tlon-forge-search~) alongside the usual Forge create, list, fetch, visit, and browse commands.

** Repository helpers

#+findex: tlon-get-local-repos
- ~tlon-get-local-repos~ :: Prompt you to select from a list of Tlon repository names that exist locally. Returns the selected name. The candidates are the intersection of all configured Tlon repo names and the locally available repos under =tlon-team=.
