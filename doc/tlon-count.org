This manual describes the features and customization options for the Emacs Lisp file =tlon-count.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

The =tlon-count= module provides functionality for counting words in files, directories, and repositories.  It counts "substantive" words (total words minus YAML front matter and local variables) and supports generating Org mode tables of per-file word counts, exporting those tables to TSV and Google Drive, counting BibTeX key occurrences across files, and computing historical word counts (via external tools).

The module depends on =tlon-md=, =tlon-yaml=, and =transient=.  Some features additionally require the external CLI tools =gdrive=, =gdu=, and =gnuplot=.

* Commands
:PROPERTIES:
:CUSTOM_ID: h:commands
:END:

** Word counting
:PROPERTIES:
:CUSTOM_ID: h:word-counting
:END:

The word counting commands share the following optional arguments:

- FORMAT :: Determines the output format.  If =list=, return a list of cons cells =((COUNT . FILENAME) ...)=.  If =count=, return the total word count as a number.  If =summary= (or any other value, including =nil=), display a message such as =%s words in %s files.= and copy the count to the kill ring.
- REGEXP :: A regular expression to filter filenames.  Defaults to =\\.md$= (Markdown files).

*** ~tlon-count-words-in-repo~
:PROPERTIES:
:CUSTOM_ID: h:tlon-count-words-in-repo
:END:
#+findex: tlon-count-words-in-repo

Count words in files matching REGEXP within a Tlon repository.  If REPO-NAME is not provided, prompt you to select one from the translation repositories.  The directory is resolved via ~tlon-repo-lookup~ and the count is performed recursively via ~tlon-count-words-in-dir~.

*** ~tlon-count-words-in-dir~
:PROPERTIES:
:CUSTOM_ID: h:tlon-count-words-in-dir
:END:
#+findex: tlon-count-words-in-dir

Count words in files matching REGEXP within directory DIR.  If DIR is =nil=, try to use the directory at point; otherwise signal an error.  When RECURSIVELY is non-nil, include files in subdirectories.  Delegates to ~tlon-count-words-in-files~.

*** ~tlon-count-words-in-files~
:PROPERTIES:
:CUSTOM_ID: h:tlon-count-words-in-files
:END:
#+findex: tlon-count-words-in-files

Count words in a list of FILES.  If FILES is =nil=, attempt to use the files marked in the current Dired buffer (via ~dired-get-marked-files~).  For each file, the substantive word count is computed with ~tlon-count-substantive-words~.  The result format is governed by FORMAT.

** Org table generation and export
:PROPERTIES:
:CUSTOM_ID: h:table-export
:END:

*** ~tlon-count-create-table-for-dir~
:PROPERTIES:
:CUSTOM_ID: h:tlon-count-create-table-for-dir
:END:
#+findex: tlon-count-create-table-for-dir

Create an Org mode table with per-file word counts for all Markdown files in DIR.  If DIR is =nil=, use the directory at point in Dired.  The command performs the following steps:

1. Obtain the word count list via ~tlon-count-words-in-dir~ with FORMAT =list=.
2. Determine the output path (typically in the =stats/= subdirectory of the repository) via ~tlon-count-get-table-output-path~.
3. Insert the data as an Org table using ~tlon-count-insert-table-from-list~ and save it.
4. Export the Org table to a TSV file via ~tlon-count-export-org-table~.
5. Upload the TSV file to Google Drive via ~tlon-count-export-to-google-drive~.

** BibTeX key counting
:PROPERTIES:
:CUSTOM_ID: h:bibtex-counting
:END:

*** ~tlon-count-bibtex-keys-in-dir~
:PROPERTIES:
:CUSTOM_ID: h:tlon-count-bibtex-keys-in-dir
:END:
#+findex: tlon-count-bibtex-keys-in-dir

Count occurrences of each BibTeX key (in =Cite= tags) across all non-hidden files in DIR.  With a prefix argument (RECURSIVE non-nil), include files in subdirectories.  The results are sorted in descending order by count and displayed in a dedicated =/BibTeX Key Counts/= buffer.

** Historical word count
:PROPERTIES:
:CUSTOM_ID: h:historic-word-count
:END:

*** ~tlon-historic-word-count~
:PROPERTIES:
:CUSTOM_ID: h:tlon-historic-word-count
:END:
#+findex: tlon-historic-word-count

Compute the historical word count of REPO-NAME for the past DAYS.  CHARS-PER-WORD (default 5.5) is used to estimate word count from file sizes.  Requires the external tools =gdu= and =gnuplot= to be installed.  The command executes the =count/historic-word-count= shell script from the =babel= repository and displays its output in the =*Directory Size*= buffer.

** Transient menu
:PROPERTIES:
:CUSTOM_ID: h:tlon-count-menu
:END:
#+findex: tlon-count-menu

The command ~tlon-count-menu~ activates a =transient= menu for counting operations.

The menu provides the following groups:

- *Count words*:
  - =w f= :: in file(s) (~tlon-count-words-in-files~)
  - =w d= :: in dir (~tlon-count-words-in-dir~)
  - =w r= :: in repo (~tlon-count-words-in-repo~)
  - =w t= :: create table for dir (~tlon-count-create-table-for-dir~)
- *Count BibTeX keys*:
  - =k d= :: in dir (~tlon-count-bibtex-keys-in-dir~)

* Functions
:PROPERTIES:
:CUSTOM_ID: h:functions
:END:

** Word count filtering
:PROPERTIES:
:CUSTOM_ID: h:word-count-filtering
:END:

#+findex: tlon-count-substantive-words
- ~tlon-count-substantive-words~ :: Return the number of substantive words in the current buffer.  Substantive words are all words minus extraneous words (as defined by ~tlon-count-extraneous-words~).

#+findex: tlon-count-extraneous-words
- ~tlon-count-extraneous-words~ :: Return the number of extraneous words in the current buffer.  Extraneous words are those in the YAML front matter section (obtained via ~tlon-yaml-get-metadata~) and the local variables section (obtained via ~tlon-md-get-local-variables~).

** Org table and export helpers
:PROPERTIES:
:CUSTOM_ID: h:table-helpers
:END:

#+findex: tlon-count-insert-table-from-list
- ~tlon-count-insert-table-from-list~ :: Insert an Org mode table into a new buffer (=*Tlon stats*=) from LIST, a list of cons cells =((COUNT . FILENAME) ...)=.  If DESTINATION is non-nil, save the buffer to that file path.

#+findex: tlon-count-get-table-output-path
- ~tlon-count-get-table-output-path~ :: Return the path of the Org table output file for DIR.  The target directory is =stats/= within the current repository, and the filename is derived from the base name of DIR with an =.org= extension.

#+findex: tlon-count-export-org-table
- ~tlon-count-export-org-table~ :: Export the first Org table in FILE (or the current buffer's file) to a TSV file using =org-table-export= with the ="orgtbl-to-tsv"= format.  The destination path is derived via ~tlon-count-get-tsv-file~.

#+findex: tlon-count-get-tsv-file
- ~tlon-count-get-tsv-file~ :: Return the TSV file path corresponding to FILE, using the same directory and base name with a =.tsv= extension.

#+findex: tlon-count-export-to-google-drive
- ~tlon-count-export-to-google-drive~ :: Upload FILE to Google Drive.  Determines the target language from the file path, looks up the corresponding folder ID in ~tlon-count-gdrive-directory-ids~, and uses the external =gdrive= CLI tool to upload.  Opens the target folder in the browser afterwards.

** BibTeX key counting helpers
:PROPERTIES:
:CUSTOM_ID: h:bibtex-helpers
:END:

#+findex: tlon-get-files-in-dir
- ~tlon-get-files-in-dir~ :: Return a list of all non-hidden, non-directory files in DIR.  If RECURSIVE is non-nil, include files in subdirectories.

#+findex: tlon-extract-bibkeys-from-file
- ~tlon-extract-bibkeys-from-file~ :: Extract and count BibTeX keys in FILE matching BIBKEY-PATTERN.  Returns a hash table mapping keys to their counts.

#+findex: tlon-merge-key-counts
- ~tlon-merge-key-counts~ :: Merge the key counts from HASH-TABLE2 into HASH-TABLE1 by adding counts for shared keys.  Modifies and returns HASH-TABLE1.

#+findex: tlon-sort-key-counts
- ~tlon-sort-key-counts~ :: Convert KEY-COUNTS hash table to a list of cons cells =((KEY . COUNT) ...)= sorted in descending order by count.

#+findex: tlon-display-key-counts
- ~tlon-display-key-counts~ :: Display SORTED-KEYS in the =/BibTeX Key Counts/= buffer, formatted as count followed by key name.

** Variables
:PROPERTIES:
:CUSTOM_ID: h:variables
:END:

#+vindex: tlon-count-gdrive-directory-ids
- ~tlon-count-gdrive-directory-ids~ :: Alist mapping language codes (e.g., ="en"=, ="es"=) to their corresponding Google Drive folder IDs.  Used by ~tlon-count-export-to-google-drive~ to determine the upload destination.
