This manual describes the features and customization options for the Emacs Lisp file =tlon-core.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

The file =tlon-core.el= provides the foundational building blocks used throughout the ~tlon~ package. Every other module in the package depends on this file, either directly or transitively. It defines the central data structures (repository metadata, user records, language tables, label configurations), the generic lookup engine that queries them, and a broad set of utility functions for tasks such as JSON processing, Org mode integration, number formatting, Git state inspection, and transient menu construction.

The key abstractions provided by this module are:

- *Repository registry*: the constant ~tlon-repos~ holds a comprehensive plist-based database of all Tlon repositories, including their names, abbreviations, types, subtypes, languages, URLs, and directory paths.
- *Generic lookup engine*: the family of ~tlon-lookup~ / ~tlon-lookup-all~ functions, together with their specialized wrappers (~tlon-repo-lookup~, ~tlon-user-lookup~, ~tlon-metadata-lookup~, ~tlon-label-lookup~, ~tlon-issue-lookup~), provide a uniform way to query any plist-based or alist-based data structure by key-value pairs.
- *Language handling*: functions for selecting, validating, and converting language identifiers between names, ISO 639-1 codes, and locale strings.
- *JSON utilities*: reading, writing, and editing JSON files.
- *Number formatting*: language-aware parsing of formatted numbers with locale-specific thousands and decimal separators.

This module requires the ~elpaca~, ~json~, ~paths~, and ~transient~ packages.

* User options
:PROPERTIES:
:CUSTOM_ID: h:user-options
:END:

** ~tlon-debug~
:PROPERTIES:
:CUSTOM_ID: h:tlon-debug
:END:

#+vindex: tlon-debug
When set to non-~nil~, the ~tlon~ package emits additional diagnostic messages via the ~tlon-message-debug~ macro. These messages are useful when investigating unexpected behavior in any module that uses the macro. The debug toggle is also exposed as a transient infix in the main dispatch menu ([[info:tlon-dispatch][tlon-dispatch]]).

The default value is ~nil~. The type is boolean.

** ~tlon-visit-file-other-window~
:PROPERTIES:
:CUSTOM_ID: h:tlon-visit-file-other-window
:END:

#+vindex: tlon-visit-file-other-window
When set to non-~nil~, the command ~tlon-visit-file-at-point~ opens the target file in another window rather than replacing the current one ([[#h:visit-file-at-point][Visiting files at point]]). This is convenient when you want to keep a log or error buffer visible while jumping to the referenced file.

The default value is ~nil~. The type is boolean.

** ~tlon-email-shared~
:PROPERTIES:
:CUSTOM_ID: h:tlon-email-shared
:END:

#+vindex: tlon-email-shared
The shared Gmail address used by the Tlon team for organizational email. Modules that send or reference team-wide email (such as newsletter or contact workflows) use this variable.

The default value is ="tlon.shared@gmail.com"=. The type is a string.

* Commands
:PROPERTIES:
:CUSTOM_ID: h:commands
:END:

** Searching across repositories
:PROPERTIES:
:CUSTOM_ID: h:searching
:END:

#+findex: tlon-grep
The command ~tlon-grep~ performs a recursive search for a string across files in the current Tlon repository, using the ~elgrep~ package as its backend. When called interactively, it prompts for the search string and a file extension filter (empty string means all files). It then asks whether you want context lines around matches, and if so, how many. This is a convenient alternative to running ~rg~ or ~grep~ manually when you want results presented in an Emacs buffer.

** Sorting Org headings
:PROPERTIES:
:CUSTOM_ID: h:sorting-headings
:END:

#+findex: tlon-sort-headings
The command ~tlon-sort-headings~ sorts all sibling headings under the current parent heading in an Org file, first alphabetically and then by TODO keyword order. This is useful for maintaining a consistent ordering of entries in Org-based metadata files. If FILE is provided, it operates on that file; otherwise it uses the current buffer.

** Visiting files at point
:PROPERTIES:
:CUSTOM_ID: h:visit-file-at-point
:END:

#+findex: tlon-visit-file-at-point
The command ~tlon-visit-file-at-point~ parses the current line for a file reference in the format =PATH.md:...position N= and opens the file at the specified character position. The file path may begin with =~= and is expanded to an absolute path. If ~tlon-visit-file-other-window~ is non-~nil~, the file opens in another window ([[#h:tlon-visit-file-other-window][the user option]]). This command is designed for navigating from log or validation output buffers where each line references a file and position.

** Pulling all Uqbar repositories
:PROPERTIES:
:CUSTOM_ID: h:uqbar-pull-all
:END:

#+findex: tlon-uqbar-pull-all
The command ~tlon-uqbar-pull-all~ runs the =pull-all.sh= script located in the root of the =uqbar= repository. This script pulls the latest changes from all Uqbar content repositories in one operation. The command compiles the script output in an Emacs compilation buffer so you can review any errors. It signals a user error if the =uqbar= repository or the script cannot be found.

** Looking up Uqbar front messages
:PROPERTIES:
:CUSTOM_ID: h:uqbar-front-messages
:END:

#+findex: tlon-uqbar-front-get-message
The command ~tlon-uqbar-front-get-message~ retrieves a value from the Uqbar front-end message JSON files. When called interactively, it prompts for a language code (e.g., ="es"=), an outer key (e.g., ="HomePage"=), and an inner key (e.g., ="title"=), then returns and displays the corresponding string. This is useful for checking localized UI strings without opening the JSON files manually. It can also be called programmatically with three arguments: LANG, OUTER, and INNER.

* Functions
:PROPERTIES:
:CUSTOM_ID: h:functions
:END:

** Repository management
:PROPERTIES:
:CUSTOM_ID: h:repo-management
:END:

#+findex: tlon-set-dir
The function ~tlon-set-dir~ sets the ~:dir~ property for a REPO entry in ~tlon-repos~ if it is not already set, by concatenating the Tlon repos base directory (~paths-dir-tlon-repos~) with the repository name. This function is called automatically for every repository at load time via ~mapc~.

#+findex: tlon-get-repo
The function ~tlon-get-repo~ returns the directory path of the Tlon repository associated with the current context. If the current file or directory belongs to a known repository, that repository is returned directly. Otherwise, if NO-PROMPT is ~nil~, the user is prompted to select one. If NO-PROMPT is the symbol ~error~, a user error is signaled instead. When INCLUDE-ALL is non-~nil~, all repository types are considered; by default, only translation repositories are offered for selection.

#+findex: tlon-get-repo-from-file
The function ~tlon-get-repo-from-file~ determines which Tlon repository FILE belongs to, based on its directory path. If FILE is ~nil~, it falls back to the current buffer's file name or ~default-directory~. When ERROR is non-~nil~, it signals an error if no matching repository is found.

#+findex: tlon-get-repo-from-dir
The function ~tlon-get-repo-from-dir~ returns the repository directory that DIR belongs to. It handles the special case where a repository's =.git= directory is stored outside the repository itself (via ~paths-dir-split-git~).

** Generic lookup
:PROPERTIES:
:CUSTOM_ID: h:generic-lookup
:END:

The lookup functions form the query engine that underlies most data access in the ~tlon~ package. They search through lists of plists or alists for entries matching key-value criteria and return the value of a target key.

#+findex: tlon-lookup
The function ~tlon-lookup~ returns the value of KEY in the first entry of LIST that matches all specified key-value PAIRS. PAIRS is an even-length list of alternating keys and values. Matching is case-sensitive.

#+findex: tlon-lookup-case-insensitive
The function ~tlon-lookup-case-insensitive~ works identically to ~tlon-lookup~ but performs case-insensitive matching on values.

#+findex: tlon-lookup-all
The function ~tlon-lookup-all~ returns a list of all unique values of KEY across all entries in LIST that match the given PAIRS. This is the multi-result counterpart of ~tlon-lookup~.

#+findex: tlon-lookup-all-case-insensitive
The function ~tlon-lookup-all-case-insensitive~ is the case-insensitive variant of ~tlon-lookup-all~.

#+findex: tlon-lookup-builder
The function ~tlon-lookup-builder~ is the underlying implementation for single-result lookups. It accepts an explicit CASE-INSENSITIVE flag. External code should prefer ~tlon-lookup~ or ~tlon-lookup-case-insensitive~.

#+findex: tlon-lookup-all-builder
The function ~tlon-lookup-all-builder~ is the underlying implementation for multi-result lookups. Like ~tlon-lookup-builder~, it accepts an explicit CASE-INSENSITIVE flag.

#+findex: tlon-all-pairs-in-entry-p
The function ~tlon-all-pairs-in-entry-p~ returns ~t~ if all key-value PAIRS are present in ENTRY. The CASE-INSENSITIVE flag controls whether value comparison uses ~cl-equalp~ (case-insensitive) or ~equal~ (case-sensitive).

#+findex: tlon-get-value-in-entry
The function ~tlon-get-value-in-entry~ extracts the value associated with KEY from ENTRY. It handles both alist entries (when KEY is a string, using ~string=~ for comparison) and plist entries (when KEY is a keyword or symbol).

** Specialized lookups
:PROPERTIES:
:CUSTOM_ID: h:specialized-lookups
:END:

These functions are thin wrappers around the generic lookup engine, pre-bound to specific data structures.

#+findex: tlon-metadata-lookup
The function ~tlon-metadata-lookup~ performs a ~tlon-lookup~ on a METADATA structure (typically the parsed YAML front matter of content files).

#+findex: tlon-metadata-lookup-all
The function ~tlon-metadata-lookup-all~ returns all matching values from a METADATA structure.

#+findex: tlon-repo-lookup
The function ~tlon-repo-lookup~ queries the global ~tlon-repos~ registry. For example, ~(tlon-repo-lookup :dir :name "uqbar-es")~ returns the directory of the =uqbar-es= repository.

#+findex: tlon-repo-lookup-all
The function ~tlon-repo-lookup-all~ returns all matching values from ~tlon-repos~. For example, ~(tlon-repo-lookup-all :name :subtype 'translations)~ returns the names of all translation repositories.

#+findex: tlon-user-lookup
The function ~tlon-user-lookup~ queries the ~tlon-users~ variable.

#+findex: tlon-user-lookup-all
The function ~tlon-user-lookup-all~ returns all matching values from ~tlon-users~.

#+findex: tlon-label-lookup
The function ~tlon-label-lookup~ queries the ~tlon-job-labels~ variable (defined in =tlon-jobs.el=).

#+findex: tlon-label-lookup-all
The function ~tlon-label-lookup-all~ returns all matching values from ~tlon-job-labels~.

#+findex: tlon-issue-lookup
The function ~tlon-issue-lookup~ searches the Forge database for the first issue whose title contains STRING in the repository at DIR (or the current directory). It returns a Forge issue object, or ~nil~ if no match is found.

** Directory and bare directory handling
:PROPERTIES:
:CUSTOM_ID: h:bare-dirs
:END:

In the Tlon content structure, a "bare directory" is the first directory component below the repository root, such as =articulos= or =tags=. These directory names vary by language, and the functions below handle translation between them.

#+findex: tlon-get-bare-dir
The function ~tlon-get-bare-dir~ returns the bare directory of FILE (or the current buffer's file). For example, for a file at =~/repos/uqbar-es/articulos/my-article.md=, it returns ="articulos"=. If IN-ENGLISH is non-~nil~, it returns the English equivalent (="articles"=).

#+findex: tlon-get-bare-dir-translation
The function ~tlon-get-bare-dir-translation~ translates BARE-DIR from SOURCE-LANG to TARGET-LANG using the ~tlon-core-bare-dirs~ translation table.

#+findex: tlon-select-bare-dir
The function ~tlon-select-bare-dir~ prompts the user to select a bare directory for language LANG from the available options.

** Language handling
:PROPERTIES:
:CUSTOM_ID: h:language-handling
:END:

#+findex: tlon-get-language-in-file
The function ~tlon-get-language-in-file~ returns the ISO 639-1 language code for the repository that FILE belongs to. If FILE is ~nil~, it uses the current buffer's file. If ERROR is non-~nil~, it signals an error when the language cannot be determined.

#+findex: tlon-get-language-in-mode
The function ~tlon-get-language-in-mode~ returns the language of the current buffer in standard format (e.g., ="spanish"=). It dispatches on ~major-mode~: in ~ebib-entry-mode~ or ~bibtex-mode~, it reads the =langid= BibTeX field; in ~markdown-mode~, it looks up the repository language and converts the ISO code to a standard name.

#+findex: tlon-validate-language
The function ~tlon-validate-language~ checks whether LANGUAGE is a valid identifier and returns it in lowercase if so, or ~nil~ otherwise. If FORMAT is the symbol ~code~, validation is performed against ISO 639-1 codes; otherwise, against full language names.

#+findex: tlon-get-iso-code
The function ~tlon-get-iso-code~ converts a LANGUAGE name (e.g., ="spanish"=) into its two-letter ISO 639-1 code (e.g., ="es"=). If the input is already a two-letter string, it is returned unchanged.

#+findex: tlon-select-language
The function ~tlon-select-language~ prompts the user to select a language and returns it in the requested FORMAT (~code~, ~locale~, or the default full name). When BABEL is non-~nil~, only languages in the Babel project are offered. MULTIPLE allows selecting multiple languages. ADDITIONAL-LANGS and EXCLUDED-LANGS modify the candidate list. PROMPT, REQUIRE-MATCH, and INITIAL-INPUT are passed through to ~completing-read~.

#+findex: tlon-read-language
The function ~tlon-read-language~ is a simpler interface for reading a single language selection. It accepts the same filtering options as ~tlon-select-language~ but always returns a raw language name string.

#+findex: tlon-read-multiple-languages
The function ~tlon-read-multiple-languages~ prompts the user to select multiple languages using ~completing-read-multiple~, with an additional ="default"= option. When BABEL is non-~nil~, only project languages are offered.

#+findex: tlon-get-language-candidates
The function ~tlon-get-language-candidates~ returns the list of language candidates for completion. When BABEL is non-~nil~, it returns ~tlon-project-languages~; otherwise, all languages from ~tlon-languages-properties~. ADDITIONAL-LANGS are appended; EXCLUDED-LANGS are removed.

#+findex: tlon-get-formatted-languages
The function ~tlon-get-formatted-languages~ converts a SELECTION (string or list of strings) into the specified FORMAT (~code~ or ~locale~) by looking up each language in ~tlon-languages-properties~.

** JSON utilities
:PROPERTIES:
:CUSTOM_ID: h:json-utilities
:END:

#+findex: tlon-read-json
The function ~tlon-read-json~ reads and parses JSON data. When FILE is provided, it reads from that file; otherwise, it searches the current buffer for the first ={= character and parses from there. OBJECT-TYPE (default ~alist~), ARRAY-TYPE (default ~list~), and KEY-TYPE (default ~string~) control the Emacs Lisp representation of the parsed data.

#+findex: tlon-write-data
The function ~tlon-write-data~ serializes Emacs Lisp DATA as JSON and writes it to FILE, with pretty-printing applied.

#+findex: tlon-get-keys
The function ~tlon-get-keys~ returns the keys of a hash table DATA as a list. This is used when working with JSON data parsed as hash tables.

#+findex: tlon-edit-json-mapping
The function ~tlon-edit-json-mapping~ provides an interactive workflow for editing a flat JSON object in FILE. It reads the file as a hash table, prompts for an outer key (using OUTER-KEY-PROMPT) with completion against existing keys, prompts for a new value (using INNER-VALUE-PROMPT, pre-filled with the current value), updates the hash table, and writes the result back.

** JSON file path helpers
:PROPERTIES:

#+findex: tlon-get-language-code-from-name
The function ~tlon-get-language-code-from-name~ converts a language name to its ISO 639-1 code, signaling an error if the language is invalid.

#+findex: tlon-url-correspondences-file
The function ~tlon-url-correspondences-file~ returns the path to the =url-correspondences.json= file for the given LANGUAGE-CODE.

#+findex: tlon-tts-abbreviations-file
The function ~tlon-tts-abbreviations-file~ returns the path to the =abbreviations.json= file for the given LANGUAGE-CODE.

#+findex: tlon-tts-phonetic-replacements-file
The function ~tlon-tts-phonetic-replacements-file~ returns the path to the =phonetic-replacements.json= file for the given LANGUAGE-CODE.

#+findex: tlon-tts-phonetic-transcriptions-file
The function ~tlon-tts-phonetic-transcriptions-file~ returns the path to the =phonetic-transcriptions.json= file for the given LANGUAGE-CODE.

** Org mode integration
:PROPERTIES:
:CUSTOM_ID: h:org-integration
:END:

#+findex: tlon-make-tag-replace-pattern
The function ~tlon-make-tag-replace-pattern~ constructs a replacement pattern for a PAIR of Org mode tags (a cons cell of old and new tag strings). The optional FORMAT argument controls the pattern template; it defaults to ="%s%%s%s"=, which wraps the tags around a =%s= placeholder.

#+findex: tlon-get-or-set-org-var
The function ~tlon-get-or-set-org-var~ returns the value of VAR if it is already set; otherwise, it sets VAR to the file containing the Org heading with the given ID and returns that file path. This implements lazy initialization for variables that point to specific Org headings.

#+findex: tlon-get-file-with-org-id
The function ~tlon-get-file-with-org-id~ returns the file containing the Org heading with the specified ID. It first tries ~org-roam-id-find~ and falls back to ~org-id-find-id-file~.

#+findex: tlon-ensure-org-mode
The function ~tlon-ensure-org-mode~ signals a user error if the current buffer is not in ~org-mode~ or a derived mode. Use this as a guard at the beginning of functions that require Org mode.

** Transient menu utilities
:PROPERTIES:
:CUSTOM_ID: h:transient-utilities
:END:

These helper functions are used to build transient menu infixes and readers throughout the ~tlon~ package.

#+findex: tlon-transient-read-symbol-choice
The function ~tlon-transient-read-symbol-choice~ prompts the user with PROMPT to select from CHOICES (a list of symbols) and returns the selected symbol. It is used as a ~:reader~ function in transient infix definitions.

#+findex: tlon-transient-read-number-choice
The function ~tlon-transient-read-number-choice~ prompts the user to select from CHOICES (a list of numbers) and returns the selected number.

#+findex: tlon-transient-read-string-choice
The function ~tlon-transient-read-string-choice~ prompts the user to select from CHOICES (a list of strings) and returns the selected string with text properties stripped.

#+findex: tlon-transient-toggle-variable-value
The function ~tlon-transient-toggle-variable-value~ toggles a boolean variable VAR-NAME between ~nil~ and ~t~. It is used as the ~:reader~ for debug and other toggle infixes.

** String and number formatting
:PROPERTIES:
:CUSTOM_ID: h:formatting
:END:

#+findex: tlon-concatenate-list
The function ~tlon-concatenate-list~ joins the elements of LIST into a human-readable string. If the list has more than one element, it separates them with commas and adds a language-appropriate conjunction (e.g., "and" in English, "y" in Spanish) before the last element. The conjunction is determined by the language of the current file.

#+findex: tlon-get-delimited-region-pos
The function ~tlon-get-delimited-region-pos~ returns a cons cell of (BEGIN-POS . END-POS) for the region delimited by BEGIN and END regular expressions. If END is ~nil~, BEGIN is used as both the opening and closing delimiter. When EXCLUDE-DELIMS is non-~nil~, the delimiter lines themselves are excluded from the returned positions.

#+findex: tlon-get-separator
The function ~tlon-get-separator~ returns the thousands or decimal separator for a LANGUAGE. TYPE must be the symbol ~thousands~ or ~decimal~. If LANGUAGE is ~nil~, the language of the current file is used.

#+findex: tlon-get-decimal-separator
The function ~tlon-get-decimal-separator~ is a convenience wrapper that returns the decimal separator for LANGUAGE (or the current file's language).

#+findex: tlon-get-thousands-separator
The function ~tlon-get-thousands-separator~ is a convenience wrapper that returns the thousands separator for LANGUAGE (or the current file's language).

#+findex: tlon-get-number-separator-pattern
The function ~tlon-get-number-separator-pattern~ returns a regular expression matching numbers formatted with locale-specific THOUSANDS and DECIMAL separators. If these are not provided, they are inferred from the language of the current file (or LANG). When BOUNDED is non-~nil~, word boundary anchors are added.

#+findex: tlon-string-to-number
The function ~tlon-string-to-number~ converts a STRING containing a formatted number (with locale-specific THOUSANDS and DECIMAL separators) into an Emacs Lisp number. Separators are inferred from the current file's language if not provided.

** Git state checks
:PROPERTIES:
:CUSTOM_ID: h:git-checks
:END:

#+findex: tlon-check-branch
The function ~tlon-check-branch~ verifies that the current Git branch in REPO is BRANCH. It signals a user error if the check fails.

#+findex: tlon-check-file
The function ~tlon-check-file~ verifies that the current buffer's file matches the file expected from the currently clocked Org task. If ORIGINAL is non-~nil~, it checks against the original file path; otherwise against the translation file path.

#+findex: tlon-check-staged-or-unstaged
The function ~tlon-check-staged-or-unstaged~ returns ~t~ if FILE has any staged or unstaged modifications in its Git repository.

#+findex: tlon-ensure-no-uncommitted-changes
The function ~tlon-ensure-no-uncommitted-changes~ signals a user error if there are uncommitted changes in the repository of FILE, excluding changes to FILE itself. This is used before committing to ensure a clean repository state.

#+findex: tlon-check-file-title-match
The function ~tlon-check-file-title-match~ verifies that the file name of FILE matches the slugified version of its YAML =title= metadata field. This check is no longer run automatically, as the backend now handles it.

#+findex: tlon-check-file-type-match
The function ~tlon-check-file-type-match~ verifies that FILE is located in a bare directory that corresponds to its YAML =type= metadata field (e.g., an article should be in the =articles= directory or its translation).

#+findex: tlon-check-image-alt-text
The function ~tlon-check-image-alt-text~ scans the current buffer for images missing alt text (matched by ~tlon-md-image-sans-alt~) and signals a user error at the first occurrence, suggesting the use of ~tlon-md-insert-alt-text~ to fix them.

* Variables
:PROPERTIES:
:CUSTOM_ID: h:variables
:END:

This section documents the major public variables and constants defined in =tlon-core.el=. Internal data constants (such as ~tlon-core-bare-dirs~, ~tlon-authorship-pattern~, ~tlon-figure-names~, etc.) are used primarily by other functions and are not detailed here individually.

** Package and file paths
:PROPERTIES:
:CUSTOM_ID: h:package-paths
:END:

#+vindex: tlon-package-dir
The constant ~tlon-package-dir~ holds the directory where the ~tlon~ package is installed (under =elpaca/repos/tlon/= within ~user-emacs-directory~).

#+vindex: tlon-refs-dir
The variable ~tlon-refs-dir~ holds the directory of the =babel-refs= repository, as obtained from ~paths-dir-babel-refs~.

#+vindex: tlon-bibtex-dir
The variable ~tlon-bibtex-dir~ holds the =bib/= subdirectory within ~tlon-refs-dir~, where BibTeX files are stored.

#+vindex: tlon-json-dir
The variable ~tlon-json-dir~ holds the =json/= subdirectory within ~tlon-refs-dir~, where JSON data files are stored.

#+vindex: tlon-file-fluid
The variable ~tlon-file-fluid~ holds the path to =fluid.bib=, the actively maintained bibliography file.

#+vindex: tlon-file-stable
The variable ~tlon-file-stable~ holds the path to =stable.bib=, the stable (less frequently changed) bibliography file.

#+vindex: tlon-file-db
The variable ~tlon-file-db~ holds the path to =db.bib=, the database bibliography file.

#+vindex: tlon-file-db-upstream
The variable ~tlon-file-db-upstream~ holds the path to =db-upstream.bib=, the upstream database bibliography file.

#+vindex: tlon-bibliography-files
The variable ~tlon-bibliography-files~ holds a list of the three main Tlon bibliography file paths (~tlon-file-fluid~, ~tlon-file-stable~, and ~tlon-file-db~).

** Repository registry
:PROPERTIES:
:CUSTOM_ID: h:repo-registry
:END:

#+vindex: tlon-repos
The constant ~tlon-repos~ is the central registry of all Tlon repositories. Each entry is a plist with properties such as ~:name~, ~:abbrev~, ~:project~, ~:subproject~, ~:type~, ~:subtype~, ~:language~, ~:dir~, ~:url~, ~:key~, and ~:help~. The ~:type~ property classifies repositories as ~content~, ~meta~, ~development~, ~docs~, ~misc~, or ~test~. The ~:subtype~ property further distinguishes between ~translations~, ~originals~, ~biblio~, ~audio~, ~meetings~, ~clock~, ~issues~, and ~emacs~.

** User registry
:PROPERTIES:
:CUSTOM_ID: h:user-registry
:END:

#+vindex: tlon-users
The variable ~tlon-users~ holds a list of plists representing Tlon team members. Each entry has properties ~:name~, ~:git~ (Git author name), ~:github~ (GitHub username), ~:nickname~, and optionally ~:substitute~ (the GitHub username of the user who stands in for this person when needed).

** Language and locale data
:PROPERTIES:
:CUSTOM_ID: h:language-data
:END:

#+vindex: tlon-project-languages
The constant ~tlon-project-languages~ holds the list of all languages in the Babel project (e.g., ="arabic"=, ="spanish"=, ="english"=, etc.).

#+vindex: tlon-project-target-languages
The constant ~tlon-project-target-languages~ holds ~tlon-project-languages~ minus ="english"=, representing the languages into which content is translated.

#+vindex: tlon-languages-properties
The constant ~tlon-languages-properties~ is a comprehensive table of language properties. Each entry is a plist with ~:name~, ~:standard~ (the standardized BibTeX language name), ~:code~ (ISO 639-1), ~:locale~, and optionally ~:iso-639-2~. This table covers over 80 languages and is used by the language selection and validation functions.

#+vindex: tlon-translation-language
The variable ~tlon-translation-language~ holds the current target translation language as an ISO 639-1 code. The default is ="es"= (Spanish). It is set by ~tlon-set-translation-language~ ([[info:tlon][tlon]]).

#+vindex: tlon-language-separators
The constant ~tlon-language-separators~ maps language codes to their thousands and decimal separator characters.

#+vindex: tlon-yaml-delimiter
The constant ~tlon-yaml-delimiter~ holds the string ="---\n"=, used to delimit YAML front matter in Markdown files.

#+vindex: tlon-max-abstract-length
The constant ~tlon-max-abstract-length~ specifies the maximum length (300 words) for an article abstract.

#+vindex: tlon-min-abstract-length
The constant ~tlon-min-abstract-length~ specifies the minimum length (45 words) for an article abstract.

#+vindex: tlon-synopsis-length
The constant ~tlon-synopsis-length~ specifies the maximum length (2000 words) for a synopsis.

#+vindex: tlon-site-data
The constant ~tlon-site-data~ maps language codes to site names and URLs for the Uqbar sites.
