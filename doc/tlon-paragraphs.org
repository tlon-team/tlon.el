This manual describes the features and customization options for the Emacs Lisp file =tlon-paragraphs.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

The =tlon-paragraphs= module provides paragraph-related functionality for
Tlon content files.  It can count paragraphs, display corresponding
paragraph pairs between a file and its counterpart (original and
translation), align paragraph counts using an AI agent, navigate to a
specific paragraph by number, and show a paragraph indicator in the mode
line.  The module operates on the main content area of Markdown files,
automatically excluding YAML front matter and local variables sections.

* User options
:PROPERTIES:
:CUSTOM_ID: h:user-options
:END:

** ~tlon-paragraphs-align-with-ai-model~
:PROPERTIES:
:CUSTOM_ID: h:tlon-paragraphs-align-with-ai-model
:END:
#+vindex: tlon-paragraphs-align-with-ai-model

Model to use for aligning paragraphs.  The value is a cons cell whose car
is the backend and whose cdr is the model itself.  See
~gptel-extras-ai-models~ for available options.  If nil, do not use a
different model for aligning paragraphs.

- Default :: =("Claude" . claude-sonnet-4-20250514)=
- Type :: =(cons string symbol)=

** ~tlon-paragraphs-mode-line-format~
:PROPERTIES:
:CUSTOM_ID: h:tlon-paragraphs-mode-line-format
:END:
#+vindex: tlon-paragraphs-mode-line-format

Format for the paragraph indicator in the mode line.  The format receives
two integers: CURRENT paragraph and TOTAL paragraphs.

- Default :: =" ¶ %d/%d"=
- Type :: =string=

** Counting paragraphs
:PROPERTIES:
:CUSTOM_ID: h:counting-paragraphs
:END:

*** ~tlon-count-paragraphs~
:PROPERTIES:
:CUSTOM_ID: h:tlon-count-paragraphs
:END:
#+findex: tlon-count-paragraphs

Count paragraphs in the active region or in a file.  If the region is
active, count paragraphs in it.  When called with START and END, count
paragraphs in that range.  If called interactively without an active
region, prompt for a file and count its paragraphs.

*** ~tlon-get-number-of-paragraphs~
:PROPERTIES:
:CUSTOM_ID: h:tlon-get-number-of-paragraphs
:END:
#+findex: tlon-get-number-of-paragraphs

Return the number of paragraphs between START and END.  START and END are
buffer positions.  If START is nil, use the beginning of the main content
(after the YAML front matter).  If END is nil, use the end of the main
content (before the local variables section).  Exclude front matter and
local variables sections.

** Displaying corresponding paragraphs
:PROPERTIES:
:CUSTOM_ID: h:displaying-corresponding-paragraphs
:END:

*** ~tlon-display-corresponding-paragraphs~
:PROPERTIES:
:CUSTOM_ID: h:tlon-display-corresponding-paragraphs
:END:
#+findex: tlon-display-corresponding-paragraphs

Display PAIRS-OR-FN of corresponding paragraphs in parallel.
PAIRS-OR-FN can be either the output of ~tlon-get-corresponding-paragraphs~
or the function itself (the interactive default).

The command creates a =/Paragraph Pairs/= buffer showing each original
paragraph paired with its translation counterpart, separated by a
horizontal rule.  Missing paragraphs are indicated with
="[Missing paragraph]"=.  If ~tlon-get-corresponding-paragraphs~ signals
a paragraph mismatch error, the buffer is still displayed so you can
inspect the discrepancies.

** AI-powered paragraph alignment
:PROPERTIES:
:CUSTOM_ID: h:ai-alignment
:END:

*** ~tlon-paragraphs-align-with-ai~
:PROPERTIES:
:CUSTOM_ID: h:tlon-paragraphs-align-with-ai
:END:
#+findex: tlon-paragraphs-align-with-ai

Use AI to fix paragraph count mismatch between FILE and its counterpart.
If FILE is nil, prompt for it.

The command finds the counterpart, compares the number of paragraphs, and
if they differ, provides the AI agent with a paragraph-by-paragraph
comparison and instructs it to edit the translation file so that its
paragraph count matches the original.  The AI is equipped with the
=edit_file=, =apply_diff=, and =replace_file_contents= tools.  The model
used is determined by ~tlon-paragraphs-align-with-ai-model~
([[#h:tlon-paragraphs-align-with-ai-model][user option]]).

If the paragraph counts already match, it simply reports this.

** Navigation
:PROPERTIES:
:CUSTOM_ID: h:navigation
:END:

*** ~tlon-goto-paragraph~
:PROPERTIES:
:CUSTOM_ID: h:tlon-goto-paragraph
:END:
#+findex: tlon-goto-paragraph

Go to paragraph N in the main content.  If N is nil, prompt for it with
default as the current paragraph.  N is 1-based.  Signal an error if
there are no paragraphs.

** Mode line indicator
:PROPERTIES:
:CUSTOM_ID: h:mode-line-indicator
:END:

*** ~tlon-paragraphs-mode-line-mode~
:PROPERTIES:
:CUSTOM_ID: h:tlon-paragraphs-mode-line-mode
:END:
#+findex: tlon-paragraphs-mode-line-mode

Minor mode that shows the paragraph indicator in the mode line for the
current buffer (e.g., =¶ 3/12=), where the first number is the paragraph
at point and the second is the total number of paragraphs in the main
content.  The count excludes YAML front matter and the local variables
section.  The indicator is hidden when point is outside the main content.

Toggle with =M-x tlon-paragraphs-mode-line-mode=, or enable automatically
by adding it to ~markdown-mode-hook~.

The format of the indicator is controlled by
~tlon-paragraphs-mode-line-format~
([[#h:tlon-paragraphs-mode-line-format][user option]]).

If you use =doom-modeline=, enabling ~tlon-paragraphs-mode-line-mode~
automatically adds a custom segment with the indicator and switches the
current buffer to a modeline layout that includes it.  No extra setup is
required.  Disabling the mode restores the default =doom-modeline= layout
for the buffer.

** Transient menu
:PROPERTIES:
:CUSTOM_ID: h:tlon-paragraphs-menu
:END:
#+findex: tlon-paragraphs-menu

The command ~tlon-paragraphs-menu~ activates a =transient= menu interface
for paragraph-related operations.  It provides the following groups:

- *Commands*:
  - =p= :: Toggle paragraphs in mode line (~tlon-paragraphs-mode-line-mode~)
  - =a= :: Align paragraphs (~tlon-paragraphs-align-with-ai~)
  - =c= :: Count paragraphs (~tlon-count-paragraphs~)
  - =g= :: Go to paragraph (~tlon-goto-paragraph~)
  - =d= :: Display corresponding paragraphs (~tlon-display-corresponding-paragraphs~)
- *Options*:
  - =-a= :: Align paragraphs model

** Paragraph iteration

:CUSTOM_ID: h:paragraph-iteration
:END:

*** ~tlon-with-paragraphs~
:PROPERTIES:
:CUSTOM_ID: h:tlon-with-paragraphs
:END:
#+findex: tlon-with-paragraphs

Execute FN for each paragraph in FILE.  If FN is nil, return the
paragraph contents.  If RETURN-POSITIONS is non-nil, return list of
=(start . end)= positions.  Otherwise, return list of FN's results for
each paragraph.  If FILE is nil, use the current buffer.  If
INCLUDE-METADATA is non-nil, include the YAML front matter (metadata) in
the paragraph list; otherwise, exclude it.

The function moves past the YAML front matter, iterates through
paragraphs using =markdown-forward-paragraph=, skips whitespace-only
regions, and collects results in order.

*** ~tlon-get-corresponding-paragraphs~
:PROPERTIES:
:CUSTOM_ID: h:tlon-get-corresponding-paragraphs
:END:
#+findex: tlon-get-corresponding-paragraphs

Return pairs of paragraphs between TRANSLATION and its ORIGINAL.  Signals
an error if files have different number of paragraphs, and displays the
paragraphs in a buffer only in that case.  If ORIGINAL is nil, infer it
from TRANSLATION.

*** ~tlon-paragraph-files-are-aligned-p~
:PROPERTIES:
:CUSTOM_ID: h:tlon-paragraph-files-are-aligned-p
:END:
#+findex: tlon-paragraph-files-are-aligned-p

Return t iff FILE and COUNTERPART have the same number of paragraphs.

** Doom modeline integration
:PROPERTIES:
:CUSTOM_ID: h:doom-modeline-integration
:END:

When =doom-modeline= is loaded, the module registers a custom
=tlon-paragraph= segment that calls ~tlon-paragraphs-mode-line-string~
whenever ~tlon-paragraphs-mode-line-mode~ is active.  To include the
segment in your modeline without loading =tlon-paragraphs= at startup,
define a lightweight shim in your config:

#+begin_src emacs-lisp
(with-eval-after-load 'doom-modeline
  (unless (fboundp 'doom-modeline-segment-tlon-paragraph)
    (doom-modeline-def-segment tlon-paragraph
      (when (and (featurep 'tlon-paragraphs)
                 (bound-and-true-p tlon-paragraphs-mode-line-mode)
                 (fboundp 'tlon-paragraphs-mode-line-string))
        (tlon-paragraphs-mode-line-string)))))
#+end_src

Then you can safely add =tlon-paragraph= to your =doom-modeline-def-modeline=
spec.  The segment will remain empty until =tlon-paragraphs= is loaded and the
minor mode is enabled.
