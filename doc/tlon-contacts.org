This manual describes the features and customization options for the Emacs Lisp file =tlon-contacts.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

The =tlon-contacts= module manages contacts stored as headings in an Org mode file.  Each contact has a role (e.g., "creator", "translator") that determines which properties are available for editing.  The module provides commands for creating contacts, editing their properties (with completion where applicable), copying property values, and searching contacts.  A =transient= menu with dynamically generated per-property edit commands is included.

The module depends on =tlon-core=, =transient=, =org=, =org-id=, and =org-contacts=.

* User options
:PROPERTIES:
:CUSTOM_ID: h:user-options
:END:

This module does not define any =defcustom= user options.  Its behavior is governed by the constants described below.

** Constants
:PROPERTIES:
:CUSTOM_ID: h:constants
:END:

#+vindex: tlon-contacts-id
- ~tlon-contacts-id~ :: The Org mode ID (="F874E332-47AF-436F-997E-1A6791DEE0BE"=) identifying the main heading under which contacts are stored.  This ID is used to locate the contacts section in the Org file.

#+vindex: tlon-contacts-properties
- ~tlon-contacts-properties~ :: Alist mapping roles to the Org properties associated with each role.  For example, the ="creator"= role has properties ="EMAIL"=, ="URL"=, ="GENDER"=, ="ENTITY"=, and ="UNIVERSAL-CONSENT"=, while the ="translator"= role has ="EMAIL"=, ="GENDER"=, and ="LANGUAGE"=.  This structure determines which properties are prompted for when editing a contact.

#+vindex: tlon-contacts-property-values
- ~tlon-contacts-property-values~ :: List defining each editable property along with its transient menu shortcut and completion candidates.  Each element has the form =(PROPERTY :shortcut SHORTCUT :candidates COMPLETION-LIST)=.  If COMPLETION-LIST is non-nil, ~completing-read~ is used when editing that property; otherwise, ~read-string~ is used.  The ="LANGUAGE"= property dynamically fetches candidates from ~tlon-get-language-candidates~.

** Variables
:PROPERTIES:
:CUSTOM_ID: h:variables
:END:

#+vindex: tlon-contacts-file
- ~tlon-contacts-file~ :: Path to the Org file containing the contacts.  This variable is set dynamically by ~tlon-contacts-get-file~ using ~tlon-contacts-id~ and should not be set manually.

* Commands
:PROPERTIES:
:CUSTOM_ID: h:commands
:END:

** Creating contacts
:PROPERTIES:
:CUSTOM_ID: h:creating-contacts
:END:

*** ~tlon-contacts-create~
:PROPERTIES:
:CUSTOM_ID: h:tlon-contacts-create
:END:
#+findex: tlon-contacts-create

Create a new contact entry in the contacts Org file.  The command performs the following steps:

1. Open the contacts file (via ~tlon-contacts-get-file~).
2. Navigate to the main contacts heading using ~tlon-contacts-id~.
3. Insert a new level 2 heading at the end of the subtree.
4. Prompt for first and last name and insert them in ="Last, First"= format.
5. Call ~tlon-contacts-edit-properties~ to set the role and initial properties.
6. Sort the headings alphabetically via ~tlon-sort-headings~.

** Editing properties
:PROPERTIES:
:CUSTOM_ID: h:editing-properties
:END:

*** ~tlon-contacts-edit-properties~
:PROPERTIES:
:CUSTOM_ID: h:tlon-contacts-edit-properties
:END:
#+findex: tlon-contacts-edit-properties

Edit the Org properties of the contact at point.  If the contact does not yet have a ="ROLE"= property, prompt you to select one.  If an optional ROLE argument is provided and it does not match the existing role, signal an error.  For each property defined for the role in ~tlon-contacts-properties~, prompt you for a value (using completion if candidates are defined in ~tlon-contacts-property-values~, or free-text input otherwise).  Save the buffer after editing.

*** ~tlon-contacts-edit-one-property~
:PROPERTIES:
:CUSTOM_ID: h:tlon-contacts-edit-one-property
:END:
#+findex: tlon-contacts-edit-one-property

Edit a single PROPERTY of the contact at point.  Prompt you to select the property from ~tlon-contacts-property-values~, then prompt for the new value using completion if candidates are available.  Save the buffer and display a confirmation message.

*** Dynamically defined edit commands
:PROPERTIES:
:CUSTOM_ID: h:dynamic-edit-commands
:END:
#+findex: tlon-contacts-define-property-edit-commands

The macro ~tlon-contacts-define-property-edit-commands~ automatically generates one interactive command per property listed in ~tlon-contacts-property-values~.  For example, for the ="EMAIL"= property it defines ~tlon-contacts-edit-email~.  Each generated command calls ~tlon-contacts-edit-one-property~ with the corresponding property name.  The macro also defines the ~tlon-contacts-menu~ transient prefix ([[#h:tlon-contacts-menu][transient menu]]).

** Copying property values
:PROPERTIES:
:CUSTOM_ID: h:copying-properties
:END:

*** ~tlon-contacts-copy-property-value~
:PROPERTIES:
:CUSTOM_ID: h:tlon-contacts-copy-property-value
:END:
#+findex: tlon-contacts-copy-property-value

Copy the value of a property of the contact at point to the kill ring.  If PROP is =nil=, prompt you to select a property from the non-empty properties of the contact.  Display a confirmation message.

** Transient menu
:PROPERTIES:
:CUSTOM_ID: h:tlon-contacts-menu
:END:
#+findex: tlon-contacts-menu

The command ~tlon-contacts-menu~ activates a =transient= menu for contact management.  This menu is defined by the ~tlon-contacts-define-property-edit-commands~ macro and includes:

- General commands:
  - =S= :: search contacts (=org-contacts=)
  - =c= :: create contact (~tlon-contacts-create~)
  - =e= :: edit all properties (~tlon-contacts-edit-properties~)
  - =y= :: copy property value (~tlon-contacts-copy-property-value~)
- Edit specific properties:
  - One entry per property in ~tlon-contacts-property-values~, using the configured =:shortcut= key (e.g., =m= for ="email"=, =g= for ="gender"=, =l= for ="language"=).

* Functions
:PROPERTIES:
:CUSTOM_ID: h:functions
:END:

#+findex: tlon-contacts-get-file
- ~tlon-contacts-get-file~ :: Return the path to the contacts Org file.  Uses ~tlon-get-or-set-org-var~ to locate the file containing ~tlon-contacts-id~ and caches it in ~tlon-contacts-file~.

#+findex: tlon-contacts-insert-name
- ~tlon-contacts-insert-name~ :: Prompt for first and last name and insert them at point in ="Last, First"= format.  Save the buffer afterwards.

#+findex: tlon-contacts-get-property-value
- ~tlon-contacts-get-property-value~ :: Return the value of PROP for the contact at point.  If PROP is =nil=, prompt you to select from the non-empty properties (via ~tlon-contacts-get-nonempty-properties~).

#+findex: tlon-contacts-get-property-value-of-contact
- ~tlon-contacts-get-property-value-of-contact~ :: Prompt you to select a contact via =org-contacts=, then return the value of PROP for that contact.

#+findex: tlon-contacts-get-nonempty-properties
- ~tlon-contacts-get-nonempty-properties~ :: Return an alist of the non-empty properties for the contact at point, based on the contact's role and the properties defined in ~tlon-contacts-properties~.

#+findex: tlon-contacts-get-contact-name
- ~tlon-contacts-get-contact-name~ :: Return a cons cell =(FIRST . LAST)= from the heading of the contact at point.  Assumes the heading format is ="Last, First"=.  Signals an error if the heading does not match this format.

#+findex: tlon-contacts-get-role
- ~tlon-contacts-get-role~ :: Return the value of the ="ROLE"= property of the contact at point.

#+findex: tlon-contacts-select-role
- ~tlon-contacts-select-role~ :: Prompt you to select a role from the keys of ~tlon-contacts-properties~.  Returns the selected role string.
