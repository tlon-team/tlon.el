This manual describes the features and customization options for the Emacs Lisp file =tlon-counterpart.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

The =tlon-counterpart= module provides functionality for finding, opening,
and comparing "counterpart" files in the Tlon translation workflow.  A
counterpart is the original version of a translated document, or a
translation in a specific language if the current file is the original.
The module resolves counterparts by looking up YAML =key= fields and
mapping between original and translation BibTeX keys via the
bibliography.  It also provides commands for translating relative Markdown
links within translation files, invalidating internal caches, reporting
missing counterparts, and opening counterpart files with automatic
paragraph-position alignment.

** Finding counterparts
:PROPERTIES:
:CUSTOM_ID: h:finding-counterparts
:END:

*** ~tlon-get-counterpart~
:PROPERTIES:
:CUSTOM_ID: h:tlon-get-counterpart
:END:
#+findex: tlon-get-counterpart

Get the counterpart file of FILE.  If FILE is nil, return the counterpart
of the file visited by the current buffer.

A file's counterpart is the original if it is a translation, and a
translation into some language if it is the original.  If the latter, you
are prompted for a language, unless TARGET-LANGUAGE-CODE is provided.

- If the file is in a =translations= repository, the command reads the
  YAML =key= (a translation BibTeX key), maps it to the original key via
  the bibliography, and searches across all English =originals= repos.
  For tags and authors, the =original_path= field is used instead.
- If the file is in an =originals= repository, you are prompted for a
  target language.  The command enumerates translation repos for that
  language and resolves the translation by mapping YAML keys back through
  the bibliography.  For tags and authors, it searches translation
  metadata for entries whose =original_path= matches the basename.
- When multiple matches are found, you are prompted to disambiguate.

*** ~tlon-get-counterpart-repo~
:PROPERTIES:
:CUSTOM_ID: h:tlon-get-counterpart-repo
:END:
#+findex: tlon-get-counterpart-repo

Get the counterpart repo of FILE.  A file's counterpart repo is the repo
of that file's counterpart.  If FILE is nil, return the counterpart repo
of the file visited by the current buffer.  It identifies the source
repository, its subproject, determines the counterpart language
([[#h:tlon-get-counterpart-language][~tlon-get-counterpart-language~]]), and looks up the
repository matching the subproject and counterpart language.

*** ~tlon-get-counterpart-dir~
:PROPERTIES:
:CUSTOM_ID: h:tlon-get-counterpart-dir
:END:
#+findex: tlon-get-counterpart-dir

Get the counterpart directory of FILE.  A file's counterpart directory is
the directory of that file's counterpart.  For example, the counterpart
directory of =~/Dropbox/repos/uqbar-es/autores/= is
=~/Dropbox/repos/uqbar-en/authors/=.

If FILE is nil, use the file visited by the current buffer.  If
TARGET-LANGUAGE-CODE is provided, use it to determine the counterpart repo
and target language, avoiding prompts.  Otherwise, determine them
automatically, which may prompt if the source file is an original.  For
counterpart repos not in the ="uqbar"= subproject, this function returns
the repository root.

*** ~tlon-get-counterpart-language~
:PROPERTIES:
:CUSTOM_ID: h:tlon-get-counterpart-language
:END:
#+findex: tlon-get-counterpart-language

Return the language code of the counterpart of REPO.  If the repository
language is ="en"= (English), you are prompted to select a target language
from ~tlon-project-target-languages~.  If the repository language is one
of the target languages, return ="en"=.  Signal an error if the language
is not recognized.

*** ~tlon-get-content-subtype~
:PROPERTIES:
:CUSTOM_ID: h:tlon-get-content-subtype
:END:
#+findex: tlon-get-content-subtype

For repo of FILE, get the value of its =:subtype= property (=originals=
or =translations=).  If FILE is nil, return the counterpart of the file
visited by the current buffer.  Signal an error if the repo is not of type
=content=.

*** ~tlon-get-image-counterpart~
:PROPERTIES:
:CUSTOM_ID: h:tlon-get-image-counterpart
:END:
#+findex: tlon-get-image-counterpart

Return the counterpart path for an image PATH.  This function translates
components of the path (repo, directories, filename) from the language of
FILE to TARGET-LANG.  It maps the repo, image directory, bare directory,
slug, and figure name component to the target language equivalents.

** Opening counterparts
:PROPERTIES:
:CUSTOM_ID: h:opening-counterparts
:END:

*** ~tlon-open-counterpart~
:PROPERTIES:
:CUSTOM_ID: h:tlon-open-counterpart
:END:
#+findex: tlon-open-counterpart

Open the counterpart of FILE and move point to the matching position.  If
FILE is nil, open the counterpart of the file visited by the current
buffer.  If called with a prefix argument, or OTHER-WIN is non-nil, open
the counterpart in the other window.

The command saves the current buffer (unless FILE is given), finds the
counterpart via ~tlon-get-counterpart~, calculates the current paragraph
number, and then positions the cursor at the corresponding paragraph in
the counterpart buffer, after skipping the YAML front matter.

*** ~tlon-open-counterpart-dwim~
:PROPERTIES:
:CUSTOM_ID: h:tlon-open-counterpart-dwim
:END:
#+findex: tlon-open-counterpart-dwim

Open the counterpart of FILE as appropriate.  If called in
=markdown-mode=, open FILE's counterpart.  If called in =dired-mode=,
jump to its counterpart's Dired buffer.

If FILE is nil, act on the file at point or visited in the current buffer.
If called with a prefix ARG, open the counterpart in the other window.

*** ~tlon-open-counterpart-in-other-window-dwim~
:PROPERTIES:
:CUSTOM_ID: h:tlon-open-counterpart-in-other-window-dwim
:END:
#+findex: tlon-open-counterpart-in-other-window-dwim

Convenience command that calls ~tlon-open-counterpart-dwim~ with
OTHER-WIN set to =t=, forcing the counterpart to open in another window.

** Translating relative links
:PROPERTIES:
:CUSTOM_ID: h:translating-relative-links
:END:

*** ~tlon-translate-relative-links~
:PROPERTIES:
:CUSTOM_ID: h:tlon-translate-relative-links
:END:
#+findex: tlon-translate-relative-links

Translate relative Markdown links in FILE.  If FILE is nil, use the file
visited by the current buffer, if available, else prompt for a file.

The command scans for relative links of the form =./filename.md= and
=../bare-dir/filename.md= that do not resolve to existing local files.
For each such link, it constructs the corresponding English path in the
originals repo using the appropriate bare directory mapping, obtains the
counterpart with ~tlon-get-counterpart~, and rewrites the link to point
to the translated relative path, preserving anchors.  A summary of the
number of translated links is displayed.

*** ~tlon-translate-relative-links-in-dired~
:PROPERTIES:
:CUSTOM_ID: h:tlon-translate-relative-links-in-dired
:END:
#+findex: tlon-translate-relative-links-in-dired

Translate relative Markdown links in marked FILES in Dired.  When called
interactively in Dired, operate on all marked files.  FILES, when
non-nil, should be a list of absolute file paths to process.  After
processing, a summary is displayed indicating how many files were
processed and how many errors occurred.  When errors occur, a dedicated
buffer listing the individual error messages is shown.

** Cache management
:PROPERTIES:
:CUSTOM_ID: h:cache-management
:END:

*** ~tlon-counterpart-invalidate-cache~
:PROPERTIES:
:CUSTOM_ID: h:tlon-counterpart-invalidate-cache
:END:
#+findex: tlon-counterpart-invalidate-cache

Invalidate the counterpart caches for REPO-DIR.  If REPO-DIR is nil,
invalidate caches for all repos.  Call this after creating or deleting
content files so that subsequent counterpart lookups reflect the new state
of the filesystem.

** Reporting
:PROPERTIES:
:CUSTOM_ID: h:reporting
:END:

*** ~tlon-counterpart-report-missing~
:PROPERTIES:
:CUSTOM_ID: h:tlon-counterpart-report-missing
:END:
#+findex: tlon-counterpart-report-missing

Report files missing counterparts between two languages for an entity
type.  You are prompted for an entity type (one of "articles", "authors",
or "tags") and two languages.  The command scans the current subproject's
repositories for those languages and reports files of the selected type
that have no counterpart in the other language.  The scan includes
Markdown files in immediate subdirectories only (non-recursive); files in
the repository root are ignored.

The results are shown in a dedicated buffer with two lists:

- Missing in LANG-B: files in LANG-A with no counterpart in LANG-B.
- Missing in LANG-A: files in LANG-B with no counterpart in LANG-A.

** Transient menu
:PROPERTIES:
:CUSTOM_ID: h:tlon-counterpart-menu
:END:
#+findex: tlon-counterpart-menu

The command ~tlon-counterpart-menu~ activates a =transient= menu interface
for counterpart operations.  It provides the following groups:

- *Navigation*:
  - =u= :: Visit counterpart (~tlon-open-counterpart-dwim~)
  - =H-u= :: Visit counterpart other window (~tlon-open-counterpart-in-other-window-dwim~)
  - =U= :: Open counterpart in Dired (~tlon-open-counterpart-in-dired~)
- *Links*:
  - =l= :: Translate relative links (~tlon-translate-relative-links~)
  - =L= :: Translate relative links in marked files (~tlon-translate-relative-links-in-dired~)
- *Metadata*:
  - =o= :: Set =original_path= (~tlon-yaml-insert-original-path~)
- *Report missing*:
  - =r= :: Report missing counterparts (~tlon-counterpart-report-missing~)

** Counterpart resolution
:PROPERTIES:
:CUSTOM_ID: h:counterpart-resolution
:END:

*** ~tlon-get-counterpart-in-translations~
:PROPERTIES:
:CUSTOM_ID: h:tlon-get-counterpart-in-translations
:END:
#+findex: tlon-get-counterpart-in-translations

Return the original counterpart of translation FILE.  For articles,
resolve by YAML key via the bibliography (translation key to original
key) and locate the original by key in the English originals repos.  For
tags and authors, require the YAML =original_path= field and resolve it
under the English counterpart directory derived from FILE.

*** ~tlon-get-counterpart-in-originals~
:PROPERTIES:
:CUSTOM_ID: h:tlon-get-counterpart-in-originals
:END:
#+findex: tlon-get-counterpart-in-originals

Return the translation counterpart of original FILE.
TARGET-LANGUAGE-CODE is the target translation language code.  If nil,
prompt.  For articles, resolve by YAML key across translation repos
(original key to translation file).  For tags and authors, search
translation metadata for entries whose =original_path= equals the
basename of FILE.

*** ~tlon-get-counterpart-key~
:PROPERTIES:
:CUSTOM_ID: h:tlon-get-counterpart-key
:END:
#+findex: tlon-get-counterpart-key

Return the bibliography key that is the counterpart of KEY.  If KEY
belongs to a translation entry, return the original key stored in its
="translation"= field.  Otherwise treat KEY as an original key and, when
LANGUAGE is non-nil, return the translation key for LANGUAGE via a direct
database lookup.  When no counterpart exists, return nil.

*** ~tlon-get-file-from-image~
:PROPERTIES:
:CUSTOM_ID: h:tlon-get-file-from-image
:END:
#+findex: tlon-get-file-from-image

Return the file path corresponding to image PATH.  For example, if PATH
is ="../uqbar-en/images/articles/on-caring/figure-1.jpg"=, the function
returns ="../uqbar-en/articles/on-caring.md"=.

*** ~tlon-is-between-paragraphs-p~
:PROPERTIES:
:CUSTOM_ID: h:tlon-is-between-paragraphs-p
:END:
#+findex: tlon-is-between-paragraphs-p

Return t iff point is right between two paragraphs.  Determined by
checking whether moving one character forward changes the paragraph count.
