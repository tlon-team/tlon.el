This manual describes the features and customization options for the Emacs Lisp file =tlon-fix.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

The =tlon-fix= module provides functions for applying automated and manual fixes to text content, particularly focusing on common typographical issues, formatting inconsistencies, and language-specific corrections within the Tlon project. It distinguishes between two categories:

- *Automatic fixes* (~tlon-autofix-*~): applied without user intervention, suitable for hooks and batch processing.
- *Manual fixes* (~tlon-manual-fix-*~): prompt you before each change via ~query-replace-regexp~, allowing selective application.

The module also provides heading hierarchy validation, unbalanced character checking, Internet Archive URL restoration, and language-specific translation fixes for French and Italian.

* User options
:PROPERTIES:
:CUSTOM_ID: h:user-options
:END:

This module does not define any user options (~defcustom~).

* Commands
:PROPERTIES:
:CUSTOM_ID: h:commands
:END:

** Automatic fix commands
:PROPERTIES:
:CUSTOM_ID: h:autofix-commands
:END:

These commands apply corrections automatically without prompting. They are suitable for use in hooks (such as ~after-save-hook~) or batch processing.

#+findex: tlon-autofix-all
~tlon-autofix-all~ executes all defined automatic fix functions in sequence on the current buffer:
- ~tlon-autofix-curly-quotes~
- ~tlon-autofix-footnote-punctuation~
- ~tlon-autofix-periods-in-headings~
- ~tlon-autofix-percent-signs~
- ~tlon-autofix-replace-thousands-separators~
- ~tlon-autofix-thin-spaces~
- ~tlon-autofix-superscripts~

It is designed to be run from ~after-save-hook~. To prevent infinite loops, it temporarily removes itself from the hook, saves the buffer, and then re-adds itself.

#+findex: tlon-autofix-curly-quotes
~tlon-autofix-curly-quotes~ replaces straight double quotes immediately preceding a square bracket with curly quotes when appropriate.

#+findex: tlon-autofix-footnote-punctuation
~tlon-autofix-footnote-punctuation~ moves footnote references (=[^N]=) to appear after adjacent punctuation marks (=,=, =.=, =:=, =;=, =?=, =!=) for English and Spanish text, as per common typographical conventions in those languages. It calls ~tlon-autofix-footnote-punctuation-amend~ afterwards to correct potential over-application issues with consecutive footnotes.

#+findex: tlon-autofix-periods-in-headings
~tlon-autofix-periods-in-headings~ removes trailing periods from Markdown headings (lines starting with =##= up to =######=).

#+findex: tlon-autofix-percent-signs
~tlon-autofix-percent-signs~ inserts a narrow no-break space (U+202F) before percent signs that immediately follow digits or closing parentheses.

#+findex: tlon-autofix-thin-spaces
~tlon-autofix-thin-spaces~ replaces thin spaces (U+2009) with narrow no-break spaces (U+202F). The two characters have the same width, but only the narrow no-break space prevents line breaks.

#+findex: tlon-autofix-superscripts
~tlon-autofix-superscripts~ converts caret-based superscript notation (e.g., =^1^=) into HTML =<sup>= tags (e.g., =<sup>1</sup>=).

#+findex: tlon-autofix-replace-thousands-separators
~tlon-autofix-replace-thousands-separators~ replaces existing thousands separators (commas or periods, depending on the language context determined by ~tlon-get-thousands-separator~) with the standard narrow no-break space (~tlon-default-thousands-separator~). It avoids making replacements within =<Math>= tags, links, and specific excluded phrases like ="80,000 Hours"=. You can optionally supply a SEPARATOR argument to override the detected one.

** Manual fix commands
:PROPERTIES:
:CUSTOM_ID: h:manual-fix-commands
:END:

These commands use ~query-replace-regexp~ to prompt you before making each change, allowing selective application.

#+findex: tlon-manual-fix-all
~tlon-manual-fix-all~ executes all defined manual fix functions sequentially, prompting you for confirmation before each potential change.

#+findex: tlon-manual-fix-em-dashes
~tlon-manual-fix-em-dashes~ prompts you to replace hyphens used as punctuation (between words, before/after spaces) with em dashes (=---=).

#+findex: tlon-manual-fix-number-ranges
~tlon-manual-fix-number-ranges~ prompts you to replace hyphens used between numbers (indicating numerical ranges) with en dashes (=--=).

#+findex: tlon-manual-fix-roman-numerals
~tlon-manual-fix-roman-numerals~ prompts you to wrap standalone Roman numerals (sequences of I, V, X, L, C, D, M found as whole words) in =<abbr>= tags for semantic markup.

#+findex: tlon-manual-fix-add-thousands-separators
~tlon-manual-fix-add-thousands-separators~ prompts you to insert the standard thousands separator (narrow no-break space) into numerals of four or more digits that currently lack separators. Unlike ~tlon-autofix-replace-thousands-separators~ ([[#h:autofix-commands][Automatic fix commands]]), which replaces existing separators, this command adds separators where they are missing. Manual confirmation is required because not all long numbers (like years or identifiers) should be separated.

#+findex: tlon-manual-fix-narrow-spaces
~tlon-manual-fix-narrow-spaces~ prompts you to insert a narrow no-break space (U+202F) between consecutive uppercase letters where the first is followed by a period (e.g., ="A.B."= becomes ="A. B."=).

#+findex: tlon-manual-fix-solo
~tlon-manual-fix-solo~ prompts you to replace the Spanish word ="solo"= (with accent) with ="solo"= (without accent), preserving case.

#+findex: tlon-manual-fix-podcast
~tlon-manual-fix-podcast~ prompts you to replace the English word ="podcast"= with the Spanish ="podcast"= (with accent), preserving case. This is useful because spell-checkers do not make the correct suggestion.

#+findex: tlon-manual-fix-emphasis
~tlon-manual-fix-emphasis~ prompts you to wrap text currently emphasized with Markdown italics (=*text*= or =_text_=) within =<emphasis level="moderate">= tags. It iterates through matches and asks for confirmation before each replacement.

#+findex: tlon-manual-fix-quote
~tlon-manual-fix-quote~ prompts you to wrap text enclosed in various quote marks or asterisks within =<q>= tags. It skips text already inside HTML/XML tags.

#+findex: tlon-manual-fix-foreign-words
~tlon-manual-fix-foreign-words~ is a placeholder command intended to prompt you to add =<lang>= tags around foreign words. The implementation is not yet complete.

** Heading hierarchy
:PROPERTIES:
:CUSTOM_ID: h:heading-hierarchy
:END:

#+findex: tlon-check-heading-hierarchy
~tlon-check-heading-hierarchy~ checks that Markdown heading levels do not skip levels. For example, a =##= followed by =####= (skipping =###=) is flagged. If a violation is found, point is moved to the first offending heading and a message reports the violation. Otherwise, a success message is displayed.

** Unbalanced character checking
:PROPERTIES:
:CUSTOM_ID: h:unbalanced-characters
:END:

#+findex: tlon-check-unbalanced-character
~tlon-check-unbalanced-character~ prompts you for a character and scans the current buffer for unbalanced instances of that character. It supports common pairs such as =()=, =[]=, ={}=, guillemets, curly quotes, angle brackets, and symmetric characters like straight quotes and asterisks. If an imbalance is found, point is moved to the first offending location and a message reports whether an opening or closing counterpart is missing.

#+findex: tlon-check-unbalanced-character-in-dir
~tlon-check-unbalanced-character-in-dir~ prompts you for a character and checks all regular files in the current directory for unbalanced instances. Results are written to a buffer named =*tlon-unbalanced*=. Returns non-nil if all files are balanced.

** Internet Archive URLs
:PROPERTIES:
:CUSTOM_ID: h:internet-archive-urls
:END:

#+findex: tlon-fix-internet-archive-urls
~tlon-fix-internet-archive-urls~ removes the =https://web.archive.org/web/TIMESTAMP/= prefix from Internet Archive URLs in the current buffer, restoring them to their original form. Reports the number of URLs fixed.

** Language-specific translation fixes
:PROPERTIES:
:CUSTOM_ID: h:language-specific-fixes
:END:

These commands apply predefined search-and-replace rules tailored for specific languages, fixing common issues in translated texts.

#+findex: tlon-fix-french-translation
~tlon-fix-french-translation~ applies the fixes defined in ~tlon-fix-french-translation~ (the constant) to the current buffer or marked Dired files. Fixes include: replacing hyphens with em dashes, standardizing list markers, correcting common phrases, adjusting spacing around French punctuation, replacing straight quotes with guillemets, and fixing footnote/sidenote marker syntax.

#+findex: tlon-fix-italian-translation
~tlon-fix-italian-translation~ applies the fixes defined in ~tlon-fix-italian-translation~ (the constant) to the current buffer or marked Dired files. Fixes include: replacing hyphens with em dashes, standardizing list markers, correcting common phrases, and fixing Markdown/HTML tag issues.

** Transient menu
:PROPERTIES:
:CUSTOM_ID: h:transient-menu
:END:

#+findex: tlon-fix-menu
~tlon-fix-menu~ activates a transient menu interface providing quick access to all fix commands, grouped into "Auto fix", "Manual fix", and "Misc" columns.

* Functions
:PROPERTIES:
:CUSTOM_ID: h:functions
:END:

** Core replacement functions
:PROPERTIES:
:CUSTOM_ID: h:core-replacement
:END:

#+findex: tlon-autofix
~tlon-autofix~ is the core automatic replacement function. It iterates through a REGEXP-LIST and replaces all occurrences of each regexp with NEWTEXT in the current buffer using ~replace-match~ within a ~re-search-forward~ loop. The buffer is widened before processing.

#+findex: tlon-manual-fix
~tlon-manual-fix~ is the core manual replacement function. It iterates through a REGEXP-LIST and uses ~query-replace-regexp~ to replace occurrences with NEWTEXT, prompting you for each match. If KEEP-CASE is non-nil, the case of the matched text is preserved.

** Thousands separator helpers
:PROPERTIES:
:CUSTOM_ID: h:thousands-separator-helpers
:END:

#+findex: tlon-number-separators-perform-replacements
~tlon-number-separators-perform-replacements~ performs the actual replacement of the language-specific thousands SEPARATOR with the default one (~tlon-default-thousands-separator~), but only if the match does not fall within any PROTECTED-RANGES.

#+findex: tlon-is-in-protected-range-p
~tlon-is-in-protected-range-p~ checks if a given text range (from START to END) overlaps with any of the ranges in PROTECTED-RANGES. Returns non-nil if there is an overlap.

** Translation fix helpers
:PROPERTIES:
:CUSTOM_ID: h:translation-fix-helpers
:END:

#+findex: tlon-fix-translation
~tlon-fix-translation~ applies language-specific fixes to the current file or marked Dired files. It looks up the fix list for LANG in ~tlon-fix-translations~ and calls ~tlon-fix-translation-in-file~ for each file.

#+findex: tlon-fix-translation-in-file
~tlon-fix-translation-in-file~ applies the language-specific search-and-replace pairs for LANG to the current buffer. Searches are case-sensitive.

** Footnote punctuation amendment
:PROPERTIES:
:CUSTOM_ID: h:footnote-punctuation-amendment
:END:

#+findex: tlon-autofix-footnote-punctuation-amend
~tlon-autofix-footnote-punctuation-amend~ corrects malformed patterns like =[[^1]^2]= or =[^[^1]2]= that can be produced by ~tlon-autofix-footnote-punctuation~ when processing consecutive footnotes, restoring them to the standard =[^1][^2]= format.

** Constants
:PROPERTIES:
:CUSTOM_ID: h:constants
:END:

#+vindex: tlon-fix-french-translation
~tlon-fix-french-translation~ is a constant alist of =(SEARCH-REGEXP . REPLACE-STRING)= pairs defining common fixes specific to French text.

#+vindex: tlon-fix-italian-translation
~tlon-fix-italian-translation~ is a constant alist of =(SEARCH-REGEXP . REPLACE-STRING)= pairs defining common fixes specific to Italian text.

#+vindex: tlon-fix-translations
~tlon-fix-translations~ is a constant alist mapping language codes (e.g., ="fr"=, ="it"=) to the variables containing their respective fix lists. Used by ~tlon-fix-translation~ to find the correct set of rules for a given language.
