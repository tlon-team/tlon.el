This manual describes the features and customization options for the Emacs Lisp file =tlon-meet.el=.

* Overview
:PROPERTIES:
:CUSTOM_ID: h:overview
:END:

=tlon-meet= provides tools for managing meetings within the Tlon workflow. It handles the full lifecycle of meetings: creating and visiting dedicated Forge issues for both individual and group meetings, linking discussions from other issues to meeting agendas, processing meeting recordings through transcription (via =whisperx=), AI-powered formatting and cleanup, and AI-powered summarization. It can also monitor recording directories for automatic processing of new files.

* User options
:PROPERTIES:
:CUSTOM_ID: h:user-options
:END:

** ~tlon-meet-recordings-directory~
:PROPERTIES:
:CUSTOM_ID: h:tlon-meet-recordings-directory
:END:
#+vindex: tlon-meet-recordings-directory

Directory where Google Meet recordings are stored.

The default value is =~/My Drive/Meet Recordings/=. The type is =directory=.

This directory is monitored for new recordings when ~tlon-meet-watch-recordings~ is active and ~tlon-default-conference-app~ is set to =meet=. It is also used as the default location when prompting for audio files.

** ~tlon-zoom-recordings-directory~
:PROPERTIES:
:CUSTOM_ID: h:tlon-zoom-recordings-directory
:END:
#+vindex: tlon-zoom-recordings-directory

Directory where Zoom recordings are stored.

The default value is =~/Documents/Zoom/=. The type is =directory=.

This directory is used when ~tlon-default-conference-app~ is set to =zoom=, both for file monitoring and as the default location when prompting for audio files.

** ~tlon-default-conference-app~
:PROPERTIES:
:CUSTOM_ID: h:tlon-default-conference-app
:END:
#+vindex: tlon-default-conference-app

Default conference app for meetings. This determines which recording directory is used for file monitoring and as the default directory when selecting audio files.

The default value is =meet=. Valid choices are =meet= (Google Meet) and =zoom= (Zoom).

:PROPERTIES:
:CUSTOM_ID: h:tlon-create-or-visit-meeting-issue

#+findex: tlon-create-or-visit-meeting-issue

Create or visit the Forge issue for a meeting. This is the main entry point for accessing meeting issues. You are prompted to select another team member (using completion from all known users) or ="group"=. If a person is selected, it delegates to ~tlon-create-or-visit-individual-meeting-issue~. *** ~tlon-create-or-visit-individual-meeting-issue~
:PROPERTIES:
:CUSTOM_ID: h:tlon-create-or-visit-individual-meeting-issue
:END:
#+findex: tlon-create-or-visit-individual-meeting-issue

Create or visit the Forge issue for a one-on-one meeting with PERSON on a given DATE. When called interactively, you are prompted for the person and for the date (via ~org-read-date~). The correct meeting repository is determined by ~tlon-get-meeting-repo~ based on the two participants.

:PROPERTIES:
:CUSTOM_ID: h:tlon-create-or-visit-group-meeting-issue

#+findex: tlon-create-or-visit-group-meeting-issue

Create or visit the Forge issue for a group meeting on DATE. When called interactively, you are prompted for the date. Uses the repository designated as =meetings-group= (via ~tlon-repo-lookup~).

** Discussion linking
:PROPERTIES:
:CUSTOM_ID: h:discussion-linking
:END:

*** ~tlon-discuss-issue-in-meeting~
:PROPERTIES:
:CUSTOM_ID: h:tlon-discuss-issue-in-meeting
:END:
#+findex: tlon-discuss-issue-in-meeting

Create a reminder to discuss the current issue in a meeting. This command must be invoked from a =forge-issue-mode= buffer. It creates cross-references between the current issue and the relevant meeting issue:

1. Gets the link to the current issue.
2. Switches to the meeting issue buffer (via ~tlon-create-or-visit-meeting-issue~).
3. Edits the last post in the meeting issue to append "- Discutir [link-to-original-issue]."
4. Switches back to the original issue and creates a new post: "A discutir en [link-to-meeting-issue]."

This encourages moving prolonged discussions from the issue tracker to in-person or call meetings.

** Recording processing
:PROPERTIES:
:CUSTOM_ID: h:recording-processing
:END:

*** ~tlon-meet-transcribe-audio~
:PROPERTIES:
:CUSTOM_ID: h:tlon-meet-transcribe-audio
:END:
#+findex: tlon-meet-transcribe-audio

Transcribe an audio or video file using =whisperx= with diarization and format the result using AI.

When called interactively, you are prompted to select an audio/video file and to confirm participants (nicknames are inferred from the filename when possible). A =*Diarization Output*= buffer is created and displayed to show progress.

The transcription pipeline works as follows:

1. Runs ~tlon-whisperx-diarize~ on the audio file with language hardcoded to ="es"=.
2. On success, deletes extraneous output files (=.vtt=, =.srt=, =.tsv=, =.json=) and keeps the =.txt= transcript.
3. Calls ~tlon-meet-format-transcript~ which uses AI to generate a formatted Markdown (=.md=) file from the raw transcript, deletes the =.txt= file, then calls ~tlon-meet-cleanup-transcript~ to refine the Markdown using AI.
4. If a CALLBACK is provided (e.g., from ~tlon-meet-transcribe-and-summarize~), it is called after cleanup with the path to the final =.md= file.

This command requires the external =whisperx= tool and a valid Hugging Face token for diarization.

*** ~tlon-meet-summarize-transcript~
:PROPERTIES:
:CUSTOM_ID: h:tlon-meet-summarize-transcript
:END:
#+findex: tlon-meet-summarize-transcript

Generate an AI summary for a formatted Markdown transcript file and save results to the appropriate meeting repository.

When called interactively, you are prompted to select a =.md= transcript file and to confirm participants (used to determine the target repository). If participants cannot be determined, you are prompted to select the meeting repository manually.

The command:

1. Extracts the meeting date from the filename.
2. Determines the target repository based on participants (two participants map to an individual meeting repo; more than two or ="group"= maps to the group meeting repo).
3. Sends the transcript content to the AI with ~tlon-meet-summarize-transcript-prompt~.
4. On success, saves the summary to =meeting-summaries.org= and copies the transcript to =transcripts/YYYY-MM-DD.md= in the repository.
5. Commits and pushes the changes, then deletes the original input transcript file.

*** ~tlon-meet-transcribe-and-summarize~
:PROPERTIES:
:CUSTOM_ID: h:tlon-meet-transcribe-and-summarize
:END:
#+findex: tlon-meet-transcribe-and-summarize

Transcribe, format, clean up, and summarize an audio file in one step. This is the main end-to-end command for processing a new recording.

When called interactively, you are prompted for the audio/video file and participants. The command:

1. Calls ~tlon-meet-transcribe-audio~ to handle transcription, AI formatting, and AI cleanup.
2. Sets up a callback that, once the cleaned =.md= transcript is ready, automatically calls ~tlon-meet-summarize-transcript~ to generate the summary and save everything to the meeting repository.

This command can also be triggered automatically by the file monitoring system ([[#h:file-monitoring][file monitoring]]).

:PROPERTIES:

** Issue handling

*** ~tlon-create-or-visit-meeting-issue-for-pair~
:PROPERTIES:
:CUSTOM_ID: h:tlon-create-or-visit-meeting-issue-for-pair
:END:
#+findex: tlon-create-or-visit-meeting-issue-for-pair

Create or visit the Forge issue for the meeting pair identified by PAIR-KEY on DATE. PAIR-KEY must be a key in ~tlon-meeting-pairs~ (e.g., ="leo-pablo"=, ="fede-pablo"=, ="fede-leo"=). The function determines which participant in the pair is the "other" person (i.e., not ~user-full-name~) and delegates to ~tlon-create-or-visit-individual-meeting-issue~.

*** ~tlon-create-or-visit-meeting-issue-date~
:PROPERTIES:
:CUSTOM_ID: h:tlon-create-or-visit-meeting-issue-date
:END:
#+findex: tlon-create-or-visit-meeting-issue-date

Create or visit the Forge issue in DIR for a meeting on DATE. This is the core logic: it synchronizes Forge data via ~tlon-wait-until-forge-updates~, looks up an existing issue matching the date, and either visits it or creates a new one via ~tlon-create-and-visit-issue~.

*** ~tlon-create-and-visit-issue~
:PROPERTIES:
:CUSTOM_ID: h:tlon-create-and-visit-issue
:END:
#+findex: tlon-create-and-visit-issue

Create a new Forge issue with TITLE in DIR, synchronize, wait for the issue to appear locally (up to 30 retries at 0.5-second intervals), and then visit it.

*** ~tlon-wait-until-forge-updates~
:PROPERTIES:
:CUSTOM_ID: h:tlon-wait-until-forge-updates
:END:
#+findex: tlon-wait-until-forge-updates

Synchronously pull Forge data for the current repository. This ensures subsequent lookups see the latest issues. Uses ~tlon-forg--pull-sync~ internally.

* Org-mode integration
:PROPERTIES:
:CUSTOM_ID: h:org-mode-integration

** ~tlon-set-meeting-buffers~
:PROPERTIES:
:CUSTOM_ID: h:tlon-set-meeting-buffers
:END:
#+findex: tlon-set-meeting-buffers

Open the meeting link and create or visit the appropriate meeting issue. This function is designed to run as advice on ~org-extras-agenda-goto-and-start-clock~. It parses the current Org heading text to identify the meeting type (e.g., "Leo<>Pablo", "Fede<>Pablo", "Group meeting") and:

1. Opens the meeting URL found in the heading via ~tlon-open-meeting-link~.
2. Calls the corresponding meeting issue command for the current date.
3. Sets the Forge issue's GitHub project status to "DOING" via ~tlon-meet--project-set-doing~.

Only runs on the machine with ~system-name~ equal to ="Pablos-MacBook-Pro.local"=.

** ~tlon-goto-meeting-when-clocking-in~
:PROPERTIES:
:CUSTOM_ID: h:tlon-goto-meeting-when-clocking-in
:END:
#+findex: tlon-goto-meeting-when-clocking-in

Go to the meeting issue when clocking in. This hook function is intended for use with ~org-clock-in-hook~. When clocking into an Org mode task, it calls ~tlon-set-meeting-buffers~ to automatically open the relevant meeting issue.

** ~tlon-open-meeting-link~
:PROPERTIES:
:CUSTOM_ID: h:tlon-open-meeting-link
:END:
#+findex: tlon-open-meeting-link

Open the meeting link (e.g., a Google Meet or Zoom URL) found in the current Org heading. The function moves to the heading's metadata and searches for a URL using ~browse-url-button-regexp~, then opens it with ~org-open-at-point~.

* File monitoring
:PROPERTIES:
:CUSTOM_ID: h:file-monitoring
:END:

This module can monitor a recording directory for new files and automatically trigger the transcription and summarization pipeline.

** ~tlon-meet-watch-recordings~
:PROPERTIES:
:CUSTOM_ID: h:tlon-meet-watch-recordings
:END:
#+findex: tlon-meet-watch-recordings

Start monitoring the default recording directory for new files. Uses Emacs' ~file-notify-add-watch~ to watch the directory specified by ~tlon-default-conference-app~. When a new file is created, ~tlon-meet--handle-new-recording~ is called to determine if it should be processed.

This function is automatically called at load time on Pablo's machine (when ~system-name~ is ="Pablos-MacBook-Pro.local"=).

** ~tlon-meet-unwatch-recordings~
:PROPERTIES:
:CUSTOM_ID: h:tlon-meet-unwatch-recordings
:END:
#+findex: tlon-meet-unwatch-recordings

Stop monitoring the recording directory for new files. Removes the file notification watch established by ~tlon-meet-watch-recordings~.
